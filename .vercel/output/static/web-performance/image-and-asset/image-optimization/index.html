Image Optimization

ရည်ရွယ်ချက်

သင့်လျော်တဲ့ ပုံ Format တွေကို ရွေးချယ်တတ်ဖို့၊ ထိရောက်တဲ့ ချုံ့နည်း (compression) တွေကို အသုံးပြုတတ်ဖို့၊ ဖုန်း၊ ကွန်ပျူတာ အစရှိတဲ့ ဖန်သားပြင် အရွယ်အစားမျိုးစုံနဲ့ ကိုက်ညီအောင် ပုံတွေကို ပြင်ဆင်တတ်ဖို့ (responsive images)၊ ပြီးတော့ ပုံတွေကြောင့် website နှေးတာကို သိသိသာသာ လျှော့ချနိုင်ဖို့အတွက် လိုအပ်မှ တင်တဲ့စနစ် (lazy loading) ကို အသုံးပြုတတ်ဖို့ သင်ယူရပါမယ်။

ဘာကြောင့် ပုံတွေကို အထူးဂရုစိုက်သင့်လဲ Website ရဲ့ ဖိုင်အရွယ်အစား (byte size) အများစုက ပုံတွေကြောင့် ဖြစ်တတ်ပါတယ်။ ပုံတွေကို ကောင်းကောင်း မပြင်ဆင်ထားရင် website ကို သိသိသာသာ နှေးစေနိုင်ပြီး သုံးစွဲသူတွေရဲ့ data တွေကိုလည်း အလဟဿ ကုန်စေပါတယ်။

သင့်လျော်တဲ့ ပုံ Format ကို ရွေးချယ်ခြင်း ပုံအမျိုးအစား အမျိုးမျိုးအတွက် မတူညီတဲ့ Format တွေက ပိုသင့်တော်ပါတယ်။

JPEG (သို့မဟုတ် JPG):

အသင့်တော်ဆုံး။ ဓာတ်ပုံများ၊ အရောင်အသွေးများပြီး အရောင် gradient တွေ ပါတဲ့ ပုံတွေ။ အားသာချက်။ ဓာတ်ပုံတွေအတွက် ကောင်းကောင်း ချုံ့နိုင်တယ်။ browser အများစုက ပံ့ပိုးတယ်။ အားနည်းချက်။ ချုံ့ရင် ပုံရဲ့ အရည်အသွေး နည်းနည်း ကျသွားနိုင်တယ်။ နောက်ခံ ပွင့်လင်း (transparency) မှုကို ပံ့ပိုးမပေးဘူး။ PNG (Portable Network Graphics):

အသင့်တော်ဆုံး။ နောက်ခံ ပွင့်လင်းဖို့ လိုအပ်တဲ့ ပုံတွေ (လိုဂိုများ၊ icon များ)၊ စာသားတွေ ဒါမှမဟုတ် မျဉ်းသားရှင်းရှင်း ရှိပြီး အရည်အသွေး အရမ်းအရေးကြီးတဲ့ ပုံတွေ (ဥပမာ- screenshot များ)။ အားသာချက်။ ချုံ့ရင် အရည်အသွေး မကျဘူး (PNG-24 အတွက်)။ နောက်ခံ ပွင့်လင်းမှုကို ပံ့ပိုးတယ်။ အားနည်းချက်။ ဓာတ်ပုံတွေအတွက် JPEG ထက် ဖိုင်အရွယ်အစား ပိုကြီးနိုင်တယ်။ PNG-8 ကတော့ အရောင်နည်းနည်းပဲ ပါတဲ့ ပုံတွေအတွက် ဖိုင်သေးနိုင်တယ်။ WebP:

အသင့်တော်ဆုံး။ JPEG နဲ့ PNG နှစ်မျိုးလုံးကို အစားထိုးဖို့အတွက် ခေတ်မီတဲ့ Format ပါ။ အရည်အသွေး မကျဘဲ ချုံ့နိုင်သလို၊ အရည်အသွေး နည်းနည်းလျှော့ပြီးလည်း ချုံ့နိုင်တယ်။ နောက်ခံပွင့်လင်းမှုနဲ့ လှုပ်ရှားတဲ့ပုံ (animation) တွေကိုလည်း ပံ့ပိုးတယ်။ အားသာချက်။ တူညီတဲ့ အရည်အသွေးမှာ JPEG/PNG ထက် ဖိုင်အရွယ်အစား သိသိသာသာ သေးငယ်တယ်။ ခေတ်မီတဲ့ browser အများစုက ပံ့ပိုးတယ်။ အားနည်းချက်။ အရမ်းဟောင်းတဲ့ browser တွေက မပံ့ပိုးပေမဲ့ အစားထိုး (fallback) ပုံတွေ ထည့်ထားလို့ရတယ်။ AVIF (AV1 Image File Format):

အသင့်တော်ဆုံး။ WebP နဲ့ ဆင်တူပေမဲ့ ပိုပြီး ကောင်းမွန်တဲ့ ချုံ့နှုန်းကို ပေးနိုင်တယ်။ အထူးသဖြင့် ဖိုင်အရွယ်အစား သေးငယ်တဲ့နေရာမှာ ပိုကောင်းတယ်။ အားသာချက်။ WebP ထက် ချုံ့နှုန်း ပိုကောင်းတယ်။ HDR (High Dynamic Range) ကို ပံ့ပိုးတယ်။ browser ပံ့ပိုးမှု တဖြည်းဖြည်း တိုးတက်လာနေပြီ။ အားနည်းချက်။ ပိုပြီး အသစ်ဖြစ်တဲ့ Format ဖြစ်လို့ WebP ထက် browser ပံ့ပိုးမှုက အားနည်းသေးပေမဲ့ အလျှင်အမြန် တိုးတက်နေပါတယ်။ SVG (Scalable Vector Graphics):

အသင့်တော်ဆုံး။ လိုဂိုများ၊ icon များ၊ အရည်အသွေး မကျဘဲ ဘယ်အရွယ်အစားမဆို ချဲ့နိုင်ဖို့ လိုအပ်တဲ့ ရိုးရှင်းတဲ့ ပုံတွေ။ XML-based ဖြစ်ပါတယ်။ အားသာချက်။ အရွယ်အစား ချဲ့လို့ရတယ်။ ဖိုင်အရွယ်အစား သေးငယ်လေ့ရှိတယ်။ CSS နဲ့ JavaScript နဲ့ ပြုပြင်လို့ရတယ်။ အားနည်းချက်။ ဓာတ်ပုံတွေအတွက် မသင့်တော်ဘူး။ ချုံ့နည်းများ (Compression Techniques): အရွယ်အစားကို လျှော့ချခြင်း Lossy Compression: ပုံအချက်အလက်တချို့ကို ဖယ်ရှားပြီး ဖိုင်အရွယ်အစားကို လျှော့ချတာပါ။ ပိုပြီး ပြင်းပြင်းထန်ထန် ချုံ့ရင် ဖိုင်ပိုသေးပေမဲ့ ပုံအရည်အသွေး သိသိသာသာ ကျဆင်းနိုင်ပါတယ်။ (ဥပမာ- JPEG, WebP (lossy), AVIF (lossy)။) Lossless Compression: ပုံအရည်အသွေး မကျဘဲ ဖိုင်အရွယ်အစားကို လျှော့ချတာပါ။ မလိုအပ်တဲ့ အချက်အလက် (metadata) တွေကို ဖယ်ရှားခြင်း ဒါမှမဟုတ် အချက်အလက်တွေကို ပိုထိရောက်အောင် စီစဉ်ခြင်းဖြင့် လုပ်ဆောင်ပါတယ်။ (ဥပမာ- PNG, WebP (lossless), AVIF (lossless), SVG တွေကို ကောင်းမွန်အောင် လုပ်ဆောင်ခြင်း။) ချုံ့ဖို့ ကိရိယာများ:

Online ကိရိယာများ။ TinyPNG/TinyJPG, Squoosh.app, iLoveIMG Desktop App များ။ ImageOptim (Mac), RIOT (Windows) Build tools/plugins များ။ imagemin (Node.js project များအတွက်) ပန်းတိုင်။ ဖိုင်အရွယ်အစားနဲ့ လက်ခံနိုင်တဲ့ ပုံအရည်အသွေးကြား အကောင်းဆုံး ဟန်ချက်ကို ရှာဖွေဖို့ပါ။

ဖန်သားပြင် အရွယ်အစားမျိုးစုံအတွက် ပုံများ ပြင်ဆင်ခြင်း (Responsive Images) ဖုန်း ဖန်သားပြင်သေးသေးလေးမှာ အရမ်းကြီးတဲ့ ကွန်ပျူတာ ပုံတစ်ပုံကို ပြသတာဟာ အင်တာနက် data ကို ဖြုန်းတီးစေပြီး website ကို နှေးစေပါတယ်။ Responsive images တွေက ဒီပြဿနာကို ဖြေရှင်းပေးပါတယ်။

srcset attribute (<img> အတွက်): မတူညီတဲ့ အရွယ်အစား/အရည်အသွေးရှိတဲ့ ပုံ Source တွေကို ပေးနိုင်ပါတယ်။ ဒါဆိုရင် browser က device ရဲ့ ဖန်သားပြင်အရွယ်အစား၊ pixel သိပ်သည်းဆ (pixel density) နဲ့ မြင်ကွင်း (viewport) အပေါ် မူတည်ပြီး အသင့်တော်ဆုံး ပုံကို ရွေးချယ်ပါလိမ့်မယ်။
