Email နဲ့ Notification Automation

ဒီသင်ခန်းစာမှာ ဆောက်မယ့် Workflows

Section titled “ဒီသင်ခန်းစာမှာ ဆောက်မယ့် Workflows”

ဒီမှာ လက်တွေ့သုံးလို့ရတဲ့ Notification Workflow (၃) ခု ဆောက်ပါမယ် -

  1. Gmail → Slack Forwarding (Email ဝင်ရင် Slack ကနေ အကြောင်းကြားဖို့)
  2. Form Submit → Email + Slack Notification (Contact Form တင်တာနဲ့ အကြောင်းကြားဖို့)
  3. Schedule Daily Report → Email (နေ့စဉ် Report တွေကို အလိုအလျောက် ပို့ဖို့)

Workflow 1: Gmail → Slack Forwarding

Section titled “Workflow 1: Gmail → Slack Forwarding”

Important Label ပါတဲ့ Email ဝင်လာတိုင်း Slack ကနေ လှမ်း Notify ပေးပါမယ် -

⚙ Gmail Trigger: New Email (Poll: 1 min)
◆ Label = "IMPORTANT"?
YES ✅
↓
⚙ Slack: Post Message in #alerts
NO ❌
↓
⚙ Do nothing

Gmail Trigger Setup

Section titled “Gmail Trigger Setup”
  1. Node Panel ထဲမှာ Gmail ကို Search လုပ်ပါ
  2. Gmail Trigger ကို ရွေးပါ
  3. Credential အနေနဲ့ Google OAuth2 ချိတ်ဆက်ပါ (ရှေ့ Credential သင်ခန်းစာမှာ လုပ်ခဲ့သလိုပါပဲ)

Gmail Trigger Configuration -

  • Event: New Email
  • Label: IMPORTANT
  • Poll Time: Every 1 minute

Section titled “Workflow 2: Contact Form → Multi-channel Notification”

Error Handling အခြေခံ

Section titled “Error Handling အခြေခံ”

Email ဒါမှမဟုတ် Notification ပို့တာ Fail ဖြစ်သွားရင် Retry နဲ့ Fallback လုပ်နည်း -

⚙
Gmail: Send Email
❌ Error?
⚙
Wait: 5 minutes Error ဖြစ်ရင်
Retry
⚙
Gmail: Retry Send
❌ Still Error?
⚙
Slack: Alert Still error ဆိုရင်

Email Fail → Wait → Retry → Slack Alert (Fallback လမ်းကြောင်း)
