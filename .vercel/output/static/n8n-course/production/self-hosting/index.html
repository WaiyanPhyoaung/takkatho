n8n Cloud (အသင့်သုံး)Self-Hosted (ကိုယ်တိုင်တင်)

Workflow အကန့်အသတ်Plan အပေါ် မူတည်ပြီး ကန့်သတ်ချက်ရှိတယ်ကန့်သတ်ချက်မရှိ (Unlimited)
Data Privacy (လုံခြုံရေး)Cloud ပေါ်မှာ သိမ်းတယ်ကိုယ်ပိုင် Server ထဲမှာမို့ ပိုလုံခြုံတယ်
Customizationလိုသလို ပြင်လို့ သိပ်မရဘူးFull Control ရတယ်၊ စိတ်ကြိုက်လုပ်လို့ ရတယ်
Maintenanceသူတို့ဘာသာ လုပ်ပေးတယ်ကိုယ်တိုင် Update လုပ်ရမယ်၊ ထိန်းသိမ်းရမယ်

Nginx Reverse Proxy နဲ့ SSL (HTTPS) ချိတ်ဆက်နည်း

Section titled “Nginx Reverse Proxy နဲ့ SSL (HTTPS) ချိတ်ဆက်နည်း”

တကယ့် Production မှာ n8n ကို လုံခြုံတဲ့ HTTPS (လော့ခ်ချိတ်) နဲ့ သုံးဖို့အတွက် Nginx နဲ့ Let’s Encrypt ကို သုံးပါမယ် -

▶
Internet HTTPS :443 ဝင်လာမယ်
⚙
Nginx SSL ကို Nginx က ဖြေရှင်းမယ်
✓
n8n localhost:5678 ဆီ လွှဲပေးမယ်

Nginx ကို အလယ်ခံ (Reverse Proxy) အဖြစ်ထားပြီး HTTPS နဲ့ လာသမျှကို n8n ဆီ လွှဲပေးတဲ့ ပုံစံပါ

Domain ရဲ့ DNS ချိတ်ဆက်မှု

ကိုယ့် Domain ရဲ့ DNS Setting ထဲက A Record မှာ VPS ရဲ့ IP Address ကို သွားညွှန်ထားဖို့ မမေ့ပါနဲ့နော် -

DNS ချိတ်ဆက်ပြီးရင် အပြည့်အဝ အလုပ်လုပ်ဖို့ (Propagation) (၁) မိနစ်၊ (၂) မိနစ်လောက်တော့ စောင့်ပေးရပါမယ်။

Backup Strategy (Data တွေ အရန်သိမ်းဆည်းနည်း)

Section titled “Backup Strategy (Data တွေ အရန်သိမ်းဆည်းနည်း)”
☰
n8n Workflow Export JSON ဖိုင်အနေနဲ့ ထုတ်မယ်
☰
PostgreSQL Dump Database ကြီးကို pg_dump နဲ့ ထုတ်မယ်
⚙
Upload to S3 / GDrive VPS အပြင်ဘက်ကို တင်သိမ်းမယ်

နေ့စဉ် (Daily) Workflows တွေနဲ့ Database တွေကို အရန်သိမ်းဆည်းခြင်း (Backup)

Workflow တွေကို Git နဲ့ Version Control လုပ်ထားပါ

n8n ထဲက Workflows တွေကို JSON အနေနဲ့ Export ထုတ်ပြီး ကိုယ့်ရဲ့ Git Repository (ဥပမာ - GitHub/GitLab) ထဲကို Push လုပ်သိမ်းထားတာ အကောင်းဆုံးပါ။ အဖွဲ့လိုက် အလုပ်လုပ်တဲ့နေရာတွေ (Team Collaboration) နဲ့ အမှားတစ်ခုခုဖြစ်ရင် နောက်ပြန်ဆုတ်ဖို့ (Rollback) လုပ်တဲ့နေရာတွေမှာ အရမ်း အသုံးဝင်ပါတယ် -

နောက်ဆုံး သင်ခန်းစာအနေနဲ့ Capstone Project ပါ — ဒီ Course တစ်ခုလုံးမှာ လေ့လာခဲ့သမျှ သင်ခန်းစာတွေ အားလုံးကို ပေါင်းစပ်ပြီး တကယ့် လက်တွေ့သုံးလို့ရမယ့် Real E-commerce Automation System အကြီးကြီးတစ်ခုကို အတူတူ တည်ဆောက်ကြည့်ကြပါမယ် ခင်ဗျာ!
