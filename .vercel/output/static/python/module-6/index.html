File Handling နှင့် Error Handling

ဒီ Module ပြီးရင် သင်လုပ်နိုင်မှာများ

  • Text File ဖတ်ရေးနိုင်မည် (encoding=‘utf-8’ ပါ)
  • JSON File ဖတ်ရေးနိုင်မည်
  • ဖိုင်ရှိမရှိ စစ်ဆေးနိုင်မည် (os.path.exists)
  • try/except ဖြင့် Error ထိန်းချုပ်နိုင်မည်
  • raise ဖြင့် Custom Error တက်ကိုင်နိုင်မည်

ဒီ Module မစခင်

Module 1-5 ကို ပြီးမှ ဆက်ပါ။ Functions နှင့် Data Structures (Dict) ကို သိပြီးမှ File မှာ Data သိမ်းနည်းကို ကောင်းကောင်းနားလည်နိုင်မည်။

Program တွေ ရေးတဲ့အခါ Data တွေကို Memory ထဲမှာပဲ သိမ်းထားလို့ မရပါဘူး — Program ပိတ်တာနဲ့ ပျောက်သွားမှာမို့ File တွေထဲမှာ သိမ်းဖို့ လိုပါတယ်။

File Handling

Section titled “File Handling”

Python မှာ File တွေကို open() Function ကို သုံးပြီး ဝင်ရောက်ပါတယ်။ Mode (၃) မျိုး ရှိပါတယ်:

  • "r" — ဖတ်ဖို့ (Read)
  • "w" — ရေးဖို့ (Write — ရှိပြီးသားဆိုရင် ဖျက်ပြီး အသစ်ရေးမည်)
  • "a" — ဆက်ထည့်ဖို့ (Append — အနောက်ကနေ ဆက်ရေးမည်)

encoding='utf-8' — မမေ့ပါနဲ့

Myanmar Unicode နဲ့ ရေးထားသော Text တွေဆိုရင် encoding='utf-8' မပါရင် ဖတ်ရတဲ့အခါ Error တက်နိုင်ပါတယ် (Windows မှာ အထူးသဖြင့်)။ File တိုင်းမှာ encoding ကို ဖော်ပြပါ:

Section titled “File ထဲကို ရေးခြင်း”

with open(...) ကို ဘာကြောင့် သုံးမလဲ? with Block ပြီးသွားတာနဲ့ File ကို အလိုအလျောက် ပိတ် (Close) ပေးပါတယ်။ ကိုယ်တိုင် file.close() ရိုက်စရာ မလိုပါဘူး — ပိတ်မပြီးဘဲ မနေနိုင်ပါ။

JSON (JavaScript Object Notation) ဆိုတာ Data ကို ဘာသာစကားမဆိုဖတ်နိုင်သော Format နဲ့ သိမ်းနိုင်သော Standard ပါ — Python Dictionary ပုံစံနဲ့ တူပြီး Program ၂ ခုကြားနဲ့ Data ပြောင်းဖို့ API Response Format အဖြစ် ကမ္ဘာတစ်ဝှမ်း သုံးကြပါတယ်:

json.dump() Params

  • ensure_ascii=False — Myanmar Unicode ကို မပြောင်းဘဲ ရေးနိုင်မည်
