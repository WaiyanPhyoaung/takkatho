ဖိုင်တွေ များလာတာနဲ့အမျှ လိုချင်တဲ့ ဖိုင် (သို့မဟုတ်) ဖိုင်ထဲက စာကြောင်းကို ရှာဖို့ ခက်ခဲလာပါတယ်။ အခု သင်ခန်းစာမှာ ရှာဖွေရလွယ်ကူစေမယ့် commands တွေနဲ့ အခြားစွမ်းရည်တွေကို လေ့လာပါမယ်။

### 1. `find` (ဖိုင်များကို ရှာဖွေခြင်း)
ကွန်ပျူတာထဲမှာရှိတဲ့ ဖိုင်တွေ၊ Folder တွေကို နာမည်၊ အမျိုးအစား၊ အရွယ်အစား စတာတွေပေါ် မူတည်ပြီး ရှာဖွေနိုင်ပါတယ်။

```bash
# လက်ရှိ Folder နေရာကစပြီး index.html ဆိုတဲ့ ဖိုင်ကို ရှာရန်
find . -name "index.html"

# Extension .js နဲ့ ဆုံးတဲ့ ဖိုင်အားလုံးကို ရှာရန်
find . -name "*.js"
```

### 2. `grep` (စာသားများကို ရှာဖွေခြင်း)
`grep` ဟا ကမ္ဘာပေါ်မှာ အသုံးအများဆုံး Command ထဲက တစ်ခုဖြစ်ပါတယ်။ ဖိုင်ထဲမှာရှိတဲ့ စာသားတွေ (Text) ထဲကနေ ကိုယ်လိုချင်တဲ့ စာလုံးပါတဲ့ စာကြောင်းတွေကို ဆွဲထုတ်ပေးပါတယ်။

```bash
# error.log ဖိုင်ထဲက "error" ဆိုတဲ့ စာလုံးပါတဲ့ အကြောင်းအရာတွေကို ရှာရန်
grep "error" error.log

# စာလုံးအကြီး/အသေး မခွဲဘဲ (case-insensitive) ရှာရန် 
grep -i "warning" error.log
```

### 3. ပိုက်လိုင်း Pipe (`|`)
Pipe (`|`) သင်္ကေတက **Command တစ်ခုက ထွက်လာတဲ့ ရလဒ် (Output) ကို နောက် Command တစ်ခုဆီသို့ (Input အဖြစ်) ပို့ပေးတာ** ဖြစ်ပါတယ်။ (Enter ပေါ်က ခလုတ်ကို Shift ဖိပြီး နှိပ်ရပါတယ်)

```bash
# ဖိုင်ထဲက အရာတွေကို cat နဲ့ထုတ်မယ်၊ ပြီးရင် "fail" ပါတဲ့စာကြောင်းကိုပဲ grep နဲ့ စစ်ထုတ်မယ်
cat server.log | grep "fail"

# ဖိုင်စာရင်း (ls) ကြည့်မယ်၊ အဲ့အထဲကမှ .md လို့ပါတဲ့ ဖိုင်တွေကိုပဲ စစ်ထုတ်မယ်
ls | grep ".md"
```

### 4. ရလဒ်များကို သိမ်းဆည်းခြင်း (Redirection)
Command တွေ run လို့ထွက်လာတဲ့ ရလဒ် (Output) တွေကို Terminal မှာပဲ မကြည့်ဘဲ ဖိုင်တစ်ခုအနေနဲ့ သိမ်းချင်ရင် `>` သို့မဟုတ် `>>` ကို သုံးနိုင်ပါတယ်။

* **`>` (Overwrite)** : ဖိုင်အသစ်ဆောက်ပြီး သိမ်းတယ်။ (ရှိပြီးသားဖိုင်ဆိုရင် အဟောင်းတွေပျက်ပြီး အသစ်တွေ ဝင်သွားမယ်)
* **`>>` (Append)** : ရှိပြီးသားဖိုင်ရဲ့ အောက်ခြေမှာ စာကြောင်းအသစ် ထပ်ပြီးဖြည့်ထည့်တယ်။

```bash
# "Hello World" ဆိုတဲ့ စာကို hello.txt ဖိုင်ထဲကို သိမ်းမယ်
echo "Hello World" > hello.txt

# ရှိပြီးသား hello.txt ဖိုင်ရဲ့အောက်မှာ စာကြောင်းအသစ် ထပ်ဖြည့်မယ်
echo "Welcome to CLI" >> hello.txt

# ls က ထွက်လာတဲ့ စာရင်းတွေကို list.txt ဆိုတဲ့ ဖိုင်ထဲ သိမ်းမယ်
ls > list.txt
```
