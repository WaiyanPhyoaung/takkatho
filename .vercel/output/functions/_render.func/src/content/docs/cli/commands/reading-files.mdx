Terminal ထဲမှာ ဖိုင်တွေကို ရှာဖွေ၊ ဖန်တီး၊ ပြောင်းရွှေ့ပြီးတဲ့နောက်မှာ အဲ့ဒီဖိုင်တွေထဲက အကြောင်းအရာတွေကို ဖွင့်ဖတ်ဖို့ လိုအပ်လာပါပြီ။ GUI (Graphic User Interface) မှာ Notepad (သို့) VS Code ကို သုံးသလိုပဲ၊ CLI မှာလည်း စာသား (Text) ဖိုင်တွေကို ဖတ်ဖို့ သီးသန့် Command တွေ ရှိပါတယ်။

### 1. `cat` (Concatenate)
`cat` က ဖိုင်ထဲမှာရှိသမျှ စာသားတွေအားလုံးကို Terminal screen ပေါ်မှာ တစ်ခါတည်း ချပြပေးပါတယ်။ ဖိုင်သေးသေးလေးတွေ ဖတ်ဖို့အတွက် အကောင်းဆုံးပါပဲ။

```bash
# text.txt ဖိုင်ထဲက အရာများကို ဖတ်ရန်
cat text.txt
```

> 💡 **Tip:** ဖိုင် ၂ ခုကိုပေါင်းပြီး ဖတ်ချင်ရင် `cat file1.txt file2.txt` ဆိုပြီး ရေးလို့ရပါတယ်။

### 2. `less`
ဖိုင်က အရမ်းရှည်နေတဲ့အခါ `cat` ကို သုံးရင် အပေါ်ဆုံးက အကြောင်းအရာတွေကို ကျော်သွားတတ်ပါတယ်။ `less` ကိုသုံးပါက တစ်မျက်နှာချင်းစီ အေးဆေးဖတ်လို့ရပါတယ်။

```bash
less long_file.txt
```
* **Spacebar** နှိပ်လျှင် နောက်တစ်မျက်နှာသို့ သွားပါမည်။
* **b** ကိုနှိပ်လျှင် အနောက်တစ်မျက်နှာသို့ ပြန်ဆုတ်ပါမည်။
* ထွက်ချင်လျှင် **`q`** (quit) ကို နှိပ်ရပါမည်။

### 3. `head`
ဖိုင်တစ်ဖိုင်ရဲ့ **အပေါ်ထိပ်ဆုံးက စာကြောင်းတွေကိုပဲ** အမြန်ဖတ်ချင်တဲ့အခါ သုံးပါတယ်။ (ဥပမာ - log ဖိုင်တွေရဲ့ အစကို ဖတ်ချင်တဲ့အခါ)

```bash
# ပုံမှန်အားဖြင့် ထိပ်ဆုံး ၁၀ ကြောင်းကို ပြပေးသည်
head document.txt

# ထိပ်ဆုံး ၅ ကြောင်းကိုသာ ကြည့်လိုလျှင် -n ကိုသုံးပါ
head -n 5 document.txt
```

### 4. `tail`
`head` ရဲ့ ပြောင်းပြန်ပါ။ ဖိုင်တစ်ဖိုင်ရဲ့ **အောက်ဆုံး (နောက်ဆုံး) က စာကြောင်းတွေကိုပဲ** ဖတ်ချင်တဲ့အခါ သုံးပါတယ်။ Server log တွေမှာ ပြဿနာအသစ် ဘာတက်နေလဲဆိုတာ ကြည့်ဖို့ အများဆုံး သုံးပါတယ်။

```bash
# ပုံမှန်အားဖြင့် အောက်ဆုံး ၁၀ ကြောင်းကို ပြပေးသည်
tail server.log

# အောက်ဆုံး ၅ ကြောင်းကိုသာ ကြည့်လိုလျှင်
tail -n 5 server.log

# 🚨 Server အလုပ်လုပ်နေတုန်း အသစ်ဝင်လာတဲ့ data တွေကို Live ဖတ်ချင်ရင် -f (follow) ကိုသုံးပါ
tail -f server.log
```
