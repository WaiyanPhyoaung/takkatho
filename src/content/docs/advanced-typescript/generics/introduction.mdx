// ၁။ Function နာမည်ရဲ့ နောက်မှာ Generic Type Variable <T> ကို ကြေညာမယ်။
// ၂။ လက်ခံမယ့် argument ရဲ့ Type ကို T လို့ သုံးမယ်။
// ၃။ ပြန်ထုတ်မယ့် return Type ကိုလည်း T လို့ သုံးမယ်။

console.log(\`received data's Type: ${typeof arg}\`);
return arg;
}

ဒီလို အခက်အခဲတွေကို ဖြေရှင်းဖို့ TypeScript မှာ ပိုအစွမ်းထက်တဲ့ Features တွေ ရှိပါတယ်။ အဲဒါတွေကတော့ Generics နဲ့ Utility Types တွေပဲ ဖြစ်ပါတယ်။

## Generic ဆိုတာ တကယ်တော့ ဘာလဲ? "Type Variable" ဆိုတဲ့ Idea

Generics ရဲ့ အဓိက Concept ကတော့ - Type တွေအတွက် နေရာယူပေးထားတာပါပဲ။ သူက Code တွေ ထပ်နေတဲ့ ပြဿနာကို ဘယ်လို ဖြေရှင်းပေးသလဲ ဆိုတာရယ်၊ အခြေခံ Generic Function လေးတစ်ခု ဘယ်လို ရေးရမလဲ ဆိုတာကို လေ့လာသွားမှာ ဖြစ်ပါတယ်။

အရင် သင်ခန်းစာမှာ တွေ့ခဲ့ရတဲ့ Code တွေ ထပ်နေတဲ့ ပြဿနာကို ဖြေရှင်းဖို့ Generics က ကူညီပေးနိုင်ပါတယ်။ စဉ်းစားကြည့်ပါ ခင်ဗျာ - Function တစ်ခုထဲကို Data အမျိုးမျိုး (`number`, `string`) ထည့်သုံးချင်ရင် Parameter တွေ ပေးရတယ် မဟုတ်လား။ Generics ကလည်း အဲဒီလိုပါပဲ၊ Function ကို Type အမျိုးမျိုး ထည့်သုံးလို့ရအောင် ကူညီပေးတာပါ။ သူတို့က Code ရေးတဲ့အချိန်မှာ ဘယ် Type ဖြစ်မလဲဆိုတာ မသိသေးခင် Type တွေအတွက် နေရာယူထားပေးတဲ့ Placeholders လေးတွေ၊ ဒါမှမဟုတ် Type တွေကို ကိုယ်စားပြုပေးတဲ့ Variables လေးတွေ ဖြစ်ပါတယ်။

<Aside>
Generic Type Variable ကို ကြေညာချင်ရင် Angle Bracket `< >` လေးတွေ သုံးရပါတယ်။ နာမည်ကို `T` (Type ကို ကိုယ်စားပြုတယ်) လို့ အများဆုံး သုံးလေ့ရှိပေမဲ့ `U`, `K`, `V` စသဖြင့် ကိုယ်ကြိုက်တဲ့ နာမည်လည်း ပေးလို့ရပါတယ် ခင်ဗျာ။
</Aside>

<Code code={code} lang="typescript" title="index.ts" />

- `<T>`: ဒါက "ဒီ `identity` Function က Generic ဖြစ်တယ်၊ Type တစ်ခုခုနဲ့ အလုပ်လုပ်မယ်၊ အဲဒီ Type ကို လောလောဆယ် `T` လို့ ခေါ်ထားမယ်" လို့ ပြောလိုက်တာပါ။
- `arg: T`: Function က လက်ခံမယ့် `arg` ဆိုတဲ့ Data ရဲ့ Type ဟာ `T` ကိုယ်စားပြုထားတဲ့ Type ပဲ ဖြစ်ပါလိမ့်မယ်။
- `: T`: Function က ပြန်ထုတ်ပေးမယ့် တန်ဖိုးရဲ့ Type ကလည်း `T` ကိုယ်စားပြုထားတဲ့ Type အတိုင်းပဲ ဖြစ်ပါလိမ့်မယ်။

<Aside type="tip" title="ကောင်းကျိုး">
  ဒီလို ရေးလိုက်တော့ Logic (Code) ကို တစ်ခါတည်း ရေးလိုက်ရုံနဲ့တင်၊ ဘယ် Data Type
  နဲ့ လာလာ အလုပ်လုပ်နိုင်တဲ့ ပြန်လည် အသုံးပြုလို့ရတဲ့ (Reusable) Function တစ်ခု ရသွားပါပြီ။
  Code တွေ ထပ်ရေးစရာ မလိုတော့ဘူးပေါ့ ခင်ဗျာ။
</Aside>
