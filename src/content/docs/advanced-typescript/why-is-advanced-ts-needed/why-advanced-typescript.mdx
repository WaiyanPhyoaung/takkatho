## JavaScript ရဲ့ မထင်မှတ်ထားတဲ့ ပြဿနာ

ကျွန်တော်တို့ Code တွေ ရေးကြတယ်၊ Program တွေ တည်ဆောက်ကြတယ်။ အစပိုင်းမှာတော့ JavaScript က အရမ်းလွယ်ပါတယ်။ လိုချင်တာ အကုန် လိုက်လျောညီထွေ လုပ်လို့ရတယ်လေ။ ဒါပေမဲ့ တစ်ခါတလေကျတော့ ဒီ လိုက်လျောညီထွေ ရှိလွန်းတာကြီးကပဲ ပြဿနာ ပြန်ဖြစ်လာတတ်ပါတယ်။

ကျွန်တော်တို့ Code တွေ ရေးပြီးပြီ၊ အားလုံး မှန်တယ်လို့ ထင်နေတာပဲ။ ဒါပေမဲ့ Program ကို Run လိုက်တဲ့အခါကျမှ... "ဟင်၊ ဒီနေရာမှာ ဘာမှ မရှိပါလား?" "ဒီလို ဖြစ်သွားရတာလား?" ဆိုပြီး မထင်ထားတဲ့ Errors တွေ ပေါ်လာတတ်တယ်။ ဥပမာ - Data ပုံစံ မမှန်တာတို့၊ မှားယွင်းတဲ့ အချက်အလက် ထည့်သုံးမိတာတို့ပေါ့ ခင်ဗျာ။ ဒီလိုမျိုး Code Run နေရင်း ရုတ်တရက် ပေါ်လာတဲ့ Errors တွေက တကယ် ခေါင်းခဲစေပါတယ်။ အထူးသဖြင့် Code တွေ အများကြီး ဖြစ်လာပြီဆို ပိုဆိုးတာပေါ့။

ဒီလိုမျိုး Run တဲ့ အချိန်ကျမှ Error တက်တာကို ဒီ Code လေးနဲ့ နမူနာ ကြည့်ရအောင်ပါ ခင်ဗျာ။

```js
let userData = { name: "Alice" /* age မပါဘူးနော် */ };

function greetUser(user) {
  // user ထဲမှာ name ရှိတယ်လို့ ယူဆပြီး ရေးလိုက်တာ...
  console.log(`Hello, ${user.name.toUpperCase()}!`);
  // ဟာ... user ထဲက name မပါလာတဲ့အခါ Error တက်ပြီ!
}

greetUser(userData);
greetUser(null); // သေချာပေါက် Crash မှာပါ
```

### ရှင်းပြချက်

ဒီ Code လေးမှာဆိုရင် `userData` ထဲမှာ `name` ပဲပါပြီး `age` မပါတာတို့၊ ဒါမှမဟုတ် `greetUser` Function ကို `null` ဆိုတဲ့ ဘာမှ မဟုတ်တဲ့ Data နဲ့ ခေါ်လိုက်တာတို့ကြောင့် Program Run ရင်း Error တက်သွားတာ ဖြစ်ပါတယ်။ ဘာလို့လဲဆိုတော့ JavaScript က Code Run တဲ့အချိန်ကျမှသာ "အော်... ဒီ Data က ဒီလို ပုံစံပါလား" ဆိုတာ သိရတာကိုး ခင်ဗျာ။
