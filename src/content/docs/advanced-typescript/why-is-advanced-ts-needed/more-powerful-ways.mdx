ဒီလို အခက်အခဲတွေကို ဖြေရှင်းဖို့ TypeScript မှာ ပိုအစွမ်းထက်တဲ့ ကိရိယာတွေ ရှိတယ်။ အဲဒါတွေကတော့ Generics, Utility Types, နဲ့ satisfies Keyword တွေပဲ ဖြစ်တယ်။

Generics: Code တစ်ခုတည်းကိုပဲ Data Type အမျိုးမျိုးနဲ့ ပြန်လည် အသုံးပြုနိုင်အောင် လုပ်ပေးတယ်။ Code တွေ ထပ်ရေးရတာ သက်သာသွားတယ်။ (ပုံစံတူ လုပ်ဆောင်ချက်ကိုပဲ မတူညီတဲ့ Type တွေနဲ့ သုံးနိုင်စေတဲ့ မှော်ဆန်တဲ့ Box လိုမျိုးပေါ့။)
Utility Types: ရှိပြီးသား Data ပုံစံ (Type) တွေကနေ လိုအပ်သလို ပြောင်းထားတဲ့ Data ပုံစံအသစ်တွေ (Type အသစ်တွေ) ကို အလွယ်တကူ မြန်မြန် ဖန်တီးနိုင်စေတယ်။ (Data ပုံစံတွေကို အမြန် ပြင်ဆင်နိုင်တဲ့ ကိရိယာမျိုးစုံ ပါတဲ့ Box လိုမျိုးပေါ့။)
satisfies Keyword: Data တစ်ခုက သတ်မှတ်ထားတဲ့ ပုံစံ (Type) နဲ့ ကိုက်ညီမှန်း စစ်ဆေးပေးတယ်။ ဒါပေမယ့် အဲဒီ Data ရဲ့ တိကျတဲ့ အချက်အလက်လေးတွေကိုပါ မပျောက်ဘဲ သိမ်းထားပေးတယ်။ (Data ကို စစ်ဆေးပြီး အမှားရှိမရှိ ပြောပေမယ့် Data ရဲ့ မူရင်းအချက်အလက်ကို မပြောင်းလိုက်တာမျိုး။)
ရှင်းပြချက်: "ဒီအရာတွေကို လေ့လာခြင်းအားဖြင့် Program တွေ ရေးတဲ့အခါ Code တွေ ပိုပြီး သပ်ရပ်လာမယ်၊ ပြန်သုံးလို့ရတာ များလာမယ်၊ ထိန်းသိမ်းရတာ ပိုလွယ်ကူလာမယ်၊ အရေးကြီးဆုံးကတော့ ပိုပြီး ခိုင်မာစိတ်ချရတဲ့ Program တွေ ရေးနိုင်လာပါလိမ့်မယ်။"
