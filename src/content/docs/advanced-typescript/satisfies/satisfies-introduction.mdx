## ဖြေရှင်းနည်း: The `satisfies` Operator

TypeScript (Version 4.9 ကစပြီး) `satisfies` Operator ကို မိတ်ဆက်ပေးခဲ့ပါတယ်။ ဒီပြဿနာကို `satisfies` Operator က ဖြေရှင်းပေးမှာပါ။

**Syntax**: `expression satisfies Type`

### ဘယ်လို အလုပ်လုပ်သလဲ: ပြောင်းလဲခြင်းမရှိဘဲ စစ်ဆေးခြင်း (Validation without Transformation)

`AppConfig` ဥပမာကို `satisfies` နဲ့ ပြန်ကြည့်ရအောင်။

<Code lang="typescript" code={satisfies} title="config.ts" />

`satisfies AppConfig` က ဘာလုပ်ပေးသလဲဆိုရင် -

- #### **Validation Check (စစ်ဆေးပေးခြင်း)**:

  `satisfies AppConfig` က ဘယ်ဘက်က Object Literal ဟာ `AppConfig` နဲ့ ကိုက်ညီမှု ရှိမရှိ TypeScript ကို စစ်ဆေးခိုင်းတာပါ။

  - `port` ကို `prt` လို့ စာလုံးပေါင်းမှားရင် TypeScript က Error ပြပါမယ်။
  - `env` က `"staging"` (`AppConfig.env` Union မှာ မပါရင်) ဆိုရင်လည်း TypeScript က Error ပြပါမယ်။
  - `featureFlags.darkMode` က `number` ဖြစ်နေရင် (`boolean` မဟုတ်ရင်) TypeScript က Error ပြပါမယ်။ (`Record<string, boolean>` က `boolean` Values တွေကိုပဲ လက်ခံလို့ပါ ခင်ဗျာ။)

<br />

- #### **Specificity (တိကျမှု) ကို ထိန်းသိမ်းပေးခြင်း**:

  အရေးကြီးတာက `myConfig` ရဲ့ Type ဟာ Type Annotation (`:`) တုန်းကလို Type Widening (Type ကျယ်ပြန့်သွားခြင်း) မဖြစ်သွားပါဘူး။ သတ်မှတ်ထားတဲ့ Type အတိုင်း အတိအကျပဲ ဆက်ရှိနေပြီး၊ `AppConfig` နဲ့လည်း ကိုက်ညီမှုရှိမရှိ စစ်ဆေးပေးတာ ဖြစ်ပါတယ်။

<Code lang="typescript" code={still} title="config.ts" />

### အကျိုးကျေးဇူးများ

အခုဆိုရင် အတိအကျ Properties တွေကို Safely Access (လုံလုံခြုံခြုံ ခေါ်သုံး) နိုင်သွားပါပြီ။

<Code lang="typescript" code={benefits} title="config.ts" />

**ဥပမာ ခိုင်းနှိုင်းချက် -**

- "`satisfies` ကို စစ်ဆေးရေးမှူး တစ်ယောက်က Checklist (`AppConfig`) နဲ့ စစ်ဆေးနေတာမျိုး စဉ်းစားကြည့်ပါ ခင်ဗျာ။"
- "သင်က Inspector (စစ်ဆေးရေးမှူး) ကို သင်ကိုယ်တိုင် အတိအကျ ဖန်တီးထားတဲ့ Item (`myConfig` Object Literal) ကို ပေးလိုက်တယ်။"
- "Inspector က သင့် Item ဟာ Checklist ထဲက စံနှုန်းတွေနဲ့ ကိုက်ညီမှု ရှိမရှိ စစ်ဆေးပါတယ်။ မကိုက်ညီရင် ငြင်းပယ်ပါလိမ့်မယ် (TypeScript Error ပြမယ်)။"
- "အကယ်၍ စစ်ဆေးမှု အောင်မြင်ရင် သင့် Item ကို ဘာမှ မပြောင်းလဲဘဲ ပြန်ပေးပါလိမ့်မယ်။ သူတို့က Item ကို ပြောင်းလဲတာ ဒါမှမဟုတ် Checklist ထဲက Generic Version (ယေဘုယျ ပုံစံ) နဲ့ အစားထိုးတာမျိုး မလုပ်ပါဘူး။ သင်က သင့်ရဲ့ မူရင်း အတိအကျ Item ကို ပြန်ရမှာ ဖြစ်သလို၊ ဒါဟာ Standards တွေနဲ့ ကိုက်ညီတယ်ဆိုတဲ့ အာမခံချက်ကိုပါ ရရှိမှာ ဖြစ်ပါတယ် ခင်ဗျာ။"
