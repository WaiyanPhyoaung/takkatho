## Database ကို n8n တွင် ချိတ်ဆက်ခြင်း

n8n မှာ MySQL, PostgreSQL, SQLite, MariaDB, MS SQL Server စတဲ့ Database တွေကို Built-in Node တွေနဲ့ တိုက်ရိုက် ချိတ်ဆက်လို့ ရပါတယ်။ Google Sheets ကို Database ပုံစံ သုံးနေတဲ့ Workflow တွေကို တကယ့် Real Database ပြောင်းလိုက်တဲ့အခါ ပိုမြန်ဆန်ပြီး Scale လုပ်ရတာ ပိုလွယ်သွားပါမယ်။

## MySQL / MariaDB Credential ဖန်တီးနည်း

n8n Credentials ထဲကနေ **"MySQL"** ကို ရွေးပြီး အောက်ပါအတိုင်း ဖြည့်ပါ -

```
MySQL Credential:
  Host:     localhost (ဒါမှမဟုတ် သင့် DB server IP)
  Port:     3306
  Database: my_database
  User:     n8n_user
  Password: your_password

  SSL:      Enable (Production အတွက်)
```

<Aside type="caution" title="Database User Permission">
n8n အတွက် Full Admin User မသုံးတာ ပိုလုံခြုံပါတယ်။ `SELECT, INSERT, UPDATE, DELETE` Permission တွေလောက်ပဲ ပေးထားတဲ့ Dedicated User သီးသန့် ဖန်တီးပါ -
```sql
CREATE USER 'n8n_user'@'%' IDENTIFIED BY 'strong_password';
GRANT SELECT, INSERT, UPDATE, DELETE ON my_db.* TO 'n8n_user'@'%';
```
</Aside>

## MySQL Node Operations

| Operation | Query |
|-----------|-------|
| **Select** | Rows တွေကို ဖတ်ဖို့ |
| **Insert** | Row အသစ် ထည့်ဖို့ |
| **Update** | ရှိပြီးသား Row ကို ပြင်ဖို့ |
| **Delete** | Row ကို ဖျက်ဖို့ |
| **Execute Query** | ကိုယ်တိုင် Custom SQL ရေးဖို့ |

### Read — Data ဖတ်ခြင်း

```sql
-- Pending Orders တွေ အားလုံးကို ဆွဲထုတ်ဖို့
SELECT
  o.id,
  o.total_amount,
  u.name AS customer_name,
  u.email
FROM orders o
JOIN users u ON o.user_id = u.id
WHERE o.status = 'pending'
ORDER BY o.created_at DESC
LIMIT 100;
```

## Transaction — Multiple Queries တစ်ကြိမ်တည်း Run ခြင်း

Order တစ်ခု တင်လိုက်တဲ့အခါ စတော့ (Stock) ကိုပါ တစ်ကြိမ်တည်း အတူတူ Update လုပ်ပြီး၊ တစ်ခုခု မှားယွင်းသွားရင် Rollback ပြန်လုပ်တတ်ဖို့ လိုပါတယ် -

```sql
-- MySQL Execute Query Node
START TRANSACTION;

-- ၁. Order အသစ်ထည့်
INSERT INTO orders (user_id, total) VALUES ({{ $json.userId }}, {{ $json.total }});
SET @order_id = LAST_INSERT_ID();

-- ၂. Stock ပမာဏ လျှော့
UPDATE products
SET stock = stock - {{ $json.quantity }}
WHERE id = {{ $json.productId }} AND stock >= {{ $json.quantity }};

-- Stock မလုံလောက်ရင် Rollback လုပ်မယ်
IF ROW_COUNT() = 0 THEN
  ROLLBACK;
  SELECT 'INSUFFICIENT_STOCK' AS result;
ELSE
  COMMIT;
  SELECT @order_id AS result;
END IF;
```

  caption="Weekly Database Maintenance — အလိုအလျောက် Cleanup နဲ့ Archive လုပ်ခြင်း"

```sql
-- Cleanup Query 1: Old Logs
DELETE FROM activity_logs
WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);

-- Cleanup Query 2: Archive Orders
INSERT INTO orders_archive SELECT * FROM orders
WHERE created_at < DATE_SUB(NOW(), INTERVAL 365 DAY)
  AND status IN ('delivered', 'cancelled');

DELETE FROM orders
WHERE created_at < DATE_SUB(NOW(), INTERVAL 365 DAY)
  AND status IN ('delivered', 'cancelled');
```

<Aside type="tip" title="Query Result ကို n8n မှာ ဖတ်နည်း">
MySQL Execute Query Node ကနေ ထွက်လာတဲ့ Output ကို Code Node ထဲမှာ `$input.all()` သုံးပြီး ဖတ်လို့ ရပါတယ် -
```javascript
const rows = $input.all();
const userCount = rows.length;
const totalRevenue = rows.reduce((sum, r) => sum + r.json.total, 0);
```
</Aside>

နောက်သင်ခန်းစာမှာ **Loop & Batching** — Items အများကြီးကို တစ်ခုချင်းစီ ဘယ်လို Loop ပတ်ပြီး အလုပ်လုပ်မလဲဆိုတာ ဆက်လေ့လာကြပါမယ် ခင်ဗျာ။
