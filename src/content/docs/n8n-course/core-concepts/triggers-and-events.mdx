## Trigger ဆိုတာ ဘာလဲ?

**Trigger** ဆိုတာ Workflow ကို **စတင်ပေးတဲ့** Signal တစ်ခုပါ။ Trigger မလာမချင်း Workflow က "အိပ်" နေမှာပါ။ Trigger ဝင်လာတာနဲ့ Workflow က နောက်က Node တွေကို တစ်ခုပြီးတစ်ခု အလိုအလျောက် ဆက်အလုပ်လုပ်သွားမှာပါ။

## Trigger အမျိုးအစားများ

### 1. Manual Trigger

**Editor ပေါ်ကနေ Button ကို ကိုယ်တိုင်နှိပ်မှ** Workflow အလုပ်လုပ်တာပါ။ Testing အတွက် အများဆုံး သုံးပါတယ်။

ဘယ်အချိန်မှာ သုံးမလဲ -
- Workflow ကို Test လုပ်ချင်တဲ့အခါ
- တစ်ကြိမ်ပဲ Run စရာလိုတဲ့ Task တွေအတွက်
- Production Automation အတွက်ဆိုရင်တော့ မသင့်တော်ပါဘူး

### 2. Schedule Trigger (Cron)

**သတ်မှတ်ထားတဲ့ အချိန်** ရောက်တိုင်း Workflow ပြေးမှာပါ။

Schedule ဥပမာများ -
- Every minute — Testing အတွက်
- Every hour — Hourly Report ပို့ဖို့
- Daily at 9am — မနက်ပိုင်း Summary Email ပို့ဖို့
- Weekly Monday — Weekly Digest ပို့ဖို့
- Monthly 1st — Monthly Invoice ပို့ဖို့

**Cron Expression** ကို n8n ရဲ့ Visual Mode ထဲကနေ အလွယ်တကူ ပြောင်းလဲနိုင်ပါတယ် -

```
Cron Syntax:  * * * * *
              │ │ │ │ └── Day of Week (0=Sun)
              │ │ │ └──── Month (1-12)
              │ │ └────── Day of Month (1-31)
              │ └──────── Hour (0-23)
              └────────── Minute (0-59)

ဥပမာများ:
  0 9 * * 1-5   → တနင်္လာကနေ သောကြာ မနက် 9:00
  0 */2 * * *   → ၂ နာရီတစ်ကြိမ်
  30 8 * * 0    → တနင်္ဂနွေ မနက် 8:30
```

### 3. Webhook Trigger

**External System** (တခြား Website ဒါမှမဟုတ် App) ကနေ HTTP Request ပို့လာတဲ့အခါ Workflow ပြေးမှာပါ။ Real-time Automation တွေအတွက် အကောင်းဆုံးပါပဲ။

**Webhook URL** ကို n8n ကနေ ထုတ်ပေးပြီး External System ထဲကို သွားထည့်ပေးရမှာပါ -

```
Webhook URL Example:
https://your-n8n-instance.com/webhook/abc123-def456-ghi789

# Test Mode URL (Editor မှ Test လုပ်နေစဉ်)
https://your-n8n.com/webhook-test/abc123-def456-ghi789

# Production URL (Workflow Active ထားသောအခါ)
https://your-n8n.com/webhook/abc123-def456-ghi789
```

<Aside type="caution" title="Test vs Production Webhook URL">
Webhook Node မှာ URL (၂) မျိုး ရှိပါတယ်။ Editor ကနေ Test လုပ်ဖို့အတွက် "Test URL" ကို သုံးပါ။ Workflow ကို Active မလုပ်မချင်း Production URL က အလုပ်လုပ်မှာ မဟုတ်ပါဘူး ခင်ဗျာ။
</Aside>

### 4. App Triggers

Gmail, Slack, GitHub စတဲ့ App တွေမှာ Event တစ်ခုခု ဖြစ်သွားတာကို Poll/Listen ကနေတဆင့် သိနိုင်ပါတယ် -

- **Gmail** — Email အသစ် ရောက်လာရင်
- **Slack** — Message အသစ် ဝင်လာရင်
- **Notion** — Database ထဲ Row အသစ် ရောက်လာရင်
- **GitHub** — Issue / PR အသစ် ဖွင့်ရင်
- **Google Forms** — Form Submit လုပ်ရင်
- **Stripe** — Payment ဝင်လာရင်

<Aside type="note" title="Polling vs Webhook">
App Trigger တော်တော်များများက **Polling** (၁ မိနစ်တစ်ကြိမ် ပုံမှန် Check လုပ်တဲ့နည်း) နဲ့ အလုပ်လုပ်ပါတယ်။ တချို့ App တွေကတော့ Webhook ကို တိုက်ရိုက် Support လုပ်တဲ့အတွက် Real-time အလုပ်လုပ်ပါတယ်။
</Aside>

## Trigger ကို Activate လုပ်နည်း

Webhook တွေ၊ Schedule Trigger တွေ အလုပ်လုပ်ဖို့အတွက် Workflow ကို **Active** လုပ်ထားပေးရပါမယ် -

- **Active** → Trigger ကို Listen နေမယ် / Schedule အတိုင်း Run မယ်
- **Inactive** → Editor ကနေ Manual Test နှိပ်မှသာ Run မယ်

Editor ရဲ့ ညာဘက်အပေါ်ထောင့်က Toggle Switch လေးကို နှိပ်ပြီး Active/Inactive ပြောင်းနိုင်ပါတယ်။

<Aside type="tip" title="Development အတွက်">
Development လုပ်နေတဲ့အချိန်မှာ Workflow ကို **Inactive** အနေအထားနဲ့ပဲ ထားပါ။ Test လုပ်ချင်ရင် Manual Trigger သုံးပြီး Run ပါ။ အားလုံးပြီးလို့ Production Ready ဖြစ်မှသာ Active ပြောင်းပေးပါ။
</Aside>

နောက်သင်ခန်းစာမှာ **Credentials** — API Keys တွေနဲ့ Passwords တွေကို n8n မှာ လုံခြုံအောင် ဘယ်လို သိမ်းဆည်းရမလဲ ဆိုတာကို ဆက်လေ့လာကြပါမယ် ခင်ဗျာ။
