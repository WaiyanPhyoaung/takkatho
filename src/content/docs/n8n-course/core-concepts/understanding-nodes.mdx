## Node ဆိုတာ ဘာလဲ?

n8n Workflow မှာ **Node** ဆိုတာ လုပ်ဆောင်ချက် (Action) တစ်ခုကို ကိုယ်စားပြုတဲ့ Building Block တစ်ခုပါ။ Workflow တစ်ခုဟာ ဒီ Node တွေကို ချိတ်ဆက်ထားတာပါ။

Workflow ကို LEGO ဆော့သလိုပါပဲ။ Node တစ်ခုစီဟာ LEGO Block လေးတစ်ခုစီ ဖြစ်ပါတယ်။ Data တွေက Node တစ်ခုကနေ နောက် Node တစ်ခုဆီကို ဘယ်ကနေ ညာဘက်ကို စီးဆင်းပါတယ်။

## Node အမျိုးအစားများ

### 1. Trigger Nodes

Workflow ကို **စတင်ပေးတဲ့** Nodes တွေပါ။ Trigger မဖြစ်မချင်း Workflow က အလုပ်မလုပ်ပါဘူး။

Trigger အမျိုးအစားများ -
- **Schedule Trigger** — ကြိုတင်သတ်မှတ်ထားတဲ့ အချိန်ရောက်မှ
- **Webhook Trigger** — HTTP Request ဝင်လာမှ
- **App Trigger** — Gmail, Slack အစရှိတဲ့ App ကနေ Event ဖြစ်မှ
- **Manual Trigger** — Button ကို ကိုယ်တိုင်နှိပ်မှ

**ဥပမာများ:**
- **Schedule** — "နေ့တိုင်း မနက် ၉ နာရီ" မှာ Report ပို့ဖို့
- **Webhook** — Website Form ကနေ Submit ဝင်လာရင် Notification ပို့ဖို့
- **Gmail** — Email အသစ်ဝင်လာရင် Process စလုပ်ဖို့

### 2. Regular (Action) Nodes

Data တွေကို **Process လုပ်တာ** ဒါမှမဟုတ် **Action ယူတာ** မျိုးတွေ လုပ်ပါတယ်။

Action Node ဥပမာများ -
- **Google Sheets** — Row ကို ဖတ်ဖို့ / အသစ်ထည့်ဖို့
- **Gmail** — Email ပို့ဖို့
- **Slack** — Message ပို့ဖို့
- **HTTP Request** — API ခေါ်ဖို့
- **Code** — JS / Python ရေးဖို့
- **IF** — Condition စစ်ဖို့
- **Set** — Data ကို ပြင်ဆင်ဖို့

### 3. Core / Utility Nodes

Data Flow ကို ထိန်းချုပ်ဖို့ သုံးပါတယ် -

| Node | ရည်ရွယ်ချက် |
|------|------------|
| **IF** | Condition (True/False) အပေါ်မူတည်ပြီး လမ်းခွဲပေးတယ် |
| **Switch** | Conditions အများကြီး စစ်ပြီး လမ်းရွေးပေးတယ် |
| **Merge** | Node (၂) ခုဆီက Data တွေကို ပေါင်းပေးတယ် |
| **Loop** | Items တွေကို Iterate (ပတ်) ပေးတယ် |
| **Wait** | နောက် Node မရောက်ခင် ခဏစောင့်ပေးတယ် |
| **Set** | Data Fields တွေကို ပြင်ဆင်တယ် / အသစ်ထည့်တယ် |
| **Code** | Custom JS / Python Logic တွေ ရေးနိုင်တယ် |

## Node ရဲ့ အစိတ်အပိုင်းများ

Node တစ်ခုမှာ အဓိက (၂) ပိုင်း ပါဝင်ပါတယ် -

- **Input** — အရှေ့ Node ကနေ ရောက်လာတဲ့ Data (JSON Format)
- **Output** — လက်ရှိ Node ပြီးသွားတဲ့အခါ ထွက်လာမယ့် Data

Node ရဲ့ Parameters (Configuration Settings) မှာတော့ ဒီ Node က ဘယ်လို အလုပ်လုပ်ရမလဲ ဆိုတာကို သတ်မှတ်ပေးနိုင်ပါတယ်။

## n8n ရဲ့ Data Format: JSON Items

n8n မှာ Data တွေကို **JSON Items** ပုံစံနဲ့ သယ်ဆောင်ပါတယ် -

```json
[
  {
    "json": {
      "name": "Aung",
      "email": "aung@example.com",
      "age": 25
    }
  },
  {
    "json": {
      "name": "Ma Aye",
      "email": "aye@example.com",
      "age": 30
    }
  }
]
```

<Aside type="note" title="Items Array">
Node တစ်ခုဟာ Item **တစ်ခု** သို့မဟုတ် **အများကြီး** ကို Process လုပ်နိုင်ပါတယ်။ ဥပမာ Gmail Node မှာ Email (၅) စောင် ဝင်လာရင် Items (၅) ခု ထုတ်ပေးမှာပါ။
</Aside>

## Node Expression ဖြင့် Data ကို ယူသုံးခြင်း

Node Parameters မှာ **`{{ }}`** Expression Syntax ကို သုံးပြီး အရှေ့ Node က Data ကို ရယူနိုင်ပါတယ် -

```
# အရှေ့ Node က Data ကို ယူသုံးနည်း
{{ $json.fieldName }}
{{ $json.email }}
{{ $json.name }}

# အရှေ့ Node ကို နာမည်နဲ့ Reference လုပ်နည်း
{{ $node["Gmail"].json.subject }}

# Node ရဲ့ Item Index ကြည့်နည်း
{{ $itemIndex }}
```

**ဥပမာ:** Google Sheet ထဲက Name ကိုဖတ်ပြီး Email ပို့တဲ့အခါ -

```
To:      {{ $json.email }}
Subject: Hello {{ $json.name }}!
Body:    Dear {{ $json.name }}, Your account is ready.
```

## Node ကို Canvas ပေါ် ထည့်နည်း

1. Canvas ရဲ့ ဘယ်ဘက်က **Node Panel** မှာ Search လုပ်ပါ။
2. Node ကို **Drag & Drop** ဆွဲထည့်ပါ ဒါမှမဟုတ် **Double-click** နှိပ်ပြီး ထည့်ပါ။
3. Node ရဲ့ Connection Point (⚫) ကို Drag ဆွဲပြီး နောက်တစ်ခုနဲ့ ချိတ်ဆက်ပါ။

Node တွေကို ချိတ်ဆက်တဲ့အခါ Node A ရဲ့ Output Point ကနေ ဆွဲပြီး Node B ရဲ့ Input Point ပေါ်ကို Drop ချပေးရုံပါပဲ။

<Aside type="tip" title="Shortcut">
Canvas ပေါ်မှာ **Double-click** နှိပ်လိုက်ရင် Node Picker တိုက်ရိုက် ပွင့်လာမှာပါ။
</Aside>

## Node ကို Configure လုပ်ခြင်း

Node တစ်ခုကို Click နှိပ်လိုက်ရင် Right Panel မှာ Configuration တွေ ပေါ်လာပါမယ်။ ဥပမာ Gmail Node မှာဆိုရင် Credential ရွေးတာ၊ Resource, Operation, To, Subject, Body စတာတွေကို သတ်မှတ်နိုင်ပါတယ်။

**"Test Node"** ကို နှိပ်လိုက်ရင် အဲ့ဒီ Node တစ်ခုတည်းကို Execute လုပ်ကြည့်နိုင်ပါတယ်။ Workflow တစ်ခုလုံးကို Run ဖို့ မလိုတဲ့အတွက် Development ကို ပိုမြန်စေပါတယ်။

နောက်သင်ခန်းစာမှာ **Triggers** နဲ့ **Events** အကြောင်း ဆက်လေ့လာကြပါမယ် ခင်ဗျာ။
