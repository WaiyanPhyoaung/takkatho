## AI Agent ဆိုတာ ဘာလဲ?

**AI Agent** ဆိုတာ ရိုးရိုး LLM (ဥပမာ ChatGPT လို) အမေးအဖြေလုပ်ရုံတင် မဟုတ်ဘဲ ပိုမြင့်တဲ့ System တစ်ခုပါ။ AI ကို Tools တွေ (ဥပမာ - Web Search, Database ဝင်ရှာတာ၊ API ခေါ်တာ) ပေးထားလိုက်ပြီး၊ ပေးလိုက်တဲ့ Task တစ်ခု ပြီးမြောက်ဖို့ သူကိုယ်တိုင် စဉ်းစားဆုံးဖြတ်ပြီး အလုပ်လုပ်ခိုင်းတဲ့ Pattern ပါ -

```
ရိုးရိုး LLM ရဲ့ အလုပ်လုပ်ပုံ:         AI Agent ရဲ့ အလုပ်လုပ်ပုံ:
  Prompt ပေးလိုက်တယ်                 Prompt ပေးလိုက်တယ်
     ↓                                   ↓
  Response တန်းပြန်ဖြေတယ်            [စဉ်းစားမယ်: ဒီအဖြေရဖို့ ငါ ဘာ Tool သုံးရမလဲ?]
                                         ↓
                                    [Tool သုံးမယ်: Web ပေါ်မှာ ရှာကြည့်မယ်]
                                         ↓
                                    [Tool သုံးမယ်: တွက်ချက်စရာရှိရင် Calculator သုံးမယ်]
                                         ↓
                                    [အကုန်ရပြီဆိုမှ နောက်ဆုံး Response ပြန်ဖြေမယ်]
```

n8n မှာပါတဲ့ **AI Agent Node** (ReAct Agent) က LangChain နည်းပညာကို အသင့်ထည့်ပေးထားလို့ သူကိုယ်တိုင် Tools တွေကို လိုအပ်သလို ရွေးချယ်သုံးစွဲသွားနိုင်ပါတယ်။

## Agent Node Setup

Canvas ထဲမှာ **"AI Agent"** Node ကို ထည့်ပြီး အောက်ပါ Sub-nodes တွေကို လိုက်ချိတ်ပေးရပါမယ် -

```
AI Agent Node
├── Chat Model (ဒါကတော့ မဖြစ်မနေ လိုပါတယ်)
│   └── [OpenAI GPT-4o] သို့မဟုတ် [Claude Sonnet]
│
├── Memory (Optional — အရင် ပြောခဲ့တဲ့ စကားတွေကို မှတ်မိချင်ရင်)
│   └── [Window Buffer Memory]
│
└── Tools (သူ့ကို ပေးသုံးချင်တဲ့ စွမ်းရည်တွေ)
    ├── [Calculator]       ← သင်္ချာ တွက်ဖို့
    ├── [SerpAPI Search]   ← Web ပေါ်က ရှာဖွေဖို့
    ├── [Wikipedia]        ← အချက်အလက် (Knowledge) ရှာဖို့
    └── [Code Executor]    ← Code တွေကို တိုက်ရိုက် Run ဖို့
```

**System Prompt** (Agent ရဲ့ ဇာတ်ကောင် / Role သတ်မှတ်ချက်):

```
You are a helpful assistant for a Myanmar e-commerce company.
You can:
- Search for product information
- Calculate prices and discounts
- Look up order status from database
- Answer questions in Myanmar language

Always be polite and concise. Respond in the same language the user wrote in.
```

## Memory ထည့်ခြင်း — Conversation History

Memory မပါရင် Agent က အရင်တစ်ခေါက် ပြောထားခဲ့တဲ့ စကား (Message) တွေကို မှတ်မိမှာ မဟုတ်ပါဘူး။ **Window Buffer Memory** လေး ထည့်ပေးလိုက်ရင်တော့ စကားပြောခဲ့တဲ့ သမိုင်းကြောင်း (Conversation History) ကို မှတ်ထားပေးနိုင်ပါပြီ -

```
Memory Configuration:
  Window Buffer Memory:
    Context Window Size: 10   ← နောက်ဆုံး ပြောခဲ့တဲ့ Messages ၁၀ ခုကို မှတ်မိမယ်
    Session Key: {{ $('Chat Trigger').item.json.sessionId }}
```

<Aside type="note" title="Session Key ဆိုတာ ဘာလဲ?">
Session Key ဟာ User တစ်ယောက်ချင်းစီရဲ့ စကားပြောမှတ်တမ်းကို သီးသန့်စီ (Separate) ခွဲခြားပေးတာပါ။ Chat UI မှာဆိုရင် `sessionId` ကို သူ့အလိုလို ထုတ်ပေးပါတယ်။ အကယ်၍ Webhook ကနေ ခေါ်သုံးတာဆိုရင်တော့ `{{ $json.userId }}` ကို Session Key အဖြစ် အစားထိုး သုံးနိုင်ပါတယ်။
</Aside>

### Order Status Tool Workflow

Agent ကနေ ဒီ Sub-Workflow ကို Tool အနေနဲ့ သုံးဖို့ ဒီလို Configure လုပ်ပါမယ် -

```javascript
// Tool Workflow — Get Order Status
// Input က ဒီလို လာမယ်: { orderId: "ORD-123" }
const orderId = $json.orderId;

// MySQL Node ကနေ Query သွားလုပ်မယ်
const query = `SELECT * FROM orders WHERE id = '${orderId}' LIMIT 1`;

// ရလာတဲ့ ရလဒ် (Result) ကို သပ်သပ်ရပ်ရပ်လေးနဲ့ Agent ဆီ ပြန်ပို့ပေးမယ်
return [{
  json: {
    orderId,
    status: $('MySQL').first().json.status,
    createdAt: $('MySQL').first().json.created_at,
    total: $('MySQL').first().json.total_amount,
    message: `Order ${orderId} ရဲ့ လက်ရှိ အခြေအနေကတော့: ${$('MySQL').first().json.status} ဖြစ်ပါတယ်`
  }
}];
```

## Agentic Workflow ဥပမာ

User: "ကျွန်တော်ရဲ့ Order ORD-789 ဘယ်ရောက်ပြီလဲ? ပစ္စည်းရောက်ပြီးပြီဆိုရင် Review ရေးဖို့ Link လေး တစ်ခါတည်း ပို့ပေးပါ"

Agent က အဆင့်ဆင့် စဉ်းစားသွားမယ့် ပုံစံ (Thinking Process):

```
Step 1: Parse Intent (ဘာလိုချင်လဲ အရင်ခွဲခြမ်းမယ်)
  → "Order status အခြေအနေ သိချင်တယ်" + "ပစ္စည်းရောက်ပြီဆိုရင် Review တောင်းပေးပါ"

Step 2: Use Order Status Tool (Tool ခေါ်သုံးမယ်)
  → Query: SELECT * FROM orders WHERE id='ORD-789'
  → Result: { status: "delivered", deliveredAt: "2025-01-10" }

Step 3: Decide Next Action (ဘာဆက်လုပ်ရမလဲ ဆုံးဖြတ်မယ်)
  → Status က delivered ဖြစ်နေပြီ → အဲဒီတော့ Review တောင်းလို့ ရပြီ

Step 4: Use Email Tool (နောက် Tool တစ်ခု ထပ်သုံးမယ်)
  → Customer ဆီကို Review တောင်းတဲ့ Email လှမ်းပို့မယ်

Step 5: Final Response (နောက်ဆုံး အဖြေ ပြန်ဖြေမယ်)
  → "ခင်ဗျားရဲ့ Order ORD-789 ကို 2025-01-10 ရက်နေ့ကတည်းက ပို့ဆောင်ပြီး ဖြစ်ပါတယ်။ 
     Review ရေးလို့ရမယ့် Email ကိုလည်း ပေးပို့ထားပြီး ဖြစ်ပါတယ်။"
```
