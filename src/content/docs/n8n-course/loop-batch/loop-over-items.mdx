## Loop ဘာကြောင့် လိုအပ်သလဲ?

n8n မှာ Node တစ်ခုဟာ Items အများကြီးကို တစ်ခါတည်း အစုလိုက် Process လုပ်လေ့ရှိပါတယ်။ ဒါပေမဲ့ တချို့ API တွေက တစ်ကြိမ်ကို Item တစ်ခုစီပဲ လှမ်းခေါ်လို့ရတဲ့ Endpoint မျိုး ရှိတတ်ပါတယ်။ ဒါမှမဟုတ် Item တစ်ခုချင်းစီကို Process လုပ်ပြီး ထွက်လာတဲ့ Result ကိုမှ ဆက်သုံးရမယ့် အခြေအနေမျိုးမှာ Loop Pattern ကို မဖြစ်မနေ သုံးရပါတယ်။

```
Loop မလိုတဲ့အချိန် (n8n ရဲ့ ပုံမှန်အလုပ်လုပ်ပုံ):
  [Get 100 Users] ──▶ [Send Email]    ← Email Node က Users ၁၀၀ ကို တစ်ခါတည်း Process လုပ်သွားတယ်

Loop လိုတဲ့အချိန်:
  [Get 100 Users]
      │
  [Loop: User တစ်ဦးချင်းစီ ခွဲထုတ်မယ်]
      │
      ├──▶ [Get User Details from Slow API]   ← တစ်ကြိမ်ကို ၁ ယောက်ပဲ ခေါ်မယ်
      └──▶ [Update Database Row]
```

## Loop Over Items (Basic)

n8n ရဲ့ **Loop Over Items** Node (အရင်ကတော့ `splitInBatches` node ရဲ့ loop mode လို့ ခေါ်ပါတယ်) ကို သုံးပြီး Item တစ်ခုချင်းစီကို ခွဲပြီး Process လုပ်နိုင်ပါတယ် -

## SplitInBatches Node

Data တွေ အများကြီး (Large Dataset) ကို **Group (Batch)** တွေ အနေနဲ့ ခွဲပြီး Process လုပ်တာပါ။ API Rate Limit တွေကို ကျော်ဖြတ်တဲ့နေရာမှာ အသုံးအဝင်ဆုံး Pattern ပါပဲ -

### SplitInBatches Configuration

```
SplitInBatches Node:
  Batch Size: 50         ← တစ်ကြိမ် (တစ်သုတ်) ကို ဘယ်နှ Item လုပ်မလဲ
  Options:
    Reset: false         ← Loop ကို အစကနေ ပြန်မစဘဲ ဆက်ပြေးစေဖို့
```

### Loop ဖြင့် Process လုပ်သော Code Pattern

```javascript
// SplitInBatches ပြီးသွားတဲ့အခါ Code Node ထဲမှာ
const batchItems = $input.all();

console.log(`Processing batch of ${batchItems.length} items...`);

// Batch ထဲက Item တစ်ခုချင်းစီကို Process လုပ်ခြင်း
const results = batchItems.map(item => ({
  json: {
    ...item.json,
    processed: true,
    processedAt: new Date().toISOString()
  }
}));

return results;
```

## Rate Limiting ကိုင်တွယ်နည်း

API Rate Limit ဆိုတာ API Server ကနေ "တစ်မိနစ်ကို Request ဘယ်နှကြိမ်ပဲ ခွင့်ပြုမယ်" ဆိုတဲ့ ကန့်သတ်ချက်ပါ -

<BranchFlow
  input="SplitInBatches (Batch: 100)"
  condition="Rate Limit ဖြစ်သလား? (429 HTTP Error)"
  truePath={["Wait: 60 seconds", "Retry Same Batch"]}
  falsePath={["Process Batch Normally", "Next Batch"]}
  trueLabel="429 ❌"
  falseLabel="200 ✅"
/>

### Wait Node ထည့်ပြီး Rate Limit ကာကွယ်ပါ

```
Workflow Pattern:
  [SplitInBatches] ──▶ [API Call] ──▶ [Wait: 1s] ──▶ (Loop ထိပ်ကို ပြန်ပြန်သွားမယ်)

Wait Node:
  Wait Amount: 1
  Wait Unit:   Seconds

  ← ဆိုလိုတာက API ကို Batch တစ်ကြိမ် ခေါ်ပြီးတိုင်း 1 စက္ကန့် ခဏနားမယ်
```

<Aside type="tip" title="API Rate Limit စစ်ကြည့်ပါ">
ကိုယ်သုံးမယ့် API ရဲ့ Documentation ထဲမှာ Rate Limit ကို သွားစစ်ကြည့်ပါ။ ဥပမာ - Gmail API မှာဆို တစ်မိနစ်ကို 250 quota units သာ ရပြီး၊ OpenAI API တွေမှာတော့ သုံးတဲ့ Model ပေါ်မူတည်ပြီး Rate Limit တွေ ကွဲပြားပါတယ်။
</Aside>

Loop ကို Condition တစ်ခုခု ပြည့်သွားရင် ရပ်တန့်ချင်တဲ့အခါ **IF Node** ကို Loop ထဲ ထည့်သုံးပါတယ် -

### Pagination သုံးပြီး API Data အကုန်ဆွဲနည်း

API တော်တော်များများမှာ Page တစ်ခုကို Items ၁၀ ခု စသဖြင့်ပဲ Return ပြန်ပေးလေ့ရှိပြီး နောက် Data တွေအတွက် နောက် Page ကို ထပ်ခေါ်ရတဲ့ Pattern မျိုးပါ -

```javascript
// Code Node — Pagination State ကို မှတ်ထားမယ်
const currentPage = $execution.resumeData?.page ?? 1;
const totalPages = $input.first().json.total_pages;

// State ကို ဆက်ပြီး Update လုပ်မယ်
const output = {
  items: $input.first().json.data,
  nextPage: currentPage + 1,
  hasMore: currentPage < totalPages,
  currentPage
};

return [{ json: output }];
```
