## Data Transformation ဆိုတာ ဘာလဲ?

Real-world Workflow တွေမှာ API တစ်ခုကနေ Data ယူပြီး နောက် App တစ်ခုမှာ သွားထည့်တဲ့အခါ Format တွေက ကွာနေတတ်ပါတယ်။ Data Transformation ဆိုတာ အဲ့ဒီလို Format မကိုက်ညီတာတွေကို ပြုပြင်ပေးတာပါ -

- Source API ရဲ့ `user_name` → Slack မှာ `username` လို့ ပြောင်းပေးရမယ်
- Source API ရဲ့ `date_ts` → Slack မှာ `text` လို့ ပြောင်းပေးရမယ်
- Source API ရဲ့ `amount` → Slack မှာ `blocks` ထဲ ထည့်ပေးရမယ်

## Set Node — Field တွေ ပြောင်းလဲခြင်း

**Set Node** ဟာ Data ထဲက Field တွေကို အသစ်ထည့်တာ၊ ပြင်တာ၊ ဖျက်တာတွေ လုပ်ဖို့ သုံးပါတယ် -

### Mode 1: Manual Mapping

Visual Editor ကနေ Field တစ်ခုချင်းစီကို သတ်မှတ်ပေးလို့ ရပါတယ် -

| Name (field) | Value |
|---|---|
| `fullName` | `{{ $json.first + " " + $json.last }}` |
| `email` | `{{ $json.email.toLowerCase() }}` |
| `joinedAt` | `{{ $now.toISO() }}` |

### Mode 2: JSON

Output ထွက်လာမယ့် JSON ကို တိုက်ရိုက် ရေးပေးလို့လည်း ရပါတယ် -

```json
{
  "fullName": "={{ $json.first + ' ' + $json.last }}",
  "email": "={{ $json.email.toLowerCase() }}",
  "status": "active"
}
```

## Code Node — Complex Transformation

Complex Logic တွေ သုံးချင်ရင်တော့ JavaScript ဒါမှမဟုတ် Python ကို Code Node ထဲမှာ ရေးနိုင်ပါတယ် -

### ဥပမာ: API Data ကို ပြင်ဆင်ခြင်း

```javascript
// Code Node - JavaScript
// API ကနေ orders list ကို ရယူပြီး လိုသလို ပြုပြင်ခြင်း

return $input.all().map(item => {
  const order = item.json;

  return {
    json: {
      orderId: order.id,
      customer: `${order.billing.first_name} ${order.billing.last_name}`,
      total: `$${(order.total / 100).toFixed(2)}`,
      status: order.status.toUpperCase(),
      date: new Date(order.date_created).toLocaleDateString('my-MM'),
      itemCount: order.line_items.length
    }
  };
});
```

### Python Mode

```python
# Code Node - Python
result = []

for item in _input.all():
    order = item['json']
    result.append({
        'json': {
            'id': order['id'],
            'total': f"${order['total'] / 100:.2f}",
            'status': order['status'].upper()
        }
    })

return result
```

## IF Node — Conditional Branching

**IF Node** က Condition ကို စစ်ပြီး Data ကို True လမ်းကြောင်းနဲ့ False လမ်းကြောင်း (၂) ခု ခွဲပေးပါတယ် -

<BranchFlow
  input="Customer Order"
  condition="amount > 100?"
  truePath={["Set: VIP Tag", "Send VIP Email"]}
  falsePath={["Set: Standard Tag", "Send Standard Email"]}
  trueLabel="TRUE ✅"
  falseLabel="FALSE ❌"
/>

### IF Node Configuration

```
Condition: {{ $json.amount }} > 100

  Value 1:   {{ $json.amount }}   (Number)
  Operation: Greater Than
  Value 2:   100
```

### Multiple Conditions

```
AND Condition (၂ ခုလုံး True ဖြစ်မှ):
  amount > 100
  AND status === "completed"

OR Condition (တစ်ခုခု True ဖြစ်ရုံနဲ့):
  status === "pending"
  OR status === "processing"
```

## Switch Node — Multiple Paths

**Switch Node** ဟာ IF လိုမျိုးပါပဲ၊ ဒါပေမဲ့ လမ်းကြောင်း **(၂) ခုထက် ပိုခွဲချင်တဲ့အခါ** သုံးပါတယ် -

<FlowDiagram
  nodes={[
    { label: "Switch Node", sublabel: "status field စစ်", type: "condition" },
    { label: "pending → Payment Reminder", type: "action" },
    { label: "shipped → Tracking Email", type: "action" },
    { label: "delivered → Review Request", type: "action" }
  ]}
  caption="Switch Node — Status ပေါ်မူတည်ပြီး Path (၃) ခုခွဲသည်"

**Switch Node Configuration:**

```
Mode: Rules

Case 1: {{ $json.status }} === "pending"    → Output 0
Case 2: {{ $json.status }} === "shipped"    → Output 1
Case 3: {{ $json.status }} === "delivered"  → Output 2
Default:                                    → Output 3
```

## Merge Node — Data တွေ ပေါင်းခြင်း

Node (၂) ခုကနေ လာတဲ့ Data တွေကို ပေါင်းချင်တဲ့အခါ **Merge Node** ကို သုံးပါတယ် -

Merge Node Modes -
- **Append** — A items အောက်မှာ B items ကို ဆက်ထည့်တယ်
- **Combine** — တူညီတဲ့ Field တွေကို Match လုပ်ပြီး Join တယ်
- **Choose Branch** — A ဒါမှမဟုတ် B ကို Condition နဲ့ ရွေးတယ်

**ဥပမာ — User နဲ့ Order Data ပေါင်းခြင်း:**

```
[Get User from DB] ──▶ ┌──────────────┐
                       │  Merge Node  │──▶ { user + order }
[Get Order from API]──▶│  (Combine)   │
                       └──────────────┘

Merge by: user_id (common field)
```

## n8n Expressions — Built-in Functions

n8n Expression တွေမှာ Built-in အနေနဲ့ သုံးလို့ရတဲ့ Helper Functions တွေ ရှိပါတယ် -

### String Functions

```
{{ $json.name.toUpperCase() }}       → "AUNG AUNG"
{{ $json.email.toLowerCase() }}      → "aung@gmail.com"
{{ $json.text.trim() }}              → အစ/အဆုံး Whitespace ဖျက်
{{ $json.name.includes("Aung") }}    → true / false
{{ $json.title.slice(0, 50) }}       → ပထမ Character ၅၀
```

### Number Functions

```
{{ Math.round($json.price) }}           → အနီးစပ်ဆုံး ကိန်းပြည့် ယူ
{{ Number($json.amount).toFixed(2) }}   → ဒသမ နေရာ ၂ နေရာ ယူ
{{ $json.items.length }}                → Array အရေအတွက်
```

### Date Functions

```
{{ $now.toISO() }}                → "2025-01-15T09:00:00.000Z"
{{ $now.toFormat('yyyy-MM-dd') }} → "2025-01-15"
{{ $now.plus({days: 7}).toISO() }}→ ဒီနေ့ကနေ (၇) ရက်ကြာ Date
```

</Aside>
