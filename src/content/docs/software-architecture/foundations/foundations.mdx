### Architecture ဆိုတာဘာလဲ - အပေါ်စီးကနေ ခြုံငုံကြည့်ခြင်း
တိုတိုနဲ့လိုရင်းပြောရရင် Software Architecture ဆိုတာ Software System တစ်ခုရဲ့ အဓိကကျတဲ့ တည်ဆောက်ပုံတွေနဲ့ အဲ့ဒီတည်ဆောက်ပုံတွေကို ဖြစ်လာစေတဲ့ အရေးကြီးတဲ့ ဆုံးဖြတ်ချက်တွေအကြောင်းပါပဲ။ Code တစ်ကြောင်းချင်းစီရဲ့ အသေးစိတ်အကြောင်းမဟုတ်ဘဲ၊ အရာအားလုံးကို တစ်စုတစ်စည်းတည်းဖြစ်အောင် ထိန်းပေးထားတဲ့ အရိုး (skeleton) နဲ့ ပိုတူပါတယ်။

#### ဥပမာတစ်ခုနဲ့ ရှင်းပြရရင် - အိမ်တစ်လုံးဆောက်မယ်

အိမ်တစ်လုံးဆောက်တာကို မြင်ယောင်ကြည့်ပါ။ Software Architecture ဆိုတာ အဲ့ဒီအိမ်ရဲ့ ပုံစံကြမ်း (blueprint) ပါပဲ။ သူက အောက်ပါအချက်တွေကို သတ်မှတ်ပေးပါတယ်:

-   အိမ်ရဲ့ အုတ်မြစ်ဘယ်လိုချမလဲ။
-   အထပ်ဘယ်နှစ်ထပ် ပါမလဲ။
-   အဓိကအခန်းတွေ (ဥပမာ - မီးဖိုချောင်၊ အိပ်ခန်း) ဘယ်နေရာမှာ ထားမလဲ။
-   မီးလိုင်း၊ ရေလိုင်း၊ လေဝင်လေထွက်စနစ်တွေက တည်ဆောက်ပုံတစ်ခုလုံးမှာ ဘယ်လိုချိတ်ဆက်သွားမလဲ။

ဒီပုံစံကြမ်းက နံရံကို ဘာဆေးအရောင်သုတ်မလဲ၊ မီးဖိုချောင်မှာ ဘယ်လိုဘုံဘိုင်ခေါင်းတပ်မလဲဆိုတာတွေအထိ အသေးစိတ်မဖော်ပြပါဘူး။ ဒါတွေက low-level details တွေပါ။ **Architecture ဆိုတာက နောက်မှပြန်ပြောင်းဖို့ အရမ်းခက်ခဲပြီး ကုန်ကျစရိတ်များတဲ့** အခြေခံတည်ဆောက်ပုံပိုင်းဆိုင်ရာ ဆုံးဖြတ်ချက်တွေပဲ ဖြစ်ပါတယ်။ အခန်းတစ်ခန်းကို ဆေးပြန်သုတ်တာ လွယ်ပေမဲ့၊ အိမ်တစ်လုံးကို နောက်ထပ်တစ်ထပ်ထပ်တိုးဖို့ကတော့ အလွန်ကြီးမားတဲ့ လုပ်ငန်းစဉ်တစ်ခု ဖြစ်သွားပါပြီ။

### Architecture, Design, နဲ့ Code ဘာတွေကွာလဲ
ဒီစကားလုံးတွေကို မကြာခဏ ရောထွေးပြီးသုံးတတ်ကြပေမဲ့၊ သူတို့က မတူညီတဲ့ အသေးစိတ်အဆင့် (level of detail) တွေကို ကိုယ်စားပြုပါတယ်။

-   **Architecture:** မြို့တစ်မြို့ရဲ့ အဓိကဇုန်တွေ (လူနေဇုန်၊ စီးပွားရေးဇုန်)၊ အဓိကလမ်းမကြီးတွေ၊ ဓာတ်အားပေးစက်ရုံနဲ့ ရေပေးဝေရေးစခန်းတွေ ဘယ်မှာထားမလဲဆိုတာကို သတ်မှတ်တာနဲ့တူတယ်။ ဒီဆုံးဖြတ်ချက်တွေက နောက်မှပြောင်းဖို့ အလွန်ခက်ခဲပါတယ်။

    -   ဥပမာ: "ကျွန်တော်တို့ရဲ့ application ကို **microservices architecture** နဲ့ တည်ဆောက်မယ်။ Service တစ်ခုနဲ့တစ်ခု event တွေသုံးပြီး ဆက်သွယ်မယ်။"

-   **Design:** မြို့ထဲက တိုက်တစ်လုံးချင်းစီရဲ့ အခန်းဖွဲ့စည်းပုံ (floor plan) ကို သတ်မှတ်တာနဲ့တူတယ်။ ဒါက မြို့ပြစီမံကိန်းရဲ့ စည်းမျဉ်းတွေကို လိုက်နာရပေမဲ့၊ component တစ်ခုရဲ့ အတွင်းပိုင်းတည်ဆောက်ပုံကိုပဲ အဓိကထားတယ်။

    -   ဥပမာ: "**User Service** ကို API, business logic layer, နဲ့ data access layer ဆိုပြီး အလွှာတွေခွဲထားတဲ့ layered pattern နဲ့ တည်ဆောက်မယ်။"

-   **Code:** အိမ်ဆောက်တဲ့အခါသုံးမယ့် တကယ့် ပစ္စည်း တွေနဲ့ တည်ဆောက်ပုံအသေးစိတ် (implementation) တွေပါပဲ။

    -   ဥပမာ: **`class UserService { ... }`**

### Software Architect တစ်ယောက်ရဲ့ အခန်းကဏ္ဍနဲ့ တာဝန်တွေ
Architect တစ်ယောက်ရဲ့ အလုပ်က ပုံဆွဲတာထက် အများကြီးပိုပါတယ်။ ဒါက အဖွဲ့ကိုဦးဆောင်ရတဲ့ အခန်းကဏ္ဍတစ်ခုဖြစ်ပြီး အောက်ပါအဓိကတာဝန်တွေ ပါဝင်ပါတယ်:

-   **အဓိကကျတဲ့ နည်းပညာပိုင်းဆိုင်ရာ ဆုံးဖြတ်ချက်တွေချမှတ်ခြင်း:** Project ရဲ့ အနာဂတ်ကို ပုံဖော်မယ့် အရေးကြီးတဲ့ **architectural ရွေးချယ်မှုတွေ**ကို သတ်မှတ်ပြီး ဆုံးဖြတ်ချက်ချရပါတယ်။

-   **Trade-offs တွေကို ချိန်ညှိသုံးသပ်ခြင်း:** Architecture တိုင်းမှာ အားသာချက်၊ အားနည်းချက်တွေရှိပါတယ်။ ဒါတွေကို နားလည်ပြီး business လိုအပ်ချက်နဲ့ technical လိုအပ်ချက်တွေကြား မျှတအောင် ချိန်ညှိရွေးချယ်ရပါတယ်။

-   **Architecture Vision ကို ရှင်းလင်းစွာ ပြောပြဆက်သွယ်ခြင်း:** Team မှာရှိတဲ့ developer တိုင်းက architecture ကို နားလည်ပြီး၊ ဘာကြောင့်ဒီလိုရွေးချယ်ရသလဲဆိုတဲ့ အကြောင်းကို သိနေဖို့လိုပါတယ်။ ဒါမှ သူတို့ရေးလိုက်တဲ့ code က architecture နဲ့ အံဝင်ခွင်ကျဖြစ်စေမှာပါ။

-   **နည်းပညာပိုင်းဆိုင်ရာ ဦးဆောင်မှုနဲ့ လမ်းညွှန်မှုပေးခြင်း (Mentorship):** Team ကို လမ်းညွှန်ပေးတာ၊ coding standard တွေ သတ်မှတ်ပေးတာ၊ developer တွေရဲ့ skill တွေ တိုးတက်လာအောင် ကူညီပေးတာတွေ လုပ်ရပါတယ်။
