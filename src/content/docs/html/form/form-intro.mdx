### Form ဆိုတာ ဘာလဲ?
Form ဆိုတာက user ဆီက အချက်အလက်တွေကို စုဆောင်းပြီး server ဆီကို ပို့ပေးတဲ့ HTML element တစ်ခုပါ။

**ဥပမာနှိုင်းယှဉ်ချက်** — Form ကို digital ဖြည့်လွှာတစ်ခုလို သဘောထားနိုင်ပါတယ်။ ဘဏ်မှာ account ဖွင့်တဲ့အခါ ဖြည့်ရတဲ့ ဖြည့်လွှာလိုမျိုးပါပဲ — user က ဖြည့်ပြီး Submit နှိပ်ရင် server ဆီ ပေးပို့သွားတာ ဖြစ်ပါတယ်။

##### အသုံးပြုပုံများ -

- Website ပေါ်က Contact form တွေ
- User register လုပ်ခြင်း သို့မဟုတ် login ဝင်ခြင်း
- Feedback နဲ့ စစ်တမ်းများ
- Booking form များ

### Form Submission Flow (ဖောင်တင်သွင်းတဲ့ ဖြစ်စဉ်)
User တစ်ယောက် Submit နှိပ်လိုက်ရင် ဘာတွေဖြစ်သွားသလဲ ဆိုတာကို ကြည့်လိုက်ရအောင်။ စာပို့တာနဲ့ ဥပမာနှိုင်းထားပါတယ် — ဖြည့်ထားတဲ့ form = ရေးထားတဲ့ စာ၊ Submit = post box ထဲထည့်တာ၊ server = post office ဖြစ်ပါတယ်။

### Form တည်ဆောက်ပုံ
Form တစ်ခုကို `<form>` tag နဲ့ ဖန်တီးပါတယ်။ အဓိကကျတဲ့ attribute နှစ်ခုကတော့-

- `action` – Form data တွေကို ဘယ် URL ဆီ ပို့ရမလဲ သတ်မှတ်ပေးတယ်။
- `method` – Data ကို `GET` (URL ထဲမှာ မြင်နိုင်) ဒါမှမဟုတ် `POST` (လုံခြုံစွာ ဖောက်ထုတ်) နည်းနဲ့ ပို့မလဲ ဆိုတာ သတ်မှတ်ပေးတယ်။

```html
<form action="submit_form.php" method="post">
  <!-- input fields တွေ ဒီထဲမှာ ထည့်ရပါတယ် -->
</form>
```

**ရှင်းလင်းချက်:**

- `action="submit_form.php"` — form data တွေကို `submit_form.php` file ဆီ ပို့ပေးမှာ ဖြစ်တယ်။ Password လိုမျိုး အထိန်းအကျောင်းဒေတာ ပို့တဲ့အခါ `method="post"` ကို သုံးပါ — URL ထဲမှာ မပြဘဲ ဖောက်ထုတ်ပို့ပေးလို့ ပိုလုံခြုံပါတယ်။

### Input Fields
**Input field** တွေက user တွေ data ထည့်ဖို့ အတွက်သုံးတာပါ။ `<input>` tag ကို `type` attribute အမျိုးမျိုးနဲ့ တွဲသုံးပါတယ်-

| Input Type | ရှင်းလင်းချက် |
|------------|-------------------------------------------------------------------|
| text       | text ထည့်ရန် (ဥပမာ - name)။            |
| password   | စကားဝှက်ထည့်ရန် (စာလုံးတွေက ဖျောက်ထားပေးတယ်)။                  |
| email      | Email လိပ်စာထည့်ရန်။                                            |
| number     | number သီးသန့်ထည့်ရန် (ဥပမာ - အသက်)။                            |
| date       | ရက်စွဲရွေးရန်။                                                  |
| checkbox   | ရွေးချယ်စရာအများကြီးကို ရွေးနိုင်ရန် (ဥပမာ - ဝါသနာ)။           |
| radio      | ရွေးချယ်စရာထဲက တစ်ခုတည်းကိုသာ ရွေးနိုင်ရန် (ဥပမာ - ကျား/မ)။    |
| file       | File တင်ရန် (ဥပမာ - Profile ပုံ)။                                |

### Labels — Input ခေါင်းစဉ်တပ်ခြင်း
`<label>` tag ကို input field နဲ့ တွဲပြီး ခေါင်းစဉ်တပ်ဖို့ သုံးပါတယ်။ `for` attribute ရဲ့ value က သက်ဆိုင်ရာ input ရဲ့ `id` value နဲ့ **ကိုက်ညီ**နေရပါမယ်။

```html
<label for="email">Email:</label>
<input type="email" id="email" name="email">
```

**ဘာကြောင့် label ထည့်ရမလဲ:**

- label ကို နှိပ်လိုက်ရင် သက်ဆိုင်ရာ input box ကို အလိုအလျောက် focus ပေးပါတယ် (ဖုန်းပေါ်မှာ သုံးတဲ့အခါ အထူးအဆင်ပြေပါတယ်)
- Screen reader တွေက label ကို ဖတ်ပြပြီး input ဘာဖြည့်ရမလဲ ပြောပြပေးပါတယ်
- `label` မပါဘဲ input ထဲမှာ `placeholder` ထဲပဲ ထည့်တာဟာ accessibility မကောင်းပါဘူး

### Fieldset နှင့် Legend — Input တွေကို အုပ်စုဖွဲ့ခြင်း
Form တစ်ခုမှာ input field တွေ အများကြီးရှိတဲ့အခါ၊ ဆက်စပ်တဲ့ field တွေကို `<fieldset>` နဲ့ စုဖွဲ့ပြီး `<legend>` နဲ့ ခေါင်းစဉ်တပ်ပေးနိုင်ပါတယ်။ ဒါက accessibility ကောင်းမွန်သလို ဖတ်ရတာလည်း ပိုသပ်ရပ်ပါတယ်။

**ဥပမာ —**

```html
<form action="register.php" method="post">
  <fieldset>
    <legend>ကိုယ်ရေးကိုယ်တာ အချက်အလက်</legend>
    <label for="name">အမည်:</label>
    <input type="text" id="name" name="name" required>

    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
  </fieldset>

  <fieldset>
    <legend>နေရပ်လိပ်စာ</legend>
    <label for="city">မြို့:</label>
    <input type="text" id="city" name="city">

    <label for="country">နိုင်ငံ:</label>
    <input type="text" id="country" name="country">
  </fieldset>

  <button type="submit">Register</button>
</form>
```

- `<fieldset>` — ဆက်စပ်တဲ့ input တွေကို ဘောင်ခတ်ပြီး စုစည်းပေးတဲ့ container
- `<legend>` — `<fieldset>` ရဲ့ ခေါင်းစဉ် (ဘောင်ကြိုးပေါ်မှာ ပေါ်တယ်)

### Form Validation Attributes
HTML မှာ JavaScript မလိုဘဲ browser ကနေတိုက်ရိုက် input data ကို စစ်ဆေးနိုင်တဲ့ validation attribute တွေ ရှိပါတယ်။

| Attribute | ရှင်းလင်းချက် | ဥပမာ |
|-----------|--------------|-------|
| `required` | ဖြည့်မှ submit လုပ်ခွင့်ပြုတယ် | `<input type="text" required>` |
| `minlength` | အနည်းဆုံး စာလုံးရေ သတ်မှတ်တယ် | `<input type="text" minlength="3">` |
| `maxlength` | အများဆုံး စာလုံးရေ သတ်မှတ်တယ် | `<input type="text" maxlength="50">` |
| `min` | ဂဏန်း/ရက်စွဲ အနည်းဆုံးတန်ဖိုး | `<input type="number" min="1">` |
| `max` | ဂဏန်း/ရက်စွဲ အများဆုံးတန်ဖိုး | `<input type="number" max="100">` |
| `pattern` | Regex pattern နဲ့ စစ်တယ် | `<input type="text" pattern="[A-Za-z]+">` |

**ဥပမာ — validation နဲ့ form တစ်ခု:**

```html
<form action="submit.html" method="post">
  <label for="username">အမည်:</label>
  <input type="text" id="username" name="username" required minlength="3" maxlength="20">

  <label for="age">အသက်:</label>
  <input type="number" id="age" name="age" min="13" max="120" required>

  <button type="submit">Submit</button>
</form>
```

<Aside type="caution" title="သတိပြုရန်">
HTML validation တွေဟာ user experience ကို ပိုကောင်းစေပေမဲ့ server-side validation ကို **အစားမထိုးနိုင်ပါဘူး**။ Security အတွက် server ဘက်ကလည်း data ကို အမြဲစစ်ဆေးရပါမယ်။
</Aside>
