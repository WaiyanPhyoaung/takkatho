### အဆင့် ၁: GitHub မှာ Empty `Repository` တစ်ခု ဖန်တီးမယ်။

GitHub.com Website ပေါ်မှာ:

- GitHub ကို Login လုပ်ဝင်ပါ။
  -"+" ပုံလေး (များသောအားဖြင့် ညာဘက် အပေါ်ထောင့်မှာ ရှိပါတယ်) ကို နှိပ်ပြီး "New repository" (Project အသစ်) ကို ရွေးပါ။
- ကိုယ့် Project အတွက် `name` လေး တစ်ခု ပေးပါ (ကိုယ့် `local` က Project `name` my-first-repo နဲ့ တူရင် မှတ်ရလွယ်တာပေါ့)။
- `visibility` ကို "Public" ကို ရွေးထားပါ။
- <Aside type="danger" title="သတိပြုရန်">
    ကိုယ့် `local machine` ထဲမှာ Project Code တွေ ရှိပြီးသားဆိုရင် "Initialize
    this repository with a README", ".gitignore", or "license" လိုမျိုး
    စာကြောင်းလေးတွေကို **အမှန်ခြစ် မခြစ်ပါနဲ့**။ ကျွန်တော်တို့ အွန်လိုင်းက
    remote ကို empty (ဗလာ) ထားပြီး ကိုယ့် local က Codes တွေကို အဲဒီ ဗလာထဲ push
    (ထည့်) မှာ မို့လို့ပါ။
  </Aside>
- "Create repository" ကို နှိပ်လိုက်ပါ။
- GitHub က ကိုယ့် `new remote repository` အတွက် URL လိပ်စာ တစ်ခု ပေးပါလိမ့်မယ် (ဥပမာ - `https://github.com/YourUsername/my-first-repo.git`)။ ဒီ URL လေးကို Copy ယူထားလိုက်ပါ။

အဆင့် ၂: ကိုယ့် `Local Repo` ကို `Remote` နဲ့ ချိတ်မယ်: `git remote add`

ဒီ `git remote add` command က ကိုယ့် `local Git repository` လေးကနေ သူ့ရဲ့ အွန်လိုင်းမှာ ရှိတဲ့ သက်ဆိုင်ရာ `remote repository` လေးကို ဘယ်နေရာမှာ ရှိတယ်ဆိုတာ လမ်းညွှန်ပေးလိုက်တာ။

Command ပုံစံ: `git remote add <nickname (origin)> <အွန်လိုင်းက Project လိပ်စာ>`

အွန်လိုင်းက `remote` ကို ကိုယ် အလွယ်မှတ်မိအောင် ပေးတဲ့ nickname ။ standard convention ကတော့ origin လို့ပဲ ပေးကြတယ်။
`<အွန်လိုင်းက Project လိပ်စာ>`: ခုနက GitHub ကနေ Copy ယူခဲ့တဲ့ Project URL ပေါ့။

### Connection မိပြီလား? စစ်ဆေးခြင်: `git remote -v`

- `terminal` မှာ `git remote -v` လို့ ရိုက်ကြည့်ပါ။ (အဲဒါဆို ကိုယ် ချိတ်ထားတဲ့ `origin` ဆိုတဲ့ နာမည်နဲ့ GitHub က Project URL ကို "fetch" နဲ့ "push" အတွက် ပြနေတာ တွေ့ရရင် အောင်မြင်ပါပြီ)။
