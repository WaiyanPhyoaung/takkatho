## Cloning ဆိုတာ ဘာလဲ?

**Cloning** ဆိုတာ အွန်လိုင်းပေါ်မှာ ရှိနေတဲ့ Repository တစ်ခုလုံးကို ကိုယ့် ကွန်ပျူတာထဲ ကူးယူလိုက်တာကို ဆိုလိုပါတယ်။ ၎င်းတွင် -

- ဖိုင်များ အားလုံး
- Commit မှတ်တမ်းများ အားလုံး (History)
- `Branch` များ အားလုံး
- Remote နှင့် ချိတ်ဆက်မှု (`origin` အဖြစ် အလိုအလျောက် ချိတ်ဆက်ပေးခြင်း) စတာတွေ ပါဝင်ပါတယ်။

## ဘာကြောင့် Clone လုပ်ကြသလဲ?

1. ရှိပြီးသား `Project` တစ်ခုမှာ စတင် ပါဝင် အလုပ်လုပ်ချင်တဲ့အခါ
2. `Open-source Project` တွေကို ဝင်ရောက် ကူညီ ရေးသားချင်တဲ့အခါ
3. အဖွဲ့ဝင်တွေရဲ့ `Code` ကို ကိုယ့်စက်ထဲမှာ ကြည့်ရှု လေ့လာချင်တဲ့အခါ

## လက်တွေ့ စမ်းသပ်ကြည့်ရန်

GitHub မှာ စိတ်ဝင်စားစရာကောင်းတဲ့ Repository တစ်ခုကို ရှာပြီး `HTTPS clone URL` ကို Copy ကူးပါ။ ပြီးရင် Terminal မှာ `git clone <URL>` လို့ ရိုက်ပြီး ရလာတဲ့ Folder ကို ဖွင့်ကြည့်ပါ။ `Repository` တစ်ခုလုံး ကိုယ့်စက်ထဲ အပြည့်အစုံ ရောက်လာတာကို မြင်တွေ့ရမှာ ဖြစ်ပါတယ် ခင်ဗျာ။

<Aside title='အနှစ်ချုပ်'>
1. **`origin` ဆိုတာ နာမည်ပေးထားခြင်းပါ**
   - ဒါက ကိုယ့်ရဲ့ အဓိက အွန်လိုင်း Project နေရာကို Git က အလွယ် မှတ်ထားတဲ့ မူလနာမည်ပါ။ (တခြား နာမည် ပြောင်းပေးလို့လည်း ရပါတယ်)။

2. **Push နဲ့ Pull ကို မကြာခဏ လုပ်ပါ**
   - Local နဲ့ Remote ကို အမြဲတမ်း Update ဖြစ်နေအောင် ထားပါ။
   - အထူးသဖြင့် အဖွဲ့လိုက် အလုပ်လုပ်တဲ့အခါ အပြောင်းအလဲ သေးသေးလေးတွေကို မကြာခဏ Update လုပ်တာက စီမံခန့်ခွဲရ ပိုလွယ်ကူစေပါတယ်။

3. **Conflicts ဖြစ်လာရင်**
   - ပြဿနာကြီးတွေ မဟုတ်ပါဘူး။ စိတ်အေးအေးထားပြီး ဖြေရှင်းနိုင်ပါတယ်။

4. **Merge လုပ်တဲ့အခါ သတိပြုရန်**
   - လက်ရှိ ကိုယ် ဘယ် Branch မှာ ရောက်နေလဲ၊ ဘယ် Branch က ပြောင်းလဲမှုတွေကို ပေါင်းထည့်မှာလဲ ဆိုတာကို Command မရိုက်ခင် သေချာ စစ်ဆေးဖို့ မမေ့ပါနဲ့ ခင်ဗျာ။

5. **Merge (သို့) Rebase လုပ်ခြင်း**
   - `git pull` ထက် ပိုပြီး ထိန်းချုပ်ချင်တယ်ဆိုရင် `git fetch` ကို သုံးပြီး Remote က အပြောင်းအလဲတွေကို အရင် ကြည့်ရှုနိုင်ပါတယ်။
   - ပြီးမှ ဘယ်လို ပေါင်းစပ်မလဲ (Merge လား၊ Rebase လား) ဆုံးဖြတ်လို့ ရပါတယ်။ (ဒါက အဆင့်နည်းနည်း မြင့်ပေမဲ့ သိထားဖို့ ကောင်းတဲ့ အချက်လေးပါ ခင်ဗျာ)။
</Aside>
