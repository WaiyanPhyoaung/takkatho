### Merging ဆိုတာ ဘာလဲ? (အလုပ်များကို စုစည်းခြင်း)

Feature အသစ်ရေးတာ ဒါမှမဟုတ် Bug ပြင်တာတွေ ပြီးသွားပြီဆိုရင်၊ ခွဲထွက်ထားတဲ့ Branch က ပြောင်းလဲမှုတွေကို Project ရဲ့ အဓိက Branch (`main`) ထဲကို ပြန်ထည့်ဖို့ လိုပါတယ်။ ဒါကို Merging လုပ်တယ်လို့ ခေါ်ပါတယ်။ သီးခြား လုပ်ထားတဲ့ အလုပ်တွေကို အဓိက Code ထဲကို စုစည်းလိုက်တဲ့ သဘောပါပဲ။

#### `git merge <branch-name>` ဖြင့် ပေါင်းစပ်ခြင်း

Merge ဘယ်လို လုပ်ရမလဲဆိုရင် -
- ပထမဆုံး ကိုယ် ပြောင်းလဲမှုတွေ ထည့်သွင်းချင်တဲ့ အဓိက Branch ဆီ အရင် သွားရပါမယ်။ (ဥပမာ - `git switch main`)
- အဲဒီကို ရောက်ပြီဆိုရင် `git merge add-user-login` လို့ ရိုက်လိုက်ပါ။ ဒီ Command က "add-user-login" Branch က ပြောင်းလဲမှုတွေကို လက်ရှိရောက်နေတဲ့ "main" Branch ထဲ ပေါင်းထည့်ပေးသွားမှာ ဖြစ်ပါတယ်။

#### Merge အလုပ်လုပ်ပုံ

များသောအားဖြင့် `git merge` Command ကို ရိုက်လိုက်တာနဲ့ Git က Branch နှစ်ခုရဲ့ ပြောင်းလဲမှုတွေကို အလိုအလျောက် ပေါင်းစပ်ပေးပါတယ်။ တချို့ အခြေအနေတွေမှာတော့ Merge လုပ်လိုက်ပြီဆိုတဲ့ မှတ်တမ်းအနေနဲ့ Commit အသစ်တစ်ခုကို အလိုအလျောက် ဖန်တီးပေးလေ့ ရှိပါတယ် ခင်ဗျာ။
