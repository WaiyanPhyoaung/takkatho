### Merge Conflict ဆိုတာ ဘာလဲ?

Merge Conflict ဆိုတာ Branch နှစ်ခုကနေ တူညီတဲ့ ဖိုင်ရဲ့ တူညီတဲ့ နေရာကို တစ်ယောက်တစ်မျိုးစီ ပြင်ထားတဲ့ အခါမျိုးမှာ ဖြစ်တတ်ပါတယ်။ ဥပမာ - စာကြောင်း တစ်ကြောင်းတည်းကို မတူညီတဲ့ ပုံစံနဲ့ ပြင်ထားတာမျိုးပါ။ အဲ့ဒီအခါ Git က ဘယ်ဟာကို ယူရမလဲ မဆုံးဖြတ်နိုင်တော့ပါဘူး။ 

<Aside type="danger" title="မှတ်ချက်">
  အဖွဲ့လိုက် အပြိုင် အလုပ်လုပ်တဲ့အခါ ဒါမျိုး ဖြစ်တတ်တာ သဘာဝပါပဲ။ စိုးရိမ်စရာ မလိုပါဘူး ခင်ဗျာ။
</Aside>

### Conflict ဖြစ်ကြောင်း ဘယ်လို သိနိုင်မလဲ?

Merge လုပ်လိုက်တဲ့အခါ Conflict ရှိနေရင် Git က Terminal မှာ ချက်ချင်း အသိပေးပါတယ်။ `git status` လို့ ရိုက်ကြည့်ရင်လည်း Conflict ဖြစ်နေတဲ့ ဖိုင်တွေကို အနီရောင်နဲ့ တွေ့ရမှာပါ။

### Conflict ဖြေရှင်းနည်း

1. **ဖိုင်ကို ဖွင့်ပါ**: Git က Conflict ဖြစ်နေတဲ့ နေရာတွေကို `<<<<<<<`, `=======`, `>>>>>>>` စတဲ့ အမှတ်အသားတွေနဲ့ ရှင်းရှင်းလင်းလင်း ပြထားပေးပါတယ်။
2. **ပြင်ဆင်ပါ**: ဘယ် Code ကို ယူမလဲ ဆုံးဖြတ်ပြီး ပြင်ဆင်ပါ။ ပြီးရင် `<<<<<<<`, `=======`, `>>>>>>>` အမှတ်အသားတွေ အားလုံးကို ဖျက်ပစ်ရပါမယ်။
3. **Staging သို့ ထည့်ပါ**: ဖြေရှင်းပြီးသား ဖိုင်ကို `git add <ဖိုင်နာမည်>` နဲ့ Staging Area ထဲ ပြန်ထည့်ပါ။
4. **Commit လုပ်ပါ**: နောက်ဆုံး အဆင့်အနေနဲ့ `git commit` ကို သုံးပြီး Merge လုပ်ငန်းစဉ်ကို အပြီးသတ်နိုင်ပါတယ် ခင်ဗျာ။ *(Git က Commit Message ကို အလိုအလျောက် ဖန်တီးပေးထားမှာ ဖြစ်ပါတယ်)*။
