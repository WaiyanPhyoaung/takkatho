### ဘာကြောင့် Branch များကို ဖျက်သင့်သလဲ?

Branch တစ်ခုကို `main` ထဲ ပေါင်းထည့်ပြီးသွားရင် အဲဒီ Branch က ဆက်ထားဖို့ မလိုတော့ပါဘူး။ မလိုအပ်တော့တဲ့ Branch တွေကို ဖျက်ပစ်ခြင်းအားဖြင့် Project ကို သပ်ရပ်စေပြီး ရှုပ်ထွေးမှုတွေ လျော့နည်းစေပါတယ်။

### ပေါင်းပြီးသား Branch များကို ဖျက်ခြင်း (`git branch -d <branch-name>`)

`-d` (အသေး) ကို သုံးတာက Branch ကို စိတ်ချရဆုံး ဖျက်နည်းပါ။ Git က Branch ကို မဖျက်ခင် အလုပ်တွေ အကုန် ပေါင်းပြီးပြီလားဆိုတာကို အရင် စစ်ဆေးပေးပါတယ်။

**လုပ်ဆောင်ရန် အဆင့်များ**:
1. ဖျက်မယ့် Branch ကနေ အရင် ထွက်ရပါမယ် (ဥပမာ - `git switch main`)။
2. ပြီးရင် `git branch -d add-user-login` လို့ ရိုက်ပြီး ဖျက်နိုင်ပါတယ်။

### အတင်းအကျပ် ဖျက်ခြင်း (`git branch -D <branch-name>`)

`-D` (အကြီး) ကို သုံးရင်တော့ Branch ကို ပေါင်းပြီးပြီလား မစစ်တော့ဘဲ အတင်းအကျပ် ဖျက်ပစ်မှာ ဖြစ်ပါတယ်။ ဒါကြောင့် ဒီ Branch က အလုပ်တွေကို လုံးဝ မလိုချင်တော့ဘူး သေချာမှသာ သုံးသင့်ပါတယ် ခင်ဗျာ။

<Aside title="အနှစ်ချုပ်">
  1. **Branch တွေ အများကြီး ခွဲပါ**: အလုပ်အသစ် သေးသေးလေး တစ်ခု စတိုင်း Branch တစ်ခု ခွဲပြီး လုပ်ပါ။ Branch ခွဲရတာ အရမ်း လွယ်ကူပါတယ်။
  <br />
  2. **Main Branch ကို ဂရုစိုက်ပါ**: ကိုယ့်ရဲ့ အဓိက Branch ကို အမြဲတမ်း သန့်ရှင်းပြီး အဆင်ပြေပြေ အလုပ်လုပ်နေတဲ့ အခြေအနေမှာ ရှိနေပါစေ။
  <br />
  3. **Conflicts တွေက ပုံမှန်ပါပဲ**: Conflicts ဖြစ်တာ ပြဿနာကြီး မဟုတ်ပါဘူး။ ဖိုင်ကို ဖွင့်၊ ပြင်၊ Save လုပ်လိုက်ရင် အလွယ်တကူ ရှင်းသွားမှာပါ။
  <br />
  4. **Merge လုပ်ရင် သေချာ စစ်ဆေးပါ**: ကိုယ် ဘယ် Branch မှာ ရောက်နေလဲ၊ ဘယ် Branch ကနေ ပေါင်းထည့်မှာလဲဆိုတာကို Command မရိုက်ခင် သေချာ စစ်ဆေးဖို့ မမေ့ပါနဲ့ ခင်ဗျာ။
</Aside>
