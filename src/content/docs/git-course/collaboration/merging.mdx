  **Approve** လုပ်ထားပြီးသား **Pull Request (PR)** တစ်ခုကို **Main Branch** ထဲ ဘယ်လို **Merge** လုပ်ရမလဲ ဆိုတာနဲ့၊ Merge ပြီးသွားရင် Local မှာ ဘာတွေ ဆက်လုပ်ဖို့ လိုအပ်သလဲ ဆိုတာ လေ့လာနိုင်ဖို့ ဖြစ်ပါတယ် ခင်ဗျာ။

## Green Light: Approval နှင့် Merging

Reviewers တွေက သဘောတူပြီ၊ လိုအပ်တဲ့ Automated Tests (အလိုအလျောက် စစ်ဆေးမှုတွေ) အားလုံးလည်း အောင်မြင်သွားပြီဆိုရင် PR ကို **Approve** လုပ်ပေးလိုက်ပါပြီ။

### ဘယ်သူတွေက Merge လုပ်တာလဲ?

များသောအားဖြင့် Repository မှာ **Merge Rights** ရှိတဲ့သူ (PR ဖွင့်သူ ကိုယ်တိုင် ဒါမှမဟုတ် ပရောဂျက် Maintainer) က **Merge** လုပ်လေ့ရှိပါတယ်။

#### ဘယ်လိုလုပ်မလဲ? (GitHub ပေါ်တွင်)

1. PR Page ပေါ်မှာ **"Merge pull request"** ဆိုတဲ့ ခလုတ်စိမ်းလေး ပေါ်လာပါလိမ့်မယ်။
2. Merge Method တွေ ရွေးလို့ ရပါတယ် (ဥပမာ - `"Create a merge commit"`, `"Squash and merge"`, `"Rebase and merge"`)။ အခု စတင်လေ့လာသူတွေ အတွက်တော့ **Default** ဖြစ်တဲ့ `"Create a merge commit"` ကိုပဲ ရွေးတာ အဆင်ပြေပါတယ်။
3. **"Confirm merge"** ကို နှိပ်လိုက်ပါ။

   > 💡 GitHub က ကိုယ့် `feature branch` က Commits တွေကို ယူပြီး၊ Remote Repository ပေါ်က Base Branch (`main`) ထဲကို အလိုအလျောက် **Merge** လုပ်ပေးသွားပါလိမ့်မယ်။

## Merge လုပ်ပြီးနောက်: သန့်ရှင်းရေး (Cleaning Up)

Merge လုပ်ပြီးသွားရင်လည်း အလုပ်လေး နည်းနည်း ကျန်ပါသေးတယ်။

### Remote Branch ဖျက်ခြင်း (မဖြစ်မနေ မဟုတ်ပါ)

PR ကို Merge လုပ်ပြီးတာနဲ့ GitHub က **"Delete branch"** ခလုတ်လေး တစ်ခါတည်း ပြပေးပါလိမ့်မယ်။ ဒါက Remote ပေါ်က `feature branch` ကို ဖျက်ပစ်ဖို့ပါ။ အဲ့ဒီ Branch က အလုပ်တွေ အားလုံးက `main` ထဲ ရောက်သွားပြီ ဖြစ်လို့ ဆက်ထားစရာ မလိုတော့လို့ပါ ခင်ဗျာ။

### Local သန့်ရှင်းရေး (Local Cleanup)

ကိုယ့်စက် (Local) ထဲမှာလည်း သန့်ရှင်းရေး လုပ်ဖို့ လိုပါတယ်။

**1. ကိုယ့် Local `main` Branch ကို Update လုပ်ပါ**
   - `git switch main`
   - `git pull origin main` 
   *(ဒါက Remote `main` မှာ အသစ်ရောက်လာတဲ့ ကိုယ့်ရဲ့ Merge Changes တွေကို Local `main` ဆီ ဆွဲယူလိုက်တာပါ)*။

**2. ကိုယ့် Local `feature branch` ကို ဖျက်ပါ**
   - `git branch -d my-new-feature`
   *( `-d` က **Safe Delete** ပါ။ ဒီ Branch က တကယ် Merge ပြီးသွားပြီလား ဆိုတာ သေချာမှ ဖျက်ပေးတာမို့ စိတ်ချရပါတယ်)*။

<Aside title="အနှစ်ချုပ်">
  PR တစ်ခု Approved ဖြစ်သွားရင် GitHub ပေါ်မှာ Merge လုပ်လိုက်ပါ။ ပြီးရင် ကိုယ့် Local `main` ကို နောက်ဆုံး အခြေအနေ Update လုပ်ပြီး၊ အသုံးမလိုတော့တဲ့ Local Feature Branch ကို ဖျက်ပစ်ဖို့ မမေ့ပါနဲ့ ခင်ဗျာ။
</Aside>
