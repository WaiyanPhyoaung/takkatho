## Pull Request (PR) ဆိုတာ ဘာလဲ?

Pull Request (PR) ဆိုတာ ကိုယ် ပြင်ဆင်ထားတဲ့ အလုပ်တွေကို ကိုယ့် Branch (`feature branch`) ကနေ နောက်ထပ် အဓိက Branch (`main`) ထဲကို ထည့်သွင်း (Merge) ဖို့ တရားဝင် တောင်းဆို တင်ပြတာ ဖြစ်ပါတယ်။ ဒါဟာ တကယ့် `git` ရဲ့ တိုက်ရိုက် Command တစ်ခု မဟုတ်ဘဲ၊ GitHub သို့မဟုတ် GitLab လို Platform တွေက ဖန်တီးပေးထားတဲ့ အသုံးဝင်ဆုံး Feature တစ်ခုပါ။

စာအုပ် တစ်အုပ် ရေးတာနဲ့ ဥပမာ ပေးရရင် - ကိုယ်က ကိုယ်ရေးရမယ့် အခန်းကို ရေးလို့ ပြီးသွားပြီ (ကိုယ့် Feature Branch)။ PR ဆိုတာက အဲဒီအခန်းကို စာတည်းမှူး (Main Branch ကို ထိန်းသိမ်းသူ) ဆီကို "ကျွန်တော့် အခန်းလေး ပြီးသွားပြီ၊ တစ်ချက်လောက် ပြန်စစ်ပေးပါ၊ အဆင်ပြေရင် စာအုပ်ထဲ ထည့်ပေးပါ" လို့ လှမ်းပြီး တင်ပြလိုက်တာနဲ့ တူပါတယ် ခင်ဗျာ။

### PR တွေ ဘာကြောင့် အသုံးဝင်သလဲ?

- **Code Review (ကုဒ် ပြန်စစ်ဆေးခြင်း)**: ကိုယ်ရေးထားတဲ့ Code ကို အဖွဲ့သားတွေက ပြန်စစ်ပေးလို့ ရပါတယ်။ ဘယ်လို ပြင်ရင် ပိုကောင်းမယ်ဆိုတာ အကြံပေးနိုင်သလို၊ အဓိက Code ထဲကို Bugs တွေ မရောက်ခင် ကြိုတင် တားဆီးပေးနိုင်ပါတယ်။
- **ဆွေးနွေးတိုင်ပင်ရန် နေရာ**: ကိုယ်လုပ်ထားတဲ့ အပြောင်းအလဲတွေနဲ့ ပတ်သက်ပြီး မေးခွန်းတွေ မေးဖို့၊ ဆွေးနွေးဖို့နဲ့ ဆုံးဖြတ်ချက်တွေကို သေချာ မှတ်တမ်းတင်ထားဖို့ နေရာတစ်ခု ဖြစ်လာပါတယ်။
- **စနစ်တကျ ပေါင်းစပ်ဖို့**: ပြင်ဆင်မှုတွေကို ရမ်းသန်းပြီး သွားထည့်တာမျိုး မဟုတ်ဘဲ၊ အဖွဲ့သားတွေရဲ့ အတည်ပြုချက် ရမှသာ စနစ်တကျ Merge လုပ်နိုင်အောင် ထိန်းချုပ်ပေးပါတယ်။
- **အလိုအလျောက် စမ်းသပ်မှုများ (Automated Tests)**: PR ဖွင့်လိုက်တာနဲ့ တစ်ပြိုင်နက် CI/CD စနစ်တွေက Tests တွေ ပြေးတာ၊ Deployment Previews တွေ ထုတ်ပေးတာတွေကို အလိုအလျောက် လုပ်ဆောင်ပေးနိုင်ပါတယ်။

### GitHub ပေါ်မှာ PR ဘယ်လို ဖွင့်မလဲ?

ကိုယ့်ရဲ့ Feature Branch ကို GitHub ဆီ Push လုပ်ပြီးသွားရင် အောက်ပါအတိုင်း ဆက်လုပ်ပါ -

1. GitHub.com က ကိုယ့် Project Repository ကို သွားပါ။
2. GitHub က အသင့် ပြပေးနေတဲ့ "your-feature-branch had recent pushes. **Compare & pull request**" ဆိုတဲ့ ခလုတ်စိမ်းလေးကို နှိပ်လိုက်ပါ။ (ဒါမှမဟုတ် "Pull requests" Tab ကို သွားပြီး "New pull request" ကို နှိပ်ပါ)။
3. **Base** နဲ့ **Compare** ကို သေချာ ရွေးချယ်ပါ။
   - Base Branch: ဘယ် Branch ထဲကို ထည့်ချင်တာလဲ (ဥပမာ - `main`)။
   - Compare Branch: အခု ကိုယ် တင်ပြမယ့် Branch က ဘာလဲ (ဥပမာ - `my-new-feature`)။
4. **Title နဲ့ Description** ကောင်းကောင်း ရေးပါ။
   - Title: ဘာကို အဓိက ပြင်ထားတယ်ဆိုတာ ရှင်းလင်းပြတ်သားစွာ ရေးပါ။
   - Description: ဘာလုပ်ထားသလဲ၊ ဘာကြောင့် လုပ်ရတာလဲ (ဥပမာ - Issue #123 ကို ဖြေရှင်းထားသည်)၊ ဘယ်လို ဖြေရှင်းထားလဲ ဆိုတာကို အခြားသူတွေ နားလည်လွယ်အောင် ရေးပြပါ။
5. **"Create pull request"** ခလုတ်ကို နှိပ်လိုက်ပါ။

#### ကိုယ်တိုင် လုပ်ကြည့်ရန်

အရင် သင်ခန်းစာတွေမှာ Branch တစ်ခုကို Push လုပ်ခဲ့ဖူးတယ်ဆိုရင်၊ အဲဒီ Branch ကနေ Local `main` ထဲကို GitHub ပေါ်မှာ PR တစ်ခု စမ်းပြီး ဖွင့်ကြည့်ပါ။ ရိုးရှင်းတဲ့ Title နဲ့ Description လေး စမ်းရေးကြည့်ပါ ခင်ဗျာ။

<Aside title="အနှစ်ချုပ်">
  Pull Request ဆိုတာ "ကျွန်တော့် Branch က အလုပ်တွေ ပြီးသွားပြီ၊ Review ပြီးရင် Merge ဖို့ အဆင်သင့် ဖြစ်နေပါပြီ" လို့ တရားဝင် အကြောင်းကြားတာပါပဲ။ Code Quality ကောင်းဖို့နဲ့ အဖွဲ့လိုက် အချိတ်အဆက်မိမိ အလုပ်လုပ်ဖို့ (Collaboration) အတွက် အရမ်း အရေးကြီးပါတယ် ခင်ဗျာ။
</Aside>
