<Aside title="ရည်ရွယ်ချက်">
  **Pull Request (PR)** ထဲမှာ **Code Review** လုပ်ရတဲ့ ရည်ရွယ်ချက်နဲ့ အခြေခံ အဆင့်ဆင့်ကို နားလည်နိုင်ဖို့ ဖြစ်ပါတယ် ခင်ဗျာ။
</Aside>

## PR ဖွင့်ပြီးရင် ဘာဆက်ဖြစ်မလဲ?

ကိုယ့် PR လေး ဖွင့်လိုက်ပြီ ဆိုတာနဲ့ Team Members တွေ ဒါမှမဟုတ် သတ်မှတ်ထားတဲ့ **Reviewers** တွေဆီကို အကြောင်းကြားစာ ရောက်သွားပါလိမ့်မယ်။ သူတို့က GitHub ပေါ်က PR ရဲ့ "Files changed" ကို ဝင်ကြည့်ပြီး ကိုယ့် Code တွေကို စတင် Review လုပ်ကြပါမယ်။

## ပြန်စစ်ခြင်းရဲ့ အဓိက အချက်: Feedback ပေးခြင်းနှင့် ဆွေးနွေးခြင်း

### ပြန်စစ်သူတွေက ဘာတွေ ကြည့်ကြသလဲ?

- ကိုယ်ရေးထားတဲ့ **Code** က ရည်ရွယ်ထားတဲ့အတိုင်း မှန်မှန်ကန်ကန် အလုပ်လုပ်ရဲ့လား?
- သိသာထင်ရှားတဲ့ **Bugs** (အမှားများ) ပါနေသလား?
- Code က **ရှင်းလင်းမှု** ရှိရဲ့လား၊ သူများ ဖတ်ရင်ရော နားလည်လွယ်ရဲ့လား?
- အဖွဲ့ရဲ့ **Team Conventions** (လိုက်နာရမယ့် စည်းကမ်းများ) သို့မဟုတ် **Best Practices** တွေနဲ့ ကိုက်ညီရဲ့လား?
- အခုရေးထားတဲ့ နည်းလမ်းထက် **ပိုကောင်းတဲ့ တခြားနည်းလမ်း** တွေ ရှိသေးလား?

### Feedback ပေးခြင်း

**Reviewers** တွေက Code ရဲ့ သီးခြား Line တွေပေါ်မှာ Comments လေးတွေ ပေးနိုင်ပါတယ်။ မေးခွန်းတွေ မေးတာ၊ ဒါမှမဟုတ် ဘယ်လိုလေး ပြင်လိုက်ရင် ပိုကောင်းမယ်ဆိုတာ အကြံပြုတာမျိုးတွေ လုပ်နိုင်ပါတယ်။ အားလုံး အဆင်ပြေရင် PR ကို **Approve** လုပ်ပေးနိုင်သလို၊ လိုအပ်ရင် ထပ်ပြင်ပေးဖို့ **Request Changes** လည်း လုပ်နိုင်ပါတယ် ခင်ဗျာ။

### PR ဖွင့်သူ (Author) အနေနဲ့ ကိုယ့်ရဲ့ တာဝန်

- Reviewers တွေ ပေးထားတဲ့ **Feedback** တွေကို ဂရုတစိုက် ဖတ်ပါ။
- **Comments** တွေကို ပြန်ဖြေပါ၊ မေးခွန်းတွေ မေးထားရင် ရှင်းပြပေးပါ။

တကယ်လို့ ပြင်ဆင်ဖို့ လိုအပ်လာပြီ ဆိုရင်တော့ - 
ကိုယ့်ရဲ့ `local feature branch` ပေါ်မှာပဲ ပြင်ဆင်မှုတွေ လုပ်ပါ၊ `commit` လုပ်ပါ၊ ပြီးရင် အဲဒီ Branch ကိုပဲ ပြန်ပြီး `git push` ထပ်လုပ်လိုက်ပါ။ GitHub ပေါ်က ကိုယ့် PR က အလိုအလျောက် Update ဖြစ်သွားပါလိမ့်မယ်။

> **ဒါက စာတည်းမှူးက ကိုယ့် စာမူကို Feedback ပြန်ပေးတာနဲ့ တူပါတယ်။ ကိုယ်က အဲဒီ Feedback အတိုင်း ပြင်ဆင်ပြီး Updated Version ကို ပြန်တင်ပြလိုက်တာမျိုးပါပဲ ခင်ဗျာ။**

<Aside title="အနှစ်ချုပ်">
  PR Review ဆိုတာ အပြုသဘောဆောင်တဲ့ အကြံဉာဏ်တွေ ဖလှယ်ပြီး Code Quality ပိုကောင်းလာအောင် အဖွဲ့လိုက် ဝိုင်းဝန်း လုပ်ဆောင်တာ ဖြစ်ပါတယ်။ လိုအပ်တဲ့ ပြင်ဆင်မှုတွေ လုပ်ပြီးတိုင်း ကိုယ့် Branch ကို ပြန် `push` ဖို့ မမေ့ပါနဲ့ ခင်ဗျာ။
</Aside>
