## ထူးခြားသော အချိန်များကို မှတ်သားခြင်း: `git tag`

**Tag ဆိုတာ ဘာလဲ?**
- သတ်မှတ်ထားတဲ့ Commit တစ်ခုကို တိတိကျကျ ညွှန်ပြပေးတဲ့ အမှတ်အသားတစ်ခု ဖြစ်ပါတယ်။
- History မှာ အဓိပ္ပာယ်ရှိတဲ့ အရေးကြီး Point တစ်ခုကို မှတ်သားဖို့ အဓိက သုံးပါတယ်။
- ဥပမာ - Version အသစ် ထုတ်ဝေမှုများ (Releases: `v1.0`, `v2.0.1`)။

**ဘာကြောင့် သုံးရသလဲ?**
- Code ရဲ့ တိကျတဲ့ Version တွေကို နောက်ပိုင်းမှာ လွယ်လွယ်ကူကူ ပြန်ရှာဖွေနိုင်ဖို့။
- အခြားသူတွေကို Code ရဲ့ သတ်မှတ်ထားတဲ့ နေရာဆီ ပြန်ညွှန်းပေးဖို့ လွယ်ကူစေရန်။

> Project History ဆိုတဲ့ စာအုပ်ကြီးထဲက အရေးကြီးတဲ့ စာမျက်နှာတစ်ခုပေါ်မှာ Bookmark လေး ညှပ်ထားသလိုပါပဲ ခင်ဗျာ။

### Tags များ ဖန်တီးခြင်း

**1. ရိုးရှင်းသော Tag (Lightweight Tag)**

လက်ရှိ `HEAD` ရောက်နေတဲ့ Commit ကို Tag လုပ်ခြင်း -
<Code code={gitTag} title='Bash' />

ကိုယ်လိုချင်တဲ့ သတ်မှတ်ထားတဲ့ Commit ကို Tag လုပ်ခြင်း -
<Code code={gitTagToCommit} title='Bash' />

**2. အချက်အလက်ပါသော Tag (Annotated Tags)**

(Release တွေအတွက် Message ပါတဲ့ ဒီနည်းလမ်းကို ပိုပြီး အကြံပြုပါတယ်)
<Code code={annotatedTag} title='Bash'/>

### Tags များကို ကြည့်ရှုခြင်း

လက်ရှိ ဖန်တီးထားတဲ့ Tags တွေ အားလုံးကို ကြည့်ချင်ရင် -
<Code code={gitTagLists} title='Bash'/>

### Tags များကို မျှဝေခြင်း (Remote သို့ Push လုပ်ခြင်း)

Tag သတ်မှတ်ပြီးပြီ ဆိုပေမဲ့၊ အဲ့ဒီ Tags တွေက Commits တွေလိုပဲ Local စက်ထဲမှာပဲ ရှိနေသေးတာ ဖြစ်ပါတယ်။ အွန်လိုင်း (Remote) ပေါ်ကို တင်ချင်တယ်ဆိုရင်တော့ -

**သတ်မှတ်ထားတဲ့ Tag တစ်ခုတည်းကို Push လုပ်ခြင်း**
<Code code={pushTag} title='Bash' />

**Local မှာရှိတဲ့ Tag အားလုံးကို တစ်ပြိုင်နက် Push လုပ်ခြင်း**
<Code code={pushAllTags} title='Bash' />

<Aside title='အနှစ်ချုပ်'>
  `git tag` က အရေးကြီးတဲ့ Commits (ဥပမာ - Version Releases) တွေကို အလွယ်တကူ မှတ်သားပေးပါတယ်။ အရေးကြီးတဲ့ Releases တွေအတွက် **Annotated Tags (`-a -m`)** ကို သုံးဖို့နဲ့ Remote ဆီကို `push` လုပ်ဖို့ မမေ့ပါနဲ့ ခင်ဗျာ။
