  Rebase နှင့် Merge ကြားက အဓိက ကွာခြားချက်များကို နားလည်ပြီး၊ ဘယ်လို အခြေအနေမှာ ဘယ်နည်းလမ်းကို အသုံးပြုသင့်သလဲ ဆိုတာ လေ့လာနိုင်ဖို့ ဖြစ်ပါတယ် ခင်ဗျာ။
</Aside>

### Merge (ပေါင်းစပ်ခြင်း)

- မူလ Commit History ကို လုံးဝ မပြောင်းလဲပါဘူး။ Branches နှစ်ခု ဘယ်နေရာမှာ ပေါင်းစပ်ခဲ့တယ်ဆိုတာကို "Merge Commit" တစ်ခုနဲ့ ရှင်းရှင်းလင်းလင်း ပြသပေးပါတယ်။
- Branches တွေ ပုံစံအမျိုးမျိုးနဲ့ ခွဲထွက်ပြီး ပြန်ပေါင်းသွားတဲ့ Graph ပုံစံမျိုးကို မြင်ရမှာပါ။
- Push လုပ်ပြီးသား Shared Commits တွေအတွက်ဆိုရင် Merge လုပ်တာက ပိုပြီး လုံခြုံစိတ်ချရပါတယ်။ (လက်ရှိ Commits တွေကို ပြန်ပြင်ရေးတာမျိုး မလုပ်လို့ပါ)။

### Rebase (အသစ်ပြန်စီခြင်း)

- Commit History ကို ပြန်လည်ရေးသားပြီး၊ ခွဲထွက်တာမျိုး မရှိဘဲ တစ်ဆင့်ပြီးတစ်ဆင့် ဖြောင့်တန်းစွာ ဖြစ်ပေါ်ခဲ့တဲ့ ပုံစံမျိုး ဖြစ်အောင် လုပ်ပါတယ်။ (Commit တွေကို နေရာအသစ်တစ်ခုပေါ် ရွှေ့လိုက်တာပါ)။
- Commit History ကို ပိုပြီး သန့်ရှင်း သပ်ရပ်စေလို့ ဖတ်ရလွယ်ကူစေပါတယ်။ (ဖြောင့်တန်းတဲ့ လိုင်းတစ်ကြောင်းလို ဖြစ်သွားပါတယ်)။
- ဒါပေမဲ့ Push လုပ်ပြီးသား Commits တွေမှာ သုံးရင်တော့ ပြဿနာ ဖြစ်နိုင်ပါတယ်။ ကိုယ့် Local မှာပဲ သပ်သပ်ရပ်ရပ် လုပ်ချင်တဲ့အခါ ဒါမှမဟုတ် Push မလုပ်ခင် ကြိုတင် ပြင်ဆင်ချင်တဲ့အခါမျိုးအတွက် အကောင်းဆုံး ဖြစ်ပါတယ် ခင်ဗျာ။
