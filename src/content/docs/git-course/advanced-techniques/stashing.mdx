## "ဒါလေး ခဏ မှတ်ထားပေးပါ!" - `git stash`

### `git stash` ဆိုတာ ဘာလဲ?

- လက်ရှိ အလုပ်လုပ်နေတဲ့ နေရာက Commit မလုပ်ရသေးတဲ့ အပြောင်းအလဲတွေကို ယာယီ (အံဆွဲထဲ ထည့်သလို) သိမ်းဆည်းပေးတာ ဖြစ်ပါတယ်။
- **Staged** ရော **Unstaged Changes** ကိုပါ နှစ်မျိုးလုံး သိမ်းပေးပါတယ်။
- သိမ်းပြီးတာနဲ့ Working Directory ကို နောက်ဆုံး Commit ရှိခဲ့တဲ့ အခြေအနေအတိုင်း သန့်ရှင်းသွားအောင် ပြန်လုပ်ပေးပါတယ်။

### ဘာကြောင့် သုံးရသလဲ?

- အလုပ်တစ်ခု လုပ်လက်စ ရှိနေတုန်း၊ အရေးပေါ် Bug တစ်ခုကို တခြား Branch မှာ ချက်ချင်း သွားပြင်ဖို့ လိုလာတဲ့အခါ။
- တစ်ဝက်တစ်ပျက် ပြီးနေတဲ့ အလုပ်ကို Commit မလုပ်ချင်သေးဘဲ၊ Branch အမြန် ပြောင်းဖို့ လိုအပ်တဲ့အခါ။
- Pull လုပ်ချင်ပေမဲ့ Local မှာ အပြောင်းအလဲတွေ ရှိနေလို့ သန့်ရှင်းတဲ့ အခြေအနေတစ်ခု လိုအပ်လာတဲ့အခါ မျိုးတွေမှာ သုံးပါတယ်။

<Aside title='ဥပမာ' type='tip'>
  ရှုပ်ပွနေတဲ့ စားပွဲတစ်ခုမှာ အလုပ်လုပ်နေတုန်း၊ စားပွဲကို ချက်ချင်း ရှင်းဖို့ လိုလာတဲ့ အချိန်မျိုးနဲ့ တူပါတယ်။ ရှိသမျှ အပြောင်းအလဲ အကုန်လုံးကို အံဆွဲတစ်ခုထဲ အမြန် ထိုးထည့်လိုက်ပြီး၊ လိုအပ်တဲ့ အလုပ်တွေ လုပ်ပြီးမှ အံဆွဲထဲက ပြန်ထုတ်ပြီး ဆက်လုပ်မယ်ဆိုတဲ့ သဘောပါပဲ ခင်ဗျာ။
</Aside>

**3. သိမ်းဆည်းထားတာကို ပြန်ယူခြင်း (ထုတ်ယူခြင်း)**

<Code code={gitStashDrop} title='Bash' />

### လက်တွေ့ စမ်းသပ်ကြည့်ရန်

1. ဖိုင်တစ်ခုခုမှာ အပြောင်းအလဲ တချို့ လုပ်ကြည့်ပါ။
2. `git stash push -m "WIP feature X"` လို့ ရိုက်လိုက်ပါ။
3. `git status` နဲ့ ပြန်ကြည့်ရင် အလုပ်လုပ်နေတဲ့ နေရာကြီး သန့်ရှင်းသွားတာကို တွေ့ရပါလိမ့်မယ်။
4. `git stash list` နဲ့ သိမ်းထားတဲ့ စာရင်းကို ကြည့်ပါ။
5. `git stash pop` လို့ ရိုက်ကြည့်ပါ။ (စောစောက အပြောင်းအလဲတွေ အကုန် ပြန်ပေါ်လာတာကို မြင်တွေ့ရမှာ ဖြစ်ပါတယ် ခင်ဗျာ)။

<Aside title='အနှစ်ချုပ်'>
  `git stash` က Branch တွေ အမြန် ပြောင်းဖို့ ဒါမှမဟုတ် Updates တွေ Pull လုပ်ဖို့ လိုအပ်တဲ့အခါမျိုးမှာ၊ မပြီးသေးတဲ့ အလုပ်တွေကို ယာယီ သိမ်းဆည်းထားဖို့ အကောင်းဆုံး နည်းလမ်းပါ။ `pop` သို့မဟုတ် `apply` ကို သုံးပြီး အလုပ်တွေကို အလွယ်တကူ ပြန်ယူနိုင်ပါတယ် ခင်ဗျာ။
</Aside>
