## Quick Fixes - Commit ပြင်ခြင်းနှင့် Local Changes ကို Undo လုပ်ခြင်း

### `git commit --amend`

နောက်ဆုံး Commit ကို ပြင်ချင်တာ၊ File တစ်ခုခု ထပ်ထည့်ချင်တာ ဒါမှမဟုတ် Commit Message စာလုံးပေါင်း မှားသွားတာတွေကို ဒီ Command နဲ့ ပြင်ဆင်နိုင်ပါတယ်။

**ဘယ်လို လုပ်မလဲ?**

အကယ်၍ File တစ်ခု ထည့်ဖို့ မေ့သွားတယ်ဆိုရင် -

<Code title="Bash" code={fileAdd} />

<Aside type="caution" title="သတိပြုရန်">
  `git commit --amend` ကို Push မလုပ်ရသေးတဲ့ Commits တွေမှာပဲ သုံးပါ။ Amending က History ကို ပြန်ပြင်ရေးတာ ဖြစ်လို့၊ Push ပြီးသားဆိုရင် ပြဿနာ တက်နိုင်ပါတယ် ခင်ဗျာ။
</Aside>

### `git reset HEAD <file>`

ဒါက File တစ်ခုကို **Stage** မှာ ထည့်ထားရာကနေ ပြန်ဖြုတ်တာပါ။ File ထဲက **Changes** တွေကတော့ မပျောက်သွားပါဘူး။ `git add` လုပ်ပြီးမှ မှားသွားတာတွေကို **Unstage** ပြန်လုပ်ချင်ရင် သုံးပါတယ်။

**Command:** `git reset HEAD <file နာမည်>`

(အကုန်လုံးကို Unstage လုပ်ချင်ရင်တော့ `git reset HEAD` လို့ သုံးပါ)

### `git checkout -- <file>` (သတိထားပြီး သုံးပါ)

ဒါက File တစ်ခုထဲက လက်ရှိ ပြင်ထားတဲ့ Changes တွေကို လုံးဝ ပယ်ဖျက်ပစ်လိုက်တာပါ။ File ကို နောက်ဆုံး Commit (HEAD) အတိုင်း ပြန်ပြောင်းပေးပါတယ်။ 

File ထဲမှာ ပြင်ထားတာတွေ မလိုချင်တော့ဘူးဆိုရင် သုံးနိုင်ပါတယ်။

**Command:** `git checkout -- <file နာမည်>`

<Aside type="danger" title="သတိပေးချက်!">
 ဒီ Command ကို သုံးလိုက်ရင် Commit မလုပ်ရသေးတဲ့ Changes တွေ အကုန် ပျက်သွားပါမယ်။ ပြန်ပြင်လို့ မရနိုင်တော့ပါဘူး။ ၁၀၀% သေချာမှသာ သုံးပါ။ (`git restore <filename>` က နည်းနည်း ပိုလုံခြုံပြီး ပိုရှင်းလင်းပါတယ်။ ရနိုင်ရင် `git restore` ကို သုံးဖို့ အကြံပြုပါတယ် ခင်ဗျာ)။
</Aside>

<Aside title="အနှစ်ချုပ်">
  `--amend` က နောက်ဆုံး Commit ကို ပြင်ဖို့၊ `reset HEAD` က Stage ကနေ ပြန်ဖြုတ်ဖို့၊ `checkout --` က Commit မလုပ်ရသေးတဲ့ Changes တွေကို ပယ်ဖျက်ဖို့ အသုံးပြုပါတယ်။
</Aside>
