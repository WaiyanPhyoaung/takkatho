## Commit ကို ပြန်ပြင်ခြင်း: `git revert`

**`git revert` ဆိုတာ ဘာလဲ?**

- အရင် Commit တစ်ခုရဲ့ ပြောင်းလဲမှုတွေကို ပြန်လည် ပယ်ဖျက်ပေးတဲ့ **Commit အသစ်တစ်ခု**ကို ဖန်တီးပေးတာ ဖြစ်ပါတယ်။
- History ထဲက အရင် Commit အဟောင်းကို လုံးဝ မဖျက်ပစ်ပါဘူး။
- မူလ Commit ရဲ့ ဆန့်ကျင်ဘက် ပြောင်းလဲမှုတွေ ပါဝင်တဲ့ Commit အသစ်တစ်ခုကို History မှာ ထပ်ထည့်ပေးလိုက်တာပါ။

**ဘာကြောင့် သုံးရသလဲ?**

- `push` လုပ်ပြီးသား History ကို တခြားသူတွေပါ အသုံးပြုနေနိုင်တဲ့အတွက်၊ ပြဿနာရှိတဲ့ Commit ကို တိုက်ရိုက် ဖျက်ပစ်မယ့်အစား (Reset လုပ်မယ့်အစား) Revert နဲ့ Commit အသစ် ဖန်တီးပြီး ပြန်ပြင်တာက အလုံခြုံဆုံး ဖြစ်ပါတယ်။
- **Commit History** ကို မဖျက်ဘဲ မူလအတိုင်း ထိန်းသိမ်းထားနိုင်လို့ စိတ်အချရဆုံး နည်းလမ်း ဖြစ်ပါတယ် ခင်ဗျာ။

### ဘယ်လို လုပ်မလဲ?

**1. ပြန်ပြင်လိုတဲ့ Commit ID ကို ရှာပါ**
- `git log` သုံးပြီး ပြန်ပြင်ချင်တဲ့ Commit ရဲ့ Hash ID ကို ရှာပါ (ဥပမာ - `a1b2c3d`)။

**3. Commit Message ရေးပါ**
- Git က Commit Message ရေးဖို့ Text Editor ဖွင့်ပေးပါလိမ့်မယ်။
- များသောအားဖြင့် `Revert "original commit message"` ဆိုပြီး အဆင်သင့် ရေးပေးထားပြီးသား ဖြစ်ပါတယ်။
- အဆင်ပြေရင် Save လုပ်ပြီး ပိတ်လိုက်ပါ။

**4. Push လုပ်ပါ**
- `git push` သုံးပြီး Revert လုပ်ထားတဲ့ Commit အသစ်ကို အွန်လိုင်းပေါ် တင်လိုက်ပါ။

## `revert` နဲ့ `reset` ကွာခြားချက်

**`revert` - Shared Branch တွေအတွက် စိတ်ချရပါတယ်**
- History ကို မဖျက်ဘဲ ပြန်ပြင်မယ့် Commit အသစ်ကို ထပ်ထည့်ပေးတာမို့ လုံခြုံပါတယ်။

**`reset` - Shared Branch တွေအတွက် အလွန် အန္တရာယ်များပါတယ်**
- History တွေကို ပြန်ပြင်ရေး (Rewrite) လုပ်ပစ်တာမို့၊ တခြား အတူလုပ်နေသူတွေရဲ့ မှတ်တမ်းတွေနဲ့ ကွဲလွဲပြီး ပြဿနာ အကြီးကြီး ဖြစ်နိုင်ပါတယ်။

<Aside type='tip' title='မှတ်ချက်'>
  `git revert <commit-id>` ဆိုတာ Shared Branch (Remote ကို Push လုပ်ပြီးသား Branch) တွေပေါ်က အမှားတွေကို ပြန်ပြင်ဖို့ အကောင်းဆုံး နည်းလမ်းပါ။ ပြဿနာရှိတဲ့ Commit ရဲ့ ဆန့်ကျင်ဘက် ပြောင်းလဲမှုတွေပါတဲ့ Commit အသစ်တစ်ခုကို ဘေးကင်းစွာ ဖန်တီးပေးတာ ဖြစ်ပါတယ် ခင်ဗျာ။
</Aside>
