### Git ရဲ့ အလုပ်လုပ်ပုံ: နေရာ ၃ မျိုး

Git မှာ ကျွန်တော်တို့ ပြောင်းလဲလိုက်တဲ့ Code တွေကို စနစ်တကျ မှတ်တမ်းတင်ဖို့အတွက် ဖြတ်သန်းရတဲ့ နေရာ (၃) ခု ရှိပါတယ်။ ဒီနေရာတွေက ဘယ်လို အလုပ်လုပ်လဲ ကြည့်လိုက်ကြစို့ ခင်ဗျာ။

<ContentImage src={threeStages} alt="three stages" />

### ၁။ Code များ ပြင်ဆင်ရေးသားရာ နေရာ (Working Directory)

ဒီနေရာကတော့ ကိုယ့် Project Folder ထဲမှာ ဖိုင်တွေကို တိုက်ရိုက် ဖွင့်ပြီး Code တွေ ရေးနေတဲ့၊ ပြင်ဆင်နေတဲ့ အခြေအနေ ဖြစ်ပါတယ်။ ပြောင်းလဲမှု အသစ်တွေ အားလုံးက ဒီနေရာမှာ စတင် ပေါ်လာတာပါ။

**ကိုယ်တိုင် လုပ်ကြည့်ရန်:**
`my-first-repo` Folder ထဲမှာ `README.md` ဖိုင်တစ်ခု ဆောက်ပါ။ အထဲမှာ 'My project's first file.' ဆိုတဲ့ စာသားလေး ထည့်ပြီး Save လိုက်ပါ။

#### `git status` ဖြင့် Project အခြေအနေကို စစ်ဆေးခြင်း

ကိုယ့် Project ထဲမှာ ဘာတွေ ပြောင်းလဲထားလဲ၊ ဘာဖိုင်အသစ်တွေ ထပ်ရောက်နေလဲ ဆိုတာကို သိချင်ရင် `git status` Command ကို သုံးနိုင်ပါတယ်။ ဒီ Command က Git ကနေ ကိုယ့် Project အခြေအနေကို ဘယ်လို မြင်နေလဲဆိုတာ အတိအကျ ပြပေးပါတယ်။ 

`git add` ဒါမှမဟုတ် `git commit` လို Command တွေ မသုံးခင်မှာ ကိုယ့် Project အခြေအနေကို သေချာ သိဖို့အတွက် `git status` ကို အမြဲတမ်း သုံးသင့်ပါတယ် ခင်ဗျာ။

Terminal မှာ `git status` လို့ ရိုက်ကြည့်ပါ။ (`README.md` ကို "Untracked" ဆိုပြီး ပြနေတာ တွေ့ရပါလိမ့်မယ်)။

### ၂။ နောက်တစ်ကြိမ် Save ရန် ပြင်ဆင်သည့် နေရာ (Staging Area)

Staging Area ကတော့ ကိုယ် ပြင်ဆင်ထားတဲ့ ဖိုင်တွေထဲကနေ နောက်တစ်ကြိမ် Save (Commit) လုပ်တဲ့အခါ ပါဝင်စေချင်တဲ့ ဖိုင်တွေကို ရွေးချယ် ပြင်ဆင်ပေးတဲ့ နေရာ ဖြစ်ပါတယ်။

#### `git add` ဖြင့် Staging Area သို့ ထည့်သွင်းခြင်း

ကိုယ် Save လုပ်ချင်တဲ့ ဖိုင်တွေကို Staging Area ထဲ ပို့ဖို့အတွက် `git add <file name>` Command ကို သုံးရပါတယ်။ ဥပမာ - `README.md` ဖိုင်ကို ထည့်ချင်ရင် `git add README.md` လို့ ရိုက်ရပါတယ်။ 

အဲဒီနောက် `git status` ကို ထပ်ရိုက်ကြည့်ရင် 'Changes to be committed' ဆိုပြီး Staging Area ထဲ ရောက်နေတာကို တွေ့ရပါလိမ့်မယ်။ ဒီလို လိုချင်တဲ့ အပိုင်းတွေကိုပဲ ရွေးချယ် ထည့်သွင်းတာက Project ရဲ့ မှတ်တမ်း (History) ကို ပိုပြီး သပ်ရပ် စနစ်ကျစေပါတယ် ခင်ဗျာ။

<ContentImage src={gitAdd} alt="git add" />

### ၃။ အပြီးအပိုင် မှတ်တမ်းတင်ရာ နေရာ (Repository)

Staging Area ထဲမှာ Save လုပ်ချင်တဲ့ အရာတွေ အဆင်သင့် ဖြစ်ပြီဆိုရင်တော့ Repository လို့ခေါ်တဲ့ Project ရဲ့ အပြီးအပိုင် မှတ်တမ်းတိုက်ကြီးထဲကို ထည့်သွင်း Save လုပ်လို့ ရပါပြီ။

#### `git commit` ဖြင့် ပြောင်းလဲမှုများကို သိမ်းဆည်းခြင်း

Staging Area ထဲက ပြင်ဆင်ထားတာတွေကို Repository ထဲ သိမ်းဆည်းဖို့ `git commit` Command ကို သုံးပါတယ်။ ပုံမှန်အားဖြင့် `git commit -m "Commit Message"` ပုံစံနဲ့ ရေးလေ့ရှိပါတယ်။ 

`-m` ရဲ့ နောက်က စာသားလေးက ဒီ Commit (Save Point) မှာ ဘာတွေ ပြောင်းလဲခဲ့လဲ ဆိုတာကို အတိုချုပ် မှတ်ထားပေးတာ ဖြစ်လို့ ရှင်းရှင်းလင်းလင်း ရေးဖို့ အရေးကြီးပါတယ်။ Commit လုပ်ပြီးသွားရင် `git status` ကို ပြန်စစ်ကြည့်တဲ့အခါ 'nothing to commit, working tree clean' ဆိုပြီး ပြနေမှာ ဖြစ်ပါတယ် ခင်ဗျာ။
