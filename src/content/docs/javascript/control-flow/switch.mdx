Condition တွေ အရမ်းများလို့ `if...else if` တွေ အများကြီး ရှည်လျားစွာ ရေးရမယ့် အစား **Switch Statement** ကို အသုံးပြုလိုက်ရင် Code တွေက ပိုမို သပ်ရပ်သွားပြီး ဖတ်ရလွယ်ကူသွားပါလိမ့်မယ်။

## ဘယ်အချိန်မှာ အသုံးပြုမလဲ?

Variable တစ်ခုတည်းရဲ့ တန်ဖိုး (Value) ကိုပဲ တိုက်ရိုက် စစ်ဆေးပြီး၊ အဲ့ဒီ တန်ဖိုးအပေါ် မူတည်ပြီး လုပ်ဆောင်ရမယ့် အလုပ်တွေ ကွဲပြားသွားမယ့် အခြေအနေမျိုးမှာ အသုံးပြုဖို့ အသင့်တော်ဆုံး ဖြစ်ပါတယ်။

## Syntax (ရေးသားပုံ)

```javascript
let day = "Monday";

switch (day) {
  case "Monday":
    console.log("အပတ်စဉ် အစည်းအဝေး ရှိပါတယ်။");
    break; // Break သာ မပါခဲ့ရင် အောက်က Code တွေပါ ဆက်လုပ်သွားလိမ့်မယ်!
  case "Friday":
    console.log("ပိတ်ရက် နီးလာပါပြီ!");
    break;
  case "Saturday":
  case "Sunday": // တန်ဖိုး နှစ်ခုအတွက် အလုပ်တစ်ခုတည်း လုပ်ချင်ရင် ဒီလို ပေါင်းရေးလို့ ရပါတယ်
    console.log("ပိတ်ရက် ဖြစ်ပါတယ်။");
    break;
  default: // အထက်ပါ ဘယ်အရာနဲ့မှ မကိုက်ညီရင် လုပ်မယ့် အလုပ် (If/Else မှ Else နဲ့ သဘောတရား တူပါတယ်)
    console.log("ပုံမှန် ရုံးဖွင့်ရက် ဖြစ်ပါတယ်။");
}
```

## Syntax ကို အသေးစိတ် ရှင်းလင်းချက်

Switch ကို ဘာကြောင့် ဒီလိုရေးရသလဲ ဆိုတာ တစ်ပိုင်းချင်းစီ ခွဲခြား ကြည့်ရအောင်။

1. **`switch (day)`**: "`day` ဆိုတဲ့ Variable ထဲက တန်ဖိုး (Value) ကို စတင် စစ်ဆေးတော့မယ်" လို့ ကြေညာလိုက်တာ ဖြစ်ပါတယ်။
2. **`case "Monday":`**: "`day` ရဲ့ တန်ဖိုးဟာ `Monday` ဖြစ်ခဲ့ရင် အောက်က အလုပ်တွေကို လုပ်ပါမယ်" လို့ သတ်မှတ်တာပါ။
3. **`break;`**: "ဒီနေရာကို ရောက်ပြီဆိုရင် ဆက်မလုပ်နဲ့တော့၊ Switch ရဲ့ အပြင်ဘက်ကို ထွက်လိုက်တော့" လို့ ညွှန်ကြားတာပါ။
    * 🚨 **အထူး သတိထားရန်:** `break;` ရေးဖို့ မေ့သွားခဲ့ရင် အောက်မှာရှိတဲ့ `case` တွေ မှန်သည်ဖြစ်စေ၊ မှားသည်ဖြစ်စေ ဂရုမစိုက်တော့ဘဲ အောက်ကို ဆက်တိုက် အလုပ်လုပ်သွားပါလိမ့်မယ်။ (ဒီလို ဖြစ်စဉ်ကို **Fallthrough** လို့ ခေါ်ဆိုပါတယ်)
4. **`default:`**: `if...else` ထဲက `else` နဲ့ သဘောတရား တူညီပါတယ်။ အပေါ်က ဘယ် `case` နဲ့မှ မကိုက်ညီခဲ့ရင် နောက်ဆုံး အနေနဲ့ အလိုအလျောက် အလုပ်လုပ်သွားမယ့် အပိုင်း ဖြစ်ပါတယ်။

## Switch နှင့် If/Else ယှဉ်တွဲလေ့လာခြင်း

* **Switch:** တန်ဖိုး တစ်ခုကို အတိအကျ (Exact Value) တိုက်ဆိုင် စစ်ဆေးချင်တဲ့ အခါမျိုးမှာ သုံးဖို့ ပိုကောင်းပါတယ်။ (ဥပမာ - `day === "Monday"`)
