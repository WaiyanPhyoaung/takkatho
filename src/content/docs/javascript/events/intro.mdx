---
title: "What Are Events?"
description: "User Interactions (Events) ဆိုတာ ဘာလဲ?"
---

အသက်ဝင်ပြီး တုံ့ပြန်မှု ကောင်းတဲ့ (Interactive) Website တစ်ခု ဖြစ်လာဖို့အတွက် User (အသုံးပြုသူ) က ဘာတွေ လုပ်ဆောင်လိုက်သလဲ ဆိုတာကို စောင့်ကြည့် သိရှိဖို့ လိုအပ်ပါတယ်။ အဲ့ဒီလို User ဘက်က ပြုလုပ်လိုက်တဲ့ အပြုအမူ လုပ်ဆောင်ချက် မှန်သမျှကို JavaScript မှာ **Events** လို့ ခေါ်ဆိုပါတယ်။

## Common Events (အသုံးအများဆုံး Events များ)

* **Mouse Events (မောက်စ် နှင့် သက်ဆိုင်သော):**
  * `click` - Element တစ်ခုကို နှိပ်လိုက်တဲ့အခါ
  * `dblclick` - နှစ်ချက် ဆက်တိုက် (Double-click) နှိပ်လိုက်တဲ့အခါ
  * `mouseover` - Mouse pointer ကို Element အပေါ်ကို တင်လိုက်တဲ့အခါ (Hover)
  * `mouseout` - Mouse pointer ကို Element အပေါ်ကနေ ပြန်ဖယ်လိုက်တဲ့အခါ

* **Keyboard Events (ကီးဘုတ် နှင့် သက်ဆိုင်သော):**
  * `keydown` - ကီးဘုတ် ခလုတ်တစ်ခုခုကို ဖိနှိပ်လိုက်တဲ့အခါ
  * `keyup` - နှိပ်ထားတဲ့ ခလုတ်ကို ပြန်လွှတ်လိုက်တဲ့အခါ

* **Form Events (ဖောင် ဖြည့်သွင်းခြင်း နှင့် သက်ဆိုင်သော):**
  * `submit` - Form ကို Submit လုပ် (အချက်အလက်များ ပေးပို့) လိုက်တဲ့အခါ
  * `input` - Input Box ထဲမှာ စာရိုက်သွင်းနေစဉ် (Real-time ချက်ချင်း အလုပ်လုပ်သည်)
  * `change` - စာရိုက်သွင်းပြီးလို့ Input Box ရဲ့ အပြင်ဘက်ကို ထွက်သွားတဲ့အခါ (Focus ပြုတ်သွားသောအခါ)

Web Developer တွေ အနေနဲ့ ဒီ Events တွေကို အမြဲ စောင့်ကြည့် (Listen) နေပြီး၊ သက်ဆိုင်ရာ Event ဖြစ်ပေါ်လာတဲ့ အခါတိုင်း JavaScript ကို အသုံးပြုကာ လိုအပ်သလို တုံ့ပြန် ဆောင်ရွက် (Respond) ကြရပါတယ်။
