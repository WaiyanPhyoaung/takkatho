- **ကြိုတင်ခန့်မှန်းနိုင်သော State (Predictable state) -** Data flow က **unidirectional** ဖြစ်တဲ့အတွက် app ရဲ့ state က အမြဲတမ်း ကြိုတင်ခန့်မှန်းနိုင်ပါတယ်။

- **Debugging ပိုမိုလွယ်ကူခြင်း -** Redux DevTools လိုမျိုး tool တွေသုံးပြီး state အပြောင်းအလဲတွေကို အလွယ်တကူ ခြေရာခံနိုင်ပါတယ်။

- **Test လုပ်ရလွယ်ကူခြင်း (Testability) -** Reducer တွေက pure function တွေဖြစ်တဲ့အတွက် test လုပ်ရတာ အလွန်လွယ်ကူပါတယ်။

- **Boilerplate Code နည်းပါးခြင်း (Redux Toolkit ကြောင့်) -** Redux Toolkit က action တွေ၊ reducer တွေကို အလိုအလျောက် generate လုပ်ပေးတဲ့အတွက် code ရေးရတာ အများကြီး သက်သာစေပါတယ်။

### Redux မှာ Asynchronous Action များကို ကိုင်တွယ်ခြင်း 
**Redux Toolkit** မှာ **Redux Thunk** ဟာ အသင့်ပါဝင်လာပြီးသားဖြစ်တဲ့အတွက် API call တွေလိုမျိုး asynchronous အလုပ်တွေကို ကိုင်တွယ်ဖို့ ထပ်ပြီး install လုပ်စရာမလိုပါဘူး။ Thunk ကိုသုံးပြီး async logic တွေကို reducer logic တွေနဲ့ သီးခြားခွဲထားနိုင်ပါတယ်။

<Aside title="အနှစ်ချုပ်" type="tip">
- **Flux Architecture** က state ကို စီမံခန့်ခွဲဖို့နဲ့ အပြောင်းအလဲတွေကို ကြိုတင်ခန့်မှန်းနိုင်စေဖို့ **unidirectional data flow** ကို အသုံးပြုတယ်။

- **Redux** က Flux pattern ကို အကောင်အထည်ဖော်ထားပြီး ခေတ်မီရေးသားနည်းဖြစ်တဲ့ **Redux Toolkit** က code တွေကို အများကြီး ပိုရိုးရှင်းစေတယ်။

- `createSlice` ကိုသုံးပြီး state, reducers, နဲ့ actions တွေကို feature တစ်ခုအတွက် တစ်နေရာတည်းမှာ စုစည်းနိုင်တယ်။

- Redux က ကြိုတင်ခန့်မှန်းနိုင်တဲ့ state၊ လွယ်ကူတဲ့ debugging နဲ့ testability ကို ပေးစွမ်းတယ်။

- **Redux Toolkit** မှာ **Redux Thunk** အသင့်ပါဝင်လာတဲ့အတွက် API call တွေလိုမျိုး asynchronous action တွေကို အလွယ်တကူ စီမံခန့်ခွဲနိုင်တယ်။
</Aside>
