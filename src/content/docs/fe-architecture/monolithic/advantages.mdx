### Monolithic Frontend Architecture ၏ အားသာချက်များ
- #####  ရိုးရှင်းမှု (Simplicity):
  အရာအားလုံးက project တစ်ခုတည်းမှာပဲ ရှိတဲ့အတွက် စတင်တည်ဆောက်ရတာ လွယ်ကူပါတယ်။ Development environment က ရိုးရှင်းပြီး ရှုပ်ထွေးတဲ့ app အချင်းချင်း ဆက်သွယ်မှုတွေကို configure လုပ်စရာမလိုပါဘူး။

- ##### Development ပိုမြန်ဆန်ခြင်း (Faster Development):
  Developer တွေက project ကို အပိုင်းတွေခွဲဖို့ စိတ်ပူစရာမလိုဘဲ feature တွေ တည်ဆောက်တဲ့အပေါ်မှာပဲ တိုက်ရိုက်အာရုံစိုက်နိုင်ပါတယ်။

- ##### Codebase တစ်ခုတည်းဖြစ်ခြင်း (Single Codebase):
  အရာအားလုံး (UI components, business logic, assets) က တစ်နေရာတည်းမှာပဲ ရှိတဲ့အတွက် ဝင်ရောက်ကြည့်ရှုရ၊ ပြင်ဆင်ရ၊ update လုပ်ရတာ လွယ်ကူပါတယ်။

- ##### Debugging နှင့် Testing လွယ်ကူခြင်း:
  App တစ်ခုလုံးက codebase တစ်ခုတည်းမှာပဲ ရှိတဲ့အတွက် အထူးသဖြင့် application အသေးစားတွေမှာ test လုပ်ရတာနဲ့ debug လုပ်ရတာ ပိုလွယ်ကူပါတယ်။

### Monolithic Frontend Architecture ၏ အားနည်းချက်များ
- ##### Scaling ပြုလုပ်ရာတွင် အခက်အခဲရှိခြင်း:
  Application က ကြီးလာတာနဲ့အမျှ codebase တစ်ခုတည်းကို စီမံခန့်ခွဲရတာ ပိုခက်ခဲလာပါတယ်။ App ရဲ့ performance ကျဆင်းလာနိုင်ပြီး ပိုရှုပ်ထွေးလာရင် scale လုပ်ရခက်လာနိုင်ပါတယ်။

- ##### တင်းကျပ်စွာ ချိတ်ဆက်နေခြင်း (Tight Coupling):
  Monolithic app မှာ component တွေက ချိတ်ဆက်နေတဲ့အတွက် app ရဲ့ တစ်နေရာမှာ ပြောင်းလဲမှုတစ်ခုလုပ်လိုက်ရင် တခြားနေရာတွေကို သွားထိခိုက်နိုင်ပါတယ်။

- ##### တစ်ခုပျက်လျှင် အားလုံးပျက်နိုင်ခြင်း (Single Point of Failure):
  အရာအားလုံးက project တစ်ခုတည်းမှာပဲ ရှိတဲ့အတွက် app ရဲ့ ဘယ်အစိတ်အပိုင်းမှာမဆို ပြဿနာတစ်ခုဖြစ်လာရင် app တစ်ခုလုံးကို ရပ်တန့်သွားစေနိုင်ပါတယ်။

- ##### ပြောင်းလွယ်ပြင်လွယ်မရှိခြင်း (Limited Flexibility):
  Code တွေကို folder တွေခွဲပြီး စုစည်းထားနိုင်ပေမယ့် ဒါဟာ project တစ်ခုတည်းပါပဲ။ အနာဂတ်မှာ scale လုပ်ချင်ရင် ဒါမှမဟုတ် feature အသစ်တွေ ထပ်ထည့်ချင်ရင် app ကို သီးခြား module တွေအဖြစ် ခွဲထုတ်ဖို့ ခက်ခဲလာနိုင်ပါတယ်။
