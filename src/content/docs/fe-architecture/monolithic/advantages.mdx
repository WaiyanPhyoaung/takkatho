### အားသာချက်များ (Studio Apartment ၏ ကောင်းကွက်များ)

1.  **Simple Deployment (ပြောင်းရွှေ့ရ လွယ်ကူခြင်း):**
    Studio ခန်းလေး ပြောင်းမယ်ဆိုရင် ပစ္စည်းသယ်ရတာ လွယ်ပါတယ်။ သေတ္တာတစ်လုံးတည်းနဲ့ အကုန်ဆန့်ပါတယ်။
    (အလားတူပဲ၊ Monolith မှာ Code ကို Build လုပ်မယ်၊ Server ပေါ်တင်မယ်၊ ပြီးပြီ။ ဘာမှ ရှုပ်ရှုပ်ထွေးထွေး Pipeline တွေ မလိုပါဘူး)။

2.  **Development Speed (အလုပ်တွင်ခြင်း):**
    ရေသောက်ချင်ရင် မီးဖိုချောင်ကို လက်လှမ်းလိုက်ရုံပဲ။ အိပ်ချင်ရင် ကုတင်ပေါ် လှဲချလိုက်ရုံပါပဲ။
    (Code တွေ၊ Component တွေ အားလုံးက နေရာတည်းမှာမလို့ ယူသုံးရတာ အရမ်းလွယ်တယ်။ Code တွေကို ပြန်ပြင်ရေး (Refactor) လုပ်ရတာလည်း မြန်ပါတယ်)။

3.  **Performance (စရိတ်သက်သာခြင်း):**
    အခန်းကျဉ်းလေးဆိုတော့ မီးဖိုး၊ ရေဖိုး သက်သာပါတယ်။
    (Browser က File တစ်ခုတည်းကိုပဲ (Bundle တစ်ခုတည်း) Load လုပ်ရလို့ မြန်ပါတယ်။ Browser က မှတ်ထားပေးတဲ့ Caching စနစ်အတွက်လည်း ပိုထိရောက်ပါတယ်)။

4.  **Testing (စစ်ဆေးရ လွယ်ကူခြင်း):**
    အခန်းထဲမှာ ကြွက်ရှိ၊ မရှိ ရှာချင်ရင် တစ်ခန်းလုံးကို ချက်ချင်း မျက်စိနဲ့ ပတ်ကြည့်နိုင်ပါတယ်။
    (End-to-End Testing လုပ်တဲ့အခါ App တစ်ခုလုံးကို တစ်ခါတည်း အစအဆုံး စမ်းသပ်စစ်ဆေးလို့ လွယ်ပါတယ်)။

### အားနည်းချက်များ (Studio Apartment ၏ ဆိုးကွက်များ)

1.  **All-or-Nothing Deployment (တခါပြင် တခါတင်):**
    မီးလုံးတစ်လုံး လဲချင်ရင်တောင် တစ်ခန်းလုံး မီးပိတ်ရသလိုမျိုး ဖြစ်နိုင်ပါတယ်။
    (စာလုံးတစ်လုံးလေး သွားပြင်ချင်ရင်တောင် App တစ်ခုလုံးကို ပြန် Build လုပ်ပြီး အသစ် ပြန်တင်ရပါတယ်)။

2.  **Scaling Issues (လူများလာရင် ကျပ်လာခြင်း):**
    Studio ခန်းလေးထဲကို လူ ၁၀ ယောက်လောက် ဝင်နေရင် ကြပ်ပြီး တစ်ယောက်နဲ့တစ်ယောက် တိုက်မိကုန်ပါလိမ့်မယ်။
    (Developer ၅၀၊ ၁၀၀ လောက်က Codebase ကြီးတစ်ခုတည်းမှာ ဝိုင်းရေးကြတဲ့အခါ တစ်ယောက်ရေးတာနဲ့ တစ်ယောက် ထပ်ပြီး (Merge Conflicts တွေ ဖြစ်ပြီး) အလုပ်လုပ်ရတာ ခက်လာပါတယ်)။

3.  **Technology Lock-in (ပြင်ဆင်ရ ခက်ခဲခြင်း):**
    အခန်း ဖွဲ့စည်းပုံကို ပြောင်းချင်ရင် တစ်ခန်းလုံး ဖြိုပစ်ရမှာပါ။
    (Project ကို React နဲ့ ရေးထားရင် ကြားထဲကနေ Vue နဲ့ ရောရေးလို့ မရပါဘူး။ Framework တစ်ခုတည်း အပေါ်မှာပဲ မှီခိုနေရပါတယ်)။

4.  **Single Point of Failure (တစ်ခုပျက် အကုန်ပျက်):**
    ရေပိုက်တစ်ခု ပေါက်တာနဲ့ တစ်ခန်းလုံး ရေကြီးသွားပါလိမ့်မယ်။
    (Build လုပ်တဲ့ လုပ်ငန်းစဉ် (Process) မှာ Error တစ်ခုတက်တာနဲ့ App တစ်ခုလုံး ဘယ်လိုမှ Deploy လုပ်လို့ မရတော့ပါဘူး)။
