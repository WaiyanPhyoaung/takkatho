  ဒီသင်ခန်းစာမှာတော့ Frontend Development မှာသုံးတဲ့ **MVC, MVVM, MVP architecture patterns** တွေအကြောင်း လေ့လာရမှာပါ။ ဒီ pattern တွေက code တွေကို အလွှာတွေခွဲခြားပြီး ရေးသားတဲ့အတွက် ပြင်ဆင်ထိန်းသိမ်းရလွယ်ကူခြင်း၊ တိုးချဲ့ရလွယ်ကူခြင်း၊ test လုပ်ရလွယ်ကူခြင်း စသည့် အကျိုးကျေးဇူးတွေ ရရှိနိုင်ပါတယ်။

### ဒီသင်ခန်းစာမှာ ဘာတွေလေ့လာရမလဲ
- MVC, MVVM, MVP pattern တွေဆိုတာ ဘာလဲ။
- Model, View နဲ့ Controller/Presenter/ViewModel တို့ရဲ့ အခန်းကဏ္ဍ။
- ဒီ pattern တွေက UI, logic နဲ့ data ကို ဘယ်လို ခွဲထုတ်ပေးသလဲ။
- ဒီ pattern တွေကို သုံးထားတဲ့ framework ဥပမာတွေ (Angular, React)။
- လက်တွေ့ project တွေမှာ ဘယ် pattern ကို ဘယ်အချိန်မှာ သုံးသင့်သလဲ။

### MV Architecture ဆိုတာ ဘာလဲ။
ဒီ Architecture က application တစ်ခုရဲ့ **data** (အချက်အလက်), **UI** (မြင်တွေ့ရတဲ့အပိုင်း), နဲ့ **control logic** (ထိန်းချုပ်တဲ့အပိုင်း) တွေကို သီးခြား layer တွေအဖြစ် ခွဲထုတ်ပေးတဲ့ architecture pattern အုပ်စုတစ်ခု ဖြစ်ပါတယ်။

အဓိက ရည်ရွယ်ချက်ကတော့ application ကို အစိတ်အပိုင်းလေးတွေနဲ့ ဖွဲ့စည်းတည်ဆောက်ဖို့၊ ပြင်ဆင်ထိန်းသိမ်းရလွယ်ကူစေဖို့နဲ့ စမ်းသပ်ရလွယ်ကူစေဖို့ ဖြစ်ပါတယ်။

### MVC (Model-View-Controller)
  - **Model:** Application ရဲ့ **data နဲ့ business logic** တွေ ရှိတဲ့နေရာ။
  - **View:** User မြင်တွေ့ရပြီး **အပြန်အလှန်သုံးစွဲရတဲ့** UI အပိုင်း။
  - **Controller:** User ရဲ့ input တွေကို လက်ခံပြီး Model သို့မဟုတ် တခါတလေ တိုက်ရိုက် View ကို **update လုပ်ပေးတဲ့** အပိုင်း။

  #### MVC ရဲ့ အလုပ်လုပ်ပုံ
    1. User က View မှာ တစ်ခုခုလုပ်လိုက်တယ်။
    2. Controller က အဲ့ဒီ input ကို နားလည်ပြီး Model ကို update လုပ်တယ်။
    3. Model က View ကို UI update လုပ်ဖို့ အကြောင်းကြားတယ်။

    > တချို့ MVC တွေမှာတော့ model က controller ကိုပဲ data ပြန်ပို့ပြီး controller က View ကို data ပေးပြီး UI ကို update လုပ်စေပါတယ်။

**ဥပမာ Frameworks:** Backbone.js

### MVVM (Model-View-ViewModel)
  - **Model:** Data နဲ့ business logic တွေကို သိမ်းထားတဲ့နေရာ။
  - **View:** User မြင်တွေ့ရတဲ့ UI အပိုင်း။
  - **ViewModel:** Model နဲ့ View ကို ချိတ်ဆက်ပေးတယ်။ Data တွေကို UI မှာ အလိုအလျောက် ချိတ်ဆက်ပြသပေးတယ် (binding)။

**အဓိက အချက် -** Two-way binding ပါ။ View မှာဖြစ်တဲ့ အပြောင်းအလဲက Model ကို အလိုအလျောက် update လုပ်သလို၊ Model မှာဖြစ်တဲ့ အပြောင်းအလဲကလည်း View ကို အလိုအလျောက် update လုပ်ပေးပါတယ်။

**ဥပမာ Frameworks:** Angular, Vue

### MVP (Model-View-Presenter)
  - **Model:** Data နဲ့ business logic တွေ ရှိတဲ့နေရာ။
  - **View:** User ကို ပြသထားတဲ့ UI အပိုင်း။
  - **Presenter:** User interaction အားလုံးကို ကိုင်တွယ်ပြီး View နဲ့ Model နှစ်ခုလုံးကို update လုပ်ပေးတယ်။

**MVC နဲ့ ဘာကွာလဲဆိုတော့** Presenter က View နဲ့ တိုက်ရိုက်ချိတ်ဆက်ပြီး UI update တွေကို ပိုပြီး ထိန်းချုပ်နိုင်စွမ်း ရှိပါတယ်။ MVC မှာတော့ Controller က Model ကိုပဲ အဓိကထား update လုပ်ပါတယ်။
