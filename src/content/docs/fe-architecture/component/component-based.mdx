### ဒီသင်ခန်းစာမှာ ဘာတွေလေ့လာရမလဲ
- Component-based architecture ဆိုတာ ဘာလဲ။

- Component နဲ့ Monolithic code (အားလုံးကို တစ်နေရာတည်းမှာ ရေးတဲ့ code) ဘာကွာလဲ။

- Component code တွေက ပြန်သုံးဖို့နဲ့ ပြင်ဆင်ထိန်းသိမ်းဖို့အတွက် ဘယ်လို လွယ်ကူစေသလဲ။

- Component တွေကို project ထဲမှာ ဘယ်လို စုစည်းထားမလဲ။

### Component-Based Architecture ဆိုတာဘာလဲ။
- UI code အားလုံးကို file တစ်ခုတည်းမှာ အကုန်စုရေးမယ့်အစား၊ သီးခြားလည်ပတ်နိုင်တဲ့ အစိတ်ပိုင်းငယ်လေးတွေအဖြစ် ခွဲထုတ်လိုက်တာကို Component လို့ခေါ်ပါတယ်။

- Component တစ်ခုချင်းစီမှာ UI ရဲ့ သတ်မှတ်ထားတဲ့ အပိုင်းတစ်ခုကိုပဲ တာဝန်ယူပြီး နေရာတော်တော်များများမှာ ပြန်ပြန်သုံးလို့ရပါတယ်။

- **ဥပမာ -** ဝဘ်ဆိုဒ်တစ်ခုရဲ့ navbar, footer, login form နဲ့ product card တွေဟာ သီးခြား Component တွေအနေနဲ့ တည်ဆောက်လို့ရပါတယ်။

### Component နဲ့ Monolithic Code ဘာကွာလဲ။

| Monolithic (အားလုံးစုရေးတဲ့ ပုံစံ) | Component-Based (အစိတ်အပိုင်းတွေခွဲရေးတဲ့ ပုံစံ) |
|---|---|
| UI အားလုံးကို ဖိုင်ကြီးတစ်ခုတည်းမှာ စုရေးထားတယ်။ | UI ကို သီးခြားအစိတ်အပိုင်းလေးတွေအဖြစ် ခွဲရေးထားတယ်။ |
| Application ကြီးလာရင် ပြင်ဆင်ထိန်းသိမ်းဖို့ ခက်ခဲတယ်။ | ပြင်ဆင်ထိန်းသိမ်းဖို့နဲ့ update လုပ်ဖို့ ပိုလွယ်ကူတယ်။ |
| ကုဒ်တွေ ထပ်ခါထပ်ခါ ရေးရနိုင်တယ်။ | ပြန်သုံးလို့ရတဲ့ Component တွေကြောင့် ကုဒ်ထပ်တာ နည်းသွားတယ်။ |

Component တွေကို ပြန်သုံးလို့ ရတဲ့အတွက် UI တစ်ခုတည်းကို အကြိမ်ကြိမ်ပြန်ရေးစရာ မလိုတော့ပါဘူး။

### Project ထဲမှာ Component တွေကို ဘယ်လို စုစည်းမလဲ။
Project တစ်ခုလုံးကို Component-Based ပုံစံနဲ့ မတည်ဆောက်ထားရင်တောင်မှ Component တွေကို `components` ဆိုတဲ့ folder တစ်ခုထဲမှာ စုစည်းထားလေ့ရှိပါတယ်။

```
/src
  /components
    Navbar.js
    Footer.js
    ProductCard.js
    Button.js
  /pages
    Home.js
    Products.js
  /services
    api.js
```
Component file တွေထဲမှာ HTML structure (React မှာဆို JSX, Angular/Vue မှာဆို templates), logic နဲ့ တစ်ခါတလေ style တွေ ပါဝင်ပါတယ်။ ထပ်ပြီး navbar folder ဆိုပြီး components folder ထဲမှာ ထပ်ခွဲထားလို့လည်း ရပါတယ်။
