<Aside title="ခရီးဆုံးသို့ ရောက်ရှိခြင်း">
ကျွန်တော်တို့ လေ့လာခဲ့တဲ့ Frontend Architecture ဆိုတဲ့ ခရီးရှည်ကြီးဟာ အခုဆိုရင် ခရီးဆုံးကို ရောက်ရှိလို့ လာပါပြီ။
ဒါဆိုရင် ကျွန်တော်တို့ ဘာတွေ လေ့လာခဲ့ကြလဲ? အနှစ်ချုပ် ပြန်ကြည့်ရအောင်။
</Aside>

### ၁။ အိမ်ဆောက်ခြင်း (Architecture Basics)
အိမ်တစ်လုံး ဆောက်သလိုပါပဲ။ အုတ်မြစ်မခိုင်ရင် အိမ်ပြိုတတ်ပါတယ်။
- **Monolith:** Studio ခန်းလေး (ရိုးရှင်းတယ်၊ အရာရာ တစ်နေရာတည်းမှာ စုစည်းနေတယ်)။
- **Micro-Frontend:** Mansion အိမ်ကြီး (ခမ်းနားတယ်၊ အခန်းတွေ အများကြီးနဲ့ ရှုပ်ထွေးတယ်)။

### ၂။ LEGO ရဲတိုက် (Component Architecture)
Code တွေကို LEGO တုံးလေးတွေလို အသေးစားလေးတွေ ကနေ စတင်ပြီး ဖွဲ့စည်းတတ်ဖို့ လိုပါတယ်။
- **Atomic Design:** အသေးဆုံး အစိတ်အပိုင်း (Atom) တွေကနေ စတင်ပြီး၊ တဖြည်းဖြည်း ကြီးမားတဲ့ (Organism) တွေဆီ တည်ဆောက်သွားခြင်း။

### ၃။ စမတ်ဖုန်း (Modular Architecture)
စမတ်ဖုန်း တစ်လုံးလိုပါပဲ။ Camera Module ပျက်သွားရင် Camera အပိုင်းကိုပဲ လဲလို့ရအောင်، သီးခြားစီ အလုပ်လုပ်နိုင်အောင် ဖွဲ့စည်းခြင်း။

### ၄။ Shopping Mall (Micro-Frontends)
ဆိုင်ခန်းပေါင်းစုံ စုထားတဲ့ Mall အကြီးကြီးတစ်ခု လိုပါပဲ။ Team တွေ အများကြီး၊ နည်းပညာတွေ (React, Vue, Angular) အများကြီးကို Shell တစ်ခုတည်းအောက်မှာ လွတ်လပ်စွာ စုစည်းပေးထားခြင်း။

### ၅။ စားသောက်ဆိုင် (MV* Patterns)
- **Model:** မီးဖိုချောင် (Data နဲ့ Logic တွေ ရှိတဲ့ နေရာ)
- **View:** စားပွဲဝိုင်း (User တွေ မြင်ရတဲ့ UI နေရာ)
- **Controller/ViewModel:** စားပွဲထိုး (Customer ဆီက Order ယူပြီး၊ မီးဖိုချောင်ကို ပို့ပေးတဲ့သူ)

### ၆။ ဘဏ်စနစ် (State Management)
- **Flux/Redux:** ဘဏ်တစ်ခုမှာလိုပဲ ငွေထုတ်/ငွေသွင်း လုပ်တဲ့အခါ စာရွက်စာတမ်း (Action/Reducer) တွေနဲ့ တိတိကျကျ၊ စနစ်တကျ မှတ်တမ်းတင် လုပ်ဆောင်ခြင်း။

### ၇။ အစားအသောက် ပို့ဆောင်ခြင်း (Rendering)
- **CSR:** ကိုယ်တိုင် ချက်စား (Meal Kit)။ (Browser မှာပဲ အကုန်အလုပ်လုပ်တယ်)
- **SSR:** ဆိုင်က မှာစား (Delivery)။ (Server ကနေ အဆင်သင့် ချက်ပြီး ပို့ပေးတယ်)
- **SSG:** စည်သွပ်ဘူး (Frozen Food)။ (Build Time မှာကတည်းက ကြိုတင် ချက်ပြုတ်ထားတယ်)
- **ISR:** မုန့်တိုက် (Bakery)။ (ကြိုတင် ဖုတ်ထားပေမယ့်၊ အချိန်တန်ရင် အသစ်ပြန်ဖုတ်တယ်)

---

### အနာဂတ် ခေတ်ရေစီးကြောင်း (The Future)

Frontend လောကကြီးက ဘယ်ကို ဦးတည်နေလဲ?
အဖြေကတော့ **"Server Components"** ပါပဲ။ (React Server Components - RSC)

အရင်တုန်းက Browser (Client) ပေါ်မှာ ဝန်တွေ အရမ်း ပိနေပါတယ်။ JavaScript တွေ အများကြီး Download လုပ်ရ၊ Browser ပေါ်မှာ Run ရနဲ့၊ ဖုန်းအဟောင်းတွေမှာဆိုရင် Website က အရမ်း လေးလံပါတယ်။
အခုတော့ **"Server Components"** ဆိုတဲ့ နည်းပညာနဲ့ အလုပ်တော်တော်များများကို Server ပေါ် ပြန်တင်ပြီး၊ Browser ကို အလုပ် သက်သာအောင် လုပ်ပေးနေကြပါပြီ။

> **"Back to the Server (Server ဆီသို့ ပြန်သွားခြင်း)"**
> စားသောက်ဆိုင် ဥပမာနဲ့ ပြန်ပြောရရင် - အရင်က စားပွဲဝိုင်း (Browser) အထိ လာပြီး ချက်ပြုတ်ပေးနေတာ (CSR)။ အခုတော့ မီးဖိုချောင် (Server) မှာပဲ အားလုံး အသင့်ချက်ပြုတ်ပြီးမှ၊ စားပွဲဝိုင်းကို အဆင်သင့် လာချပေးတာမျိုး ပြန်ဖြစ်လာပါပြီ။
