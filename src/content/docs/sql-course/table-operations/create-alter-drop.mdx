အခုချိန်ထိ လေ့လာခဲ့တာကတော့ ရှိပြီးသား Data တွေကို ဘယ်လို ရှာမလဲ၊ အသစ် ဘယ်လို ထည့်မလဲ ဆိုတာကိုပါ။ ဒါပေမဲ့ အဲ့ဒီ Data တွေကို သိမ်းဖို့အတွက် ဆိုရင် အရင်ဆုံး **Table (ဇယား)** တစ်ခု တည်ဆောက်ထားဖို့ လိုအပ်ပါတယ်။

Table တွေရဲ့ ဖွဲ့စည်းပုံကို တည်ဆောက်တာ၊ ပြင်ဆင်တာတွေကို Data Type (ဒေတာ အမျိုးအစား) တွေနဲ့ တွဲပြီး လုပ်ဆောင်တဲ့ အပိုင်းကို **DDL (Data Definition Language)** လို့ ခေါ်ပါတယ်။

## ဇယားများ ဖန်တီးခြင်း (`CREATE TABLE`)

Table တစ်ခု စဆောက်တော့မယ် ဆိုရင် ဘယ် Column မှာ ဘယ်လို Data မျိုး ထည့်မလဲ ဆိုတာ သတ်မှတ်ပေးရပါတယ်။

**အသုံးများသော Data Types (ဒေတာ အမျိုးအစားများ):**
- `VARCHAR(255)` : စာလုံးရေ 255 လုံးအထိ ထည့်လို့ရတဲ့ စာသား (Text) ကို ဆိုလိုပါတယ်။
- `INT` : ကိန်းပြည့်ဂဏန်းများ (1, 2, 3...) ကို သိမ်းဆည်းဖို့ပါ။
- `BOOLEAN` : မှန်/မှား (True/False) တန်ဖိုးများကို သိမ်းဖို့ပါ။

ဥပမာ - User တွေရဲ့ အချက်အလက်ကို မှတ်သားဖို့ `users` ဆိုတဲ့ Table သစ်တစ်ခု ဖန်တီးမယ် ဆိုပါစို့။

```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
*(အဓိပ္ပာယ်: Table နာမည်က `users` ဖြစ်ပြီး၊ Column (၄) ခု ပါဝင်ပါမယ်။ 
`id` က User တစ်ယောက်ချင်းစီကို သီးခြား ခွဲသိဖို့ (Primary Key) ဖြစ်ပြီး နံပါတ်တွေက သူ့အလိုလို တိုးသွားပါမယ် (Auto Increment)။ 
`name` နဲ့ `email` က မဖြစ်မနေ ထည့်ရပါမယ် (NOT NULL)။ `email` ဟာ တစ်ယောက်နဲ့ တစ်ယောက် တူလို့ မရပါဘူး (UNIQUE)။ 
Account ဖွင့်တဲ့ အချိန်ကို မှတ်ထားဖို့ `created_at` ပါဝင်ပါတယ်)*။

## ဇယားများ ပြင်ဆင်ခြင်း (`ALTER TABLE`)

### Column အသစ် ထပ်ပေါင်းထည့်ခြင်း
Table ကို ဖန်တီးပြီးသွားမှ နောက်ထပ် Column အသစ် တစ်ခုခု ထပ်တိုးချင်တဲ့အခါ သုံးပါတယ်။

```sql
ALTER TABLE users 
ADD phone_number VARCHAR(15);
```
*(အဓိပ္ပာယ်: `users` Table ထဲမှာ `phone_number` ဆိုတဲ့ Column အသစ်ကို ထပ်ပေါင်း ထည့်လိုက်ပါတယ်)*။

## ဇယားများ ဖျက်ပစ်ခြင်း (`DROP TABLE`)

### သတိထားရမယ့် အပိုင်း (All Data Lost)
ဒါက Table တစ်ခုလုံးကိုရော၊ အထဲမှာ ရှိနေတဲ့ Data တွေ အားလုံးကိုပါ အစအနမကျန် ဖျက်ပစ်တာပါ။ `DELETE` လိုမျိုး မှတ်တမ်း (Row) တစ်ခုတည်းကို ရွေးဖျက်တာမျိုး မဟုတ်ဘဲ အားလုံးကို ဖျက်ဆီးပစ်တာ ဖြစ်ပါတယ်။

```sql
DROP TABLE users;
```

> [!CAUTION]
> `DROP TABLE` ကို သုံးလိုက်ရင် အထဲက Data တွေ အကုန်လုံး ပျက်သွားပြီး Backup မရှိရင် ပြန်ယူလို့ မရတော့ပါဘူး။ ဒါကြောင့် အလွန် သတိထားပြီး အသုံးပြုသင့်ပါတယ်။

## အကျဉ်းချုပ်

ဒီလောက်ဆိုရင် Database ထဲမှာ Table တွေ ဘယ်လို ဖန်တီးမလဲ၊ လိုအပ်ရင် ဘယ်လို ပြင်ဆင်မလဲ၊ ပြီးတော့ Data တွေကို ဘယ်လို စီမံခန့်ခွဲမလဲ ဆိုတဲ့ အခြေခံတွေကို ကောင်းကောင်း နားလည်သွားပြီ ဖြစ်ပါတယ်။ **Web Development** ရဲ့ Backend ပိုင်းမှာ Data တွေ ဘယ်လို အလုပ်လုပ်သလဲ ဆိုတာကို သဘောပေါက်သွားပါပြီ ခင်ဗျာ။
