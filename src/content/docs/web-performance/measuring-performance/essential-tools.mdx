## မရှိမဖြစ် လိုအပ်သော Tools များ 🛠️

Web Performance ကို တိုင်းတာဖို့ သုံးလို့ကောင်းတဲ့ Tools တွေ အများကြီး ရှိပါတယ်။ ဝမ်းသာစရာ ကောင်းတာက အဲ့ဒီ Tools အများစုဟာ အခမဲ့ (Free) အသုံးပြုနိုင်တာပဲ ဖြစ်ပါတယ်။

### ၁။ Browser Developer Tools (Chrome, Firefox စသည်တို့တွင် ပါဝင်ပြီးသား)

Browser တိုင်းမှာ ပါဝင်တဲ့ Developer Tools (F12 နှိပ်ပြီး ခေါ်လို့ရတဲ့ဟာ) ဟာ ပထမဆုံးနဲ့ အသုံးအဝင်ဆုံး Tool ပါပဲ။ အဓိကအားဖြင့် အောက်ပါ Tab သုံးခုကို အသုံးပြုရပါတယ်။

#### Network Tab:
- ကိုယ့် Website ကနေ လှမ်းတောင်းသမျှ ဖိုင်တွေ (HTML, စာသား၊ ပုံ၊ JavaScript, CSS) အားလုံးကို တန်းစီ ပြသပေးပါတယ်။
- ဖိုင်တစ်ခုချင်းစီကို Download ဆွဲဖို့ ဘယ်လောက် အချိန်ကြာသလဲ ဆိုတာကို **Waterfall** (ရေတံခွန်) ပုံစံ ဇယားလေးနဲ့ ရှင်းရှင်းလင်းလင်း ပြပေးပါတယ်။
- ဒီ Tab ကနေတစ်ဆင့် ဘယ်ဖိုင်က အရွယ်အစား အကြီးဆုံးဖြစ်နေလဲ (Size ကြီးနေလဲ)၊ ဘယ်ဖိုင်က ဒေါင်းလုပ်ဆွဲရတာ အကြာဆုံး ဖြစ်နေလဲ (Block ဖြစ်နေလဲ) ဆိုတာကို ချက်ချင်း သိရှိနိုင်ပါတယ်။

#### Performance Tab:
- Website တစ်ခုလုံး စတင် ပွင့်လာပုံကို အသေးစိတ် မှတ်တမ်းတင် (Record) ပြီး ပြသပေးပါတယ်။
- Website ဘာလို့ နှေးနေရသလဲဆိုတဲ့ အဓိက တရားခံ (ဥပမာ - ဘယ် JavaScript က အချိန်အကြာကြီး ယူနေသလဲ၊ Layout Shift ဘယ်အချိန်မှာ ဖြစ်သွားသလဲ) တွေကို အသေးစိတ် ရှာဖွေဖို့ (Debugging လုပ်ဖို့) အရမ်းကို အသုံးဝင်ပါတယ်။

#### Lighthouse Tab:
- Website ရဲ့ Speed (မြန်နှုန်း)၊ Accessibility (သုံးရလွယ်ကူမှု)၊ Best Practices နဲ့ SEO (Google မှာ ရှာဖွေရ လွယ်ကူမှု) စတာတွေကို အလိုအလျောက် စစ်ဆေးပေးပါတယ်။
- အရေးကြီးတဲ့ တိုင်းတာမှု (Metrics) တွေအတွက် အမှတ် (၀ ကနေ ၁၀၀ အထိ) ပြသပေးပါတယ်။
- Website ကို ဘယ်လို ပိုကောင်းအောင် ပြင်ဆင်ရမလဲ ဆိုတဲ့ တိုက်ရိုက် အကြံပြုချက် (Actionable Advice) တွေကိုပါ တစ်ပါတည်း ဖော်ပြပေးပါတယ်။

### ၂။ Google PageSpeed Insights (PSI)

- [pagespeed.web.dev](https://pagespeed.web.dev/) မှာ အခမဲ့ သုံးနိုင်တဲ့ Google ရဲ့ တရားဝင် Tool တစ်ခုပါ။
- Website ရဲ့ URL ကို ထည့်လိုက်တာနဲ့၊ ကိုယ်တိုင် စမ်းသပ်ထားတဲ့ ရလဒ် (Lab Data) သာမကဘဲ၊ တကယ် လက်တွေ့ သုံးစွဲနေတဲ့ သူတွေဆီကနေ ကောက်ယူထားတဲ့ ရလဒ် (Field Data / Chrome User Experience Report - CrUX) ကိုပါ နှိုင်းယှဉ် ပြသပေးပါတယ်။
- Website ရဲ့ အရေးကြီးတဲ့ Core Web Vitals အခြေအနေတွေနဲ့ ဘာတွေ ပြင်ရမလဲ ဆိုတာကို အလွယ်တကူ သိနိုင်ပါတယ်။

### ၃။ WebPageTest.org

- [WebPageTest.org](https://www.webpagetest.org/) ဟာ ကမ္ဘာပေါ်က မတူညီတဲ့ နေရာဒေသတွေ၊ မတူညီတဲ့ ဖုန်း/ကွန်ပျူတာ အမျိုးအစားတွေနဲ့ အင်တာနက်လိုင်း အမြန်နှုန်း အနေအထား အမျိုးမျိုး (ဥပမာ - 3G, 4G) ကနေ ကိုယ့် Website ကို လှမ်းပြီး စမ်းသပ်ပေးနိုင်တဲ့ Tool ပါ။
- အလွန် အသေးစိတ်ကျတဲ့ အချက်အလက်တွေ ရရှိနိုင်သလို၊ Website ပွင့်လာပုံ အဆင့်ဆင့်ကို ဗီဒီယို (Filmstrip) ပုံစံနဲ့ပါ မြင်သာအောင် ပြသပေးတဲ့အတွက် အလွန် အသုံးဝင်ပါတယ်။

### ၄။ Google Search Console (GSC)

- Google ကပေးတဲ့ အခမဲ့ ဝန်ဆောင်မှုတစ်ခု ဖြစ်ပြီး၊ ကိုယ့် Website ဟာ Google Search မှာ ဘယ်လို ပုံစံ ပေါ်ထွက်နေသလဲ ဆိုတာကို စောင့်ကြည့်နိုင်ပါတယ်။
- **Performance အတွက် အရေးပါပုံ:** တကယ့် သုံးစွဲသူ (Real Users) တွေအတွက် ကိုယ့် Website က လက်တွေ့မှာ ဘယ်လောက် မြန်ဆန်နေသလဲ ဆိုတာကို စောင့်ကြည့် ပြသပေးတဲ့ "Core Web Vitals" Report တွေ သီးသန့် ပါဝင်ပါတယ်။

### တခြား Tools များ (အကျဉ်းချုပ်)

- **Synthetic Monitoring Services (စက်ရုပ်များဖြင့် အလိုအလျောက် စောင့်ကြည့်ခြင်း):** GTmetrix, Calibre, SpeedCurve, DebugBear (ဒါတွေက အချိန်နဲ့အမျှ Performance အတက်အကျကို ဇယားတွေနဲ့ ပြပေးပါတယ်)။
- **Real User Monitoring - RUM Tools (တကယ့် သုံးစွဲသူများဆီမှ စောင့်ကြည့်ခြင်း):** Sentry, Datadog, New Relic (ကိုယ့် User အစစ်တွေဆီကနေ Field Data တွေကို အမြဲတမ်း စုဆောင်းပြီး ခွဲခြမ်းစိတ်ဖြာဖို့အတွက် အသုံးပြုပါတယ်)။

## လက်တွေ့ လေ့ကျင့်မှု

၁။ သင် သဘောကျတဲ့ နာမည်ကြီး Website နှစ်ခုကို ရွေးချယ်ကြည့်ပါ (တစ်ခုက အရမ်းမြန်ပြီး၊ နောက်တစ်ခုက နည်းနည်း နှေးပါစေ)။
၂။ Browser DevTools ထဲက `Lighthouse` ကို သုံးပြီး အဲ့ဒီ Website တွေကို စစ်ဆေးကြည့်ပါ။ အမှတ်တွေ ဘယ်လောက် ရသလဲ ယှဉ်ကြည့်ပါ။
၃။ `PageSpeed Insights` (PSI) မှာလည်း URL ထည့်ပြီး ခွဲခြမ်းစိတ်ဖြာကြည့်ပါ။
၄။ `WebPageTest.org` မှာပါ သွားရောက် စမ်းသပ်ကြည့်ပါ။
၅။ ရလာတဲ့ အဖြေတွေကို ယှဉ်ကြည့်ပြီး၊ အဲ့ဒီ Website တွေ ပိုမြန်လာအောင် ဘာတွေ ပြင်ဆင်သင့်သလဲ ဆိုတာကို လေ့လာ ရှာဖွေကြည့်ပါ ခင်ဗျာ။

<Aside title="💡 အနှစ်ချုပ်">
  - **LCP, FID/INP, CLS (Core Web Vitals)** တို့အပြင် **FCP, TTFB, TTI, TBT** ဆိုတဲ့ အဓိက တိုင်းတာမှု (Metrics) တွေက User တစ်ယောက် Website သုံးတဲ့အခါ ဘယ်လို ခံစားရသလဲ (မြန်သလား၊ နှေးသလား၊ ချောမွေ့သလား) ဆိုတာကို ရှုထောင့် အမျိုးမျိုးကနေ တိုင်းတာပေးပါတယ်။ 
  - **Browser DevTools, PageSpeed Insights, WebPageTest.org နဲ့ Google Search Console** တို့လို Tools တွေက ကိုယ့် Website ရဲ့ Speed ကို တိုင်းတာဖို့၊ ပြဿနာတွေ ရှာဖွေ ဖြေရှင်းဖို့ (Debugging) နဲ့ အမြဲတမ်း စောင့်ကြည့်ဖို့ (Monitoring) အတွက် အကောင်းဆုံး နည်းလမ်းတွေကို ပံ့ပိုးပေးထားပါတယ်။
</Aside>
