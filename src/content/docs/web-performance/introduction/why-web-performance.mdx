<Aside title="💡 ရည်ရွယ်ချက်">
 **Web Performance** (Website ရဲ့ မြန်နှုန်းနဲ့ စွမ်းဆောင်ရည်) ဆိုတာ သုံးစွဲသူ (Users) တွေ ပျော်ရွှင်ဖို့၊ စီးပွားရေးလုပ်ငန်းတွေ အောင်မြင်ဖို့နဲ့၊ Website ရေးသားသူ (Developer) တစ်ယောက်အနေနဲ့ ဘယ်လောက် အရေးကြီးသလဲဆိုတာကို နားလည် သဘောပေါက်စေဖို့ ဖြစ်ပါတယ်။
</Aside>

## Web Performance ဆိုတာ ဘာလဲ?

Web Performance ဆိုတာ Website တစ်ခုလုံး အလုပ်လုပ်ပုံ ဘယ်လောက် ချောမွေ့ ကောင်းမွန်သလဲ ဆိုတာကို တိုင်းတာတာပါ။ Page တစ်ခု ပွင့်လာတာ မြန်ရုံတင် မဟုတ်ပါဘူး ခင်ဗျာ။

- သုံးစွဲသူတွေအနေနဲ့ အချက်အလက်တွေကို ဘယ်လောက် မြန်မြန် မြင်တွေ့ရသလဲ။
- Page တစ်ခုက ဘယ်လောက် မြန်မြန် **Interactive** ဖြစ်သလဲ (ဥပမာ - ခလုတ်နှိပ်တာ၊ Scroll ဆွဲတာတွေကို ချက်ချင်း တုံ့ပြန်သလား)။
- Loading ဖြစ်နေတုန်းမှာ စာတွေ၊ ပုံတွေ ဟိုဘက်ဒီဘက် ရွေ့သွားပြီး (Layout Shift) စိတ်မရှည်စရာ ဖြစ်သလား... စတာတွေကိုပါ ခြုံငုံ ခေါ်ဆိုတာပါ။

> အလွယ်ပြောရရင်၊ သုံးရတာ ချောမွေ့ပြီး စိတ်ချမ်းသာစေတဲ့ Website နဲ့၊ သုံးရတာ နှေးကွေးပြီး စိတ်ရှုပ်စရာ ကောင်းတဲ့ Website တို့ရဲ့ အဓိက ကွာခြားချက်ပါပဲ။

**ဥပမာ -** Online Store Websites နှစ်ခုကို ယှဉ်စဉ်းစားကြည့်ပါ။

 - **ဆိုဒ် (A)** က ချက်ချင်း ပွင့်လာတယ်။ ပုံတွေ ချက်ချင်း ပေါ်တယ်။ ပစ္စည်းတွေကို နှိပ်ပြီး ချက်ချင်း ကြည့်လို့ရတယ်။
 - **ဆိုဒ် (B)** ကတော့ အကြာကြီး နေမှ ပွင့်လာတယ်။ ပြီးမှ "ဝယ်ရန်" ခလုတ် ပေါ်လာတယ်။ အဲ့ဒီ ခလုတ်ကို နှိပ်လိုက်တော့လည်း ခဏလောက်ကြာတဲ့အထိ ဘာမှ မတုံ့ပြန်ဘူး။

> ဒီနှစ်ခုဆိုရင် ဘယ်ဆိုင်ကနေ ပစ္စည်းဝယ်ဖြစ်ဖို့ ပိုများမလဲ? Users တွေရဲ့ **ခံစားချက် (User Experience)** က အလွန် အရေးကြီးပါတယ် ခင်ဗျာ။

## Website ကြီး နှေးနေရင် သုံးစွဲသူတွေ ဘာဖြစ်မလဲ?

အွန်လိုင်းမှာ ကျွန်တော်တို့ အားလုံးက **စိတ်မရှည်** ကြပါဘူး။ အချက်အလက်တွေကို **ချက်ချင်း** လိုချင်ကြတယ်။ Website က ကြာနေရင် နောက်တစ်ခုကို ပြောင်းချင်စိတ် ချက်ချင်း ဖြစ်လာပါတယ်။

### လူတွေ ထွက်သွားကြတယ် (Bounce Rate မြင့်လာတယ်)

Link တစ်ခုကို နှိပ်ပြီး မျက်နှာပြင် အဖြူကြီးကို ခဏတာ စိုက်ကြည့်ဖူးပါသလား? ပြီးတော့ စိတ်ပျက်ပြီး ပြန်ထွက်သွားဖူးပါသလား? ဒါကို **Bounce** လုပ်တယ်လို့ ခေါ်ပါတယ်။ Website နှေးတာကြောင့် သုံးစွဲသူတွေကို အလကား ဆုံးရှုံးလိုက်ရတာပါ။

- လေ့လာမှုတွေအရ၊ စာမျက်နှာ တက်ဖို့ တစ်စက္ကန့် ပိုကြာတိုင်း၊ လူတွေ ပိုပိုပြီး ထွက်သွားကြပါတယ်။ သူတို့က ထိုင်မစောင့်ကြပါဘူး။

### အာရုံစိုက်မှု နည်းတယ်

အွန်လိုင်းမှာ ကျွန်တော်တို့က တစ်နေရာကနေ တစ်နေရာကို အမြန် ပြောင်းလဲ သွားလာတတ်ကြပါတယ်။ သင့်ဆိုဒ်က မြန်မြန် မပွင့်ဘူး၊ ချက်ချင်း ဆွဲဆောင်မှု မပေးနိုင်ဘူးဆိုရင် သူတို့က အခြား Website တစ်ခုကို ပြောင်းသွားကြမှာ သေချာပါတယ်။

## ဘာကြောင့် Website တစ်ခု မြန်ဖို့ ဒီလောက် အရေးကြီးတာလဲ?

Website က နှေးနေရင် Users တွေ ဆုံးရှုံးရုံတင် မဟုတ်ပါဘူး။ တကယ်တော့ တခြား အရာတွေပါ အများကြီး ဆုံးရှုံးနိုင်ပါတယ်။

### ၁။ သုံးစွဲသူ ပျော်ရွှင်မှု (User Satisfaction) ❤️

မြန်ဆန်တဲ့ Websites တွေက သုံးစွဲသူတွေကို ပျော်ရွှင်စေပါတယ်။ သူတို့က အချိန်ကုန် သက်သာပြီး အလုပ်မြန်မြန် ပြီးတယ်လို့ ခံစားရတယ်။ သုံးစွဲသူတွေ ပျော်ရွှင်ရင် ဆိုဒ်မှာ ကြာကြာနေမယ်၊ နောက်ထပ်လည်း ထပ်လာကြပါလိမ့်မယ်။

### ၂။ ဝင်ငွေနဲ့ ရောင်းအား တက်စေတယ် (Increased Conversions) 💰

ပစ္စည်းရောင်းတဲ့ လုပ်ငန်း (E-commerce) တွေအတွက် ဒါက အသက်ပါပဲ။ Page မြန်မြန် တက်လေ၊ ရောင်းအား ပိုတက်လေပါပဲ။ အကောင့် ဖွင့်ခိုင်းတာ (Register) ပဲဖြစ်ဖြစ်၊ ပစ္စည်း ဝယ်တာပဲဖြစ်ဖြစ်၊ မြန်လေ ပိုပြီး ကောင်းလေပါပဲ။

### ၃။ Google မှာ အဆင့် (Ranking) ပိုကောင်းတယ် (Better SEO) 📈

Google က သူ့ရဲ့ သုံးစွဲသူတွေကို အကောင်းဆုံး ဆိုဒ်တွေကိုပဲ ပြချင်ပါတယ်။ မြန်တဲ့ ဆိုဒ်တွေက Google ရှာဖွေမှု ရလဒ် (Search Results) တွေမှာ အပေါ်ဆုံးကို ပိုရောက်နိုင်ပါတယ်။ ဒါကြောင့် Google က "Site Speed" နဲ့ "Core Web Vitals" ဆိုတာတွေကို အရေးကြီးတဲ့ အချက်တွေအဖြစ် တရားဝင် သတ်မှတ်ထားပါတယ်။

> ဆိုဒ်မြန်လေ = Google ရှာဖွေမှုမှာ အဆင့် ပိုမြင့်လေ = လူတွေ သင့်ဆိုဒ်ကို ပိုရောက်လာလေပါပဲ။

### ၄။ လူတိုင်း သုံးဖို့ ပိုအဆင်ပြေတယ် (Accessibility) ♿

ဒါကလည်း အရမ်း အရေးကြီးပါတယ်။ ဖုန်းအဟောင်း သုံးတဲ့သူတွေ၊ အင်တာနက် လိုင်းသိပ်မကောင်းတဲ့ နေရာက သူတွေအတွက် နှေးတဲ့ Website တွေက သုံးရတာ ပိုခက်ခဲပါတယ်။ သင့်ဆိုဒ်ကို မြန်မြန်ဆန်ဆန်နဲ့ ပေါ့ပေါ့ပါးပါး ဖြစ်အောင် လုပ်ပေးခြင်းက လူတိုင်းအတွက် ပိုပြီး အဆင်ပြေအောင် ဖန်တီးပေးတာပါပဲ။

### ၅။ နာမည်ကောင်း ရစေတယ်၊ ယုံကြည်မှု တည်ဆောက်နိုင်တယ် ✨

သင့် Website ဟာ သင့်လုပ်ငန်းရဲ့ ပထမဆုံး မြင်ရတဲ့ မျက်နှာစာ (First Impression) ဖြစ်နေတတ်ပါတယ်။ နှေးကွေးတဲ့ ဆိုဒ်က "ဒီလူတွေက သိပ် Professional မဆန်ဘူး" ဒါမှမဟုတ် "ယုံလို့မရဘူး" ဆိုတဲ့ အထင်အမြင်မျိုး ပေးနိုင်ပါတယ်။ မြန်ဆန်ပြီး သပ်ရပ်တဲ့ ဆိုဒ်တစ်ခုကတော့ "ကျွန်တော်တို့က ကိုယ့်အလုပ်ကို သေသေချာချာ ဂရုစိုက်တယ်၊ ယုံကြည်လို့ရတယ်" လို့ သွယ်ဝိုက် ပြောပြနေသလိုပါပဲ။

### ၆။ ဖုန်းနဲ့ သုံးလို့ ရဖို့ အရေးကြီးတယ် (Mobile First) 📱

အခုခေတ်မှာ Website သုံးတဲ့သူ အများစုက ဖုန်းနဲ့ (Mobile) သုံးကြပါတယ်။ Google ကလည်း **"Mobile-first Indexing"** လို့ခေါ်တဲ့၊ မိုဘိုင်းဆိုဒ်ရဲ့ စွမ်းဆောင်ရည်ကို အဓိကထား အကဲဖြတ်တဲ့ စနစ်ကို သုံးနေပါပြီ။ မိုဘိုင်းဖုန်းတွေက Desktop တွေလောက် စွမ်းဆောင်ရည် မမြင့်သလို၊ အင်တာနက် (Mobile Data) လည်း အမြဲတမ်း သိပ်မကောင်းတတ်ပါဘူး。 ဒါကြောင့် သင့်ဆိုဒ်ကို မိုဘိုင်းဖုန်းတွေမှာပါ မြန်အောင် မလုပ်ထားဘူးဆိုရင်၊ သုံးစွဲသူ အများကြီးကို ဆုံးရှုံးနိုင်ပါတယ်။

> အခုဆိုရင် Website မြန်ဖို့ ဘယ်လောက် အရေးကြီးလဲဆိုတာ ခံစားမိပြီလား ခင်ဗျာ? ဒီ Course မှာ ကျွန်တော်တို့က ဒီပြဿနာတွေကို ဘယ်လို ဖြေရှင်းရမလဲ ဆိုတာကို အသေးစိတ် သင်ယူကြမှာပါ။ Website Performance ပိုင်းမှာ ဘယ်နေရာတွေက အားနည်းနေသလဲ ရှာဖွေမယ်၊ ဘယ်လို အကောင်းဆုံး ပြုပြင်မလဲ (Optimize) နဲ့ သင့် Websites တွေကို ဘယ်လို အံ့မခန်း မြန်ဆန်ပြီး သုံးရတာ အဆင်ပြေအောင် လုပ်မလဲဆိုတာ သင်ယူကြပါမယ်။

  အချုပ်အားဖြင့် ပြောရရင် Website မြန်ဆန်ခြင်းဟာ အရမ်းကို အရေးကြီးပါတယ်။ ဒါဟာ သုံးစွဲသူတွေကို ပျော်ရွှင်စေတာ၊ လုပ်ငန်း အောင်မြင်စေတာ၊ လူသိများစေတာနဲ့ လူတိုင်းအတွက် ပိုကောင်းအောင် လုပ်ဆောင်ပေးခြင်းပဲ ဖြစ်ပါတယ်။ Web Development မှာ Performance ဟာ Feature တစ်ခုမဟုတ်ပါဘူး၊ အဓိက အကျဆုံး အခြေခံအုတ်မြစ်ပါပဲ။
