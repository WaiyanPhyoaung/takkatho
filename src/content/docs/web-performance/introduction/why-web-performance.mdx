 **Website performance** ဆိုတာ သုံးစွဲသူတွေ ပျော်ရွှင်ဖို့၊ စီးပွားရေးများ အောင်မြင်ဖို့နဲ့ website ရေးတဲ့သူတစ်ယောက်အနေနဲ့ ဘယ်လောက် အရေးကြီးလဲဆိုတာ သိရဖို့ပါ။

## Web Performance ဆိုတာ ဘာလဲ။

Web Performance ဆိုတာ Website တစ်ခုလုံး အလုပ်လုပ်ပုံ ဘယ်လောက် ကောင်းလဲ ဆိုတာကို ပြောတာပါ။ Page တစ်ခု တက်လာတာ မြန်ရုံတင် မဟုတ်ပါဘူး။

- သုံးစွဲသူတွေ ဘယ်လောက် မြန်မြန် မြင်ရလဲ။
- Page တစ်ခုက ဘယ်လောက် မြန်မြန် **interactive** ဖြစ်လဲ (ဥပမာ- ခလုတ်နှိပ်တာ၊ scroll ဆွဲတာ)။
- loading ဖြစ်နေတုန်း စာတွေ၊ ပုံတွေ ရွေ့သွားပြီး စိတ်မရှည်စရာ ဖြစ်မဖြစ် စတာတွေကို ပြောတာပါ။

> ဒါက သုံးရတာ ချောချောမွေ့မွေ့ ဖြစ်စေတဲ့ website နဲ့၊ သုံးရတာ နှေးကွေးပြီး စိတ်ရှုပ်စရာ ကောင်းတဲ့ website တို့ရဲ့ ကွာခြားချက်ပါပဲ။

**ဥပမာ။** Online Store Websites နှစ်ခုကို စဉ်းစားကြည့်ပါ။

 - ဆိုဒ် (A) က ချက်ချင်းတက်တယ်။ ပုံတွေ ပေါ်တယ်။ ပစ္စည်းတွေကို ချက်ချင်းနှိပ်ပြီး ကြည့်လို့ရတယ်။
 - ဆိုဒ် (B) ကတော့ အကြာကြီး နေမှ တတ်တယ်။ ပြီးမှ "ဝယ်ရန်" ခလုတ် ပေါ်လာတယ်။ အဲဒီခလုတ်ကို နှိပ်တော့လည်း နည်းနည်းကြာသည်အထိ ဘာမှ မဖြစ်ဘူး။

> ဘယ်ဆိုင်ကနေ ပစ္စည်းဝယ်ဖို့ ပိုများမလဲ။ Users တွေ**ခံစားရတဲ့ Feeling** က အရမ်းအရေးကြီးပါတယ်။

## Website ကြီးနှေးနေရင် သုံးစွဲသူတွေ ဘာဖြစ်လဲ?

အွန်လိုင်းမှာ ကျွန်တော်တို့အားလုံးက **စိတ်မရှည်** ကြပါဘူး။ အချက်အလက်တွေကို **ချက်ချင်း** လိုချင်ကြတယ်။ Website က ကြာနေရင် နောက်တစ်ခုကို ပြောင်းချင်စိတ် ဖြစ်လာပါတယ်။

- ### လူတွေ ထွက်သွားကြတယ်။

        Link တစ်ခုကို နှိပ်ပြီး မျက်နှာပြင် ဗလာကြီးကို ခဏတာ စိုက်ကြည့်ဖူးလား။ ပြီးတော့ စိတ်ပျက်ပြီး ထွက်သွားဖူးလား။ ဒါကို **Bounce** လို့ ခေါ်ပါတယ်။ Website နှေးတာကြောင့် သုံးစွဲသူ ဆုံးရှုံးသွားတာပါ။

        - လေ့လာမှုတွေအရ၊ စာမျက်နှာ တက်ဖို့ တစ်စက္ကန့် ပိုကြာတိုင်း၊ လူတွေ ပိုပိုပြီး ထွက်သွားကြပါတယ်။ သူတို့ ထိုင်မစောင့်ကြပါဘူး။

- ### အာရုံ စိုက်မှု နည်းတယ်။

အွန်လိုင်းမှာ ကျွန်တော်တို့က တစ်နေရာကနေ တစ်နေရာကို အမြန်ပြောင်းလဲကြပါတယ်။ သင့်ဆိုဒ်က မြန်မြန်မတက်ဘူး၊ ချက်ချင်း ဆွဲဆောင်မှုမပေးနိုင်ဘူးဆိုရင် သူတို့က တခြား Website ကို ထွက်သွားကြမှာပါ။


## ဘာကြောင့် Website တစ်ခုက မြန်ဖို့ ဒီလောက် အရေးကြီးတာလဲ။

Website က နှေးနေရင် Users တွေ ဆုံးရှုံးရုံတင် မဟုတ်ပါဘူး။ တကယ်တော့ အများကြီး ဆုံးရှုံးနိုင်ပါတယ်။

### 1. သုံးစွဲသူ ပျော်ရွှင်မှု ❤️

မြန်တဲ့ Websites တွေက သုံးစွဲသူတွေကို ပျော်ရွှင်စေတယ်။ သူတို့က အချိန်ကုန်သက်သာပြီး အလုပ်မြန်မြန် ပြီးတယ်လို့ ခံစားရတယ်။ သုံးစွဲသူတွေ ပျော်ရွှင်ရင် ဆိုဒ်မှာ ကြာကြာနေမယ်၊ နောက်ထပ် ထပ်လာကြမယ်။

### 2. ပိုက်ဆံ ပိုရတယ်။ ရောင်းအား တက်တယ်။ 💰

ပစ္စည်းရောင်းတဲ့ လုပ်ငန်းတွေအတွက် ဒါက အရမ်းအရေးကြီးတယ်။ Page မြန်မြန် တက်လေ၊ ရောင်းအား ပိုတက်လေ ပါပဲ။ သင် Register လုပ်ခိုင်းတာပဲ ဖြစ်ဖြစ်၊ ပစ္စည်းဝယ်တာပဲ ဖြစ်ဖြစ်၊ မြန်လေ ပိုပြီး ကောင်းလေပါပဲ။

### Google မှာ Ranking ပိုကောင်းတယ်။ (SEO ကောင်းလာတယ်။) 📈

Google က သူ့သုံးစွဲသူတွေကို အကောင်းဆုံး ဆိုဒ်တွေပဲ ပြချင်တယ်။ မြန်တဲ့ဆိုဒ်တွေက Google မှာ ပိုပေါ်နိုင်တယ်။ ဒါကြောင့် Google က "ဆိုဒ် မြန်နှုန်း" (Site Speed) နဲ့ "Core Web Vitals" ဆိုတာတွေကို အရေးကြီးတဲ့ အချက်တွေအဖြစ် သတ်မှတ်ထားတယ်။ 

> ဆိုဒ်မြန်လေ = Google မှာ search result ပိုမြင့်လေ = လူတွေ သင့်ဆိုဒ်ကို ပိုလာလေပါပဲ။

### လူတိုင်း သုံးဖို့ ပိုအဆင်ပြေတယ်။ ♿

ဒါက အရမ်းအရေးကြီးတယ်။ ဖုန်းအဟောင်းတွေ၊ အင်တာနက် နှေးတဲ့သူတွေအတွက် နှေးတဲ့ website တွေက သုံးရတာ ပိုခက်ခဲတယ်။ သင့်ဆိုဒ်ကို မြန်မြန်ဆန်ဆန်နဲ့ ပေါ့ပေါ့ပါးပါး ဖြစ်အောင် လုပ်ခြင်းက လူတိုင်းအတွက် ပိုပြီး သုံးလို့ကောင်းအောင် လုပ်ပေးတာပါပဲ။

### နာမည်ကောင်း ရစေတယ်။ ယုံကြည်မှု တည်ဆောက်တယ်။ ✨

သင့် Website ဟာ သင့်လုပ်ငန်းရဲ့ ပထမဆုံး မြင်ရတဲ့ နေရာ ဖြစ်နေတတ်တယ်။ နှေးတဲ့ဆိုဒ်က "ဒီလူတွေက သိပ် Professional မဆန်ဘူး" ဒါမှမဟုတ် "ယုံလို့မရဘူး" လို့ ထင်စေနိုင်တယ်။ မြန်ပြီး သပ်ရပ်တဲ့ ဆိုဒ်တစ်ခုက "ကျွန်တော်တို့က ကိုယ့်အလုပ်ကို ဂရုစိုက်တယ်၊ ယုံကြည်လို့ရတယ်" လို့ ပြောပြနေသလိုပါပဲ။

### ဖုန်း နဲ့ သုံးလို့ ရဖို့ အရေးကြီးတယ်။ 📱

အခုဆို Website သုံးတဲ့သူ အများစုက ဖုန်းနဲ့ သုံးကြတယ်။ Google ကလည်း **"mobile-first indexing"** လို့ခေါ်တဲ့၊ မိုဘိုင်းဆိုဒ်ရဲ့ စွမ်းဆောင်ရည်ကို ပိုအလေးထားတဲ့ စနစ်ကို သုံးတယ်။ မိုဘိုင်းဖုန်းတွေက desktop တွေထက် စွမ်းဆောင်နိုင်စွမ်းနည်းပြီး အင်တာနက်လည်း သိပ်မကောင်းတတ်ဘူး။ ဒါကြောင့် သင့်ဆိုဒ်ကို မိုဘိုင်းဖုန်းနဲ့ မြန်အောင် မလုပ်ထားဘူးဆိုရင်၊ လူတွေ အများကြီးကို ဆုံးရှုံးနိုင်ပါတယ်။

> အခုဆို Website မြန်ဖို့ ဘယ်လောက်အရေးကြီးလဲဆိုတာ ခံစားမိပြီလား။ ဒီ Course မှာ ကျွန်တော်တို့က ဒီပြဿနာတွေကို ဘယ်လို ဖြေရှင်းရမလဲ ဆိုတာ သင်ယူကြမှာပါ။ Website Performance ပိုင်းမှာ ဘယ်နေရာတွေ အားနည်းလဲ ရှာမယ်၊ ဘယ်လို အကောင်းဆုံး လုပ်မလဲ၊ သင့် Websites တွေကို ဘယ်လို အံ့မခန်း မြန်ပြီး သုံးရတာ ကောင်းအောင် လုပ်မလဲဆိုတာ သင်ယူကြပါမယ်။
