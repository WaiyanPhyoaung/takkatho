### Content များကို ကြိုတင် Load လုပ်ခြင်း
Website ပေါ်တွင် Page တစ်ခုမှတစ်ခုသို့ ကူးပြောင်းခြင်းကို မျက်စိတစ်မှိတ်အတွင်း ဖြစ်သွားသကဲ့သို့ လျင်မြန်အောင် Predictive loading နည်းပညာများနှင့် လုပ်ဆောင်နိုင်ပါတယ်။

### Predictive Prefetching (ကြိုတင်ခန့်မှန်းပြီး Data များ Fetch ခြင်း)

- **ဘာလဲ -** User က နောက်ထပ် ဘယ် link ကို click လုပ်နိုင်ခြေအရှိဆုံးလဲဆိုတာကို  ခန့်မှန်းကာ၊ User က လက်ရှိ Page ကို ဖတ်နေစဉ်မှာပင် နောက် Page အတွက် content များကို နောက်ကွယ်မှ တိတ်တဆိတ် download လုပ်ထားခြင်း ဖြစ်သည်။
- **ဘယ်လိုအလုပ်လုပ်လဲ -**
    - အသုံးများသော signal တစ်ခုမှာ အသုံးပြုသူက link တစ်ခုပေါ်တွင် သူ၏ mouse ကို အချိန်ခဏမျှ (ဥပမာ - 100ms) တင်ထားလိုက်ခြင်း ဖြစ်သည်။ ဒါက ထို link ကို click လုပ်နိုင်ခြေရှိကြောင်း ခိုင်မာသော အရိပ်အယောင်တစ်ခုဖြစ်သည်။
    - Quicklink ကဲ့သို့သော JavaScript library များက အဲ့လိုအခြေအနေမျိုးကို သိရှိနိုင်ပြီး Page ကို အလိုအလျောက် prefetch လုပ်ပေးနိုင်သည်။
- **ဘာကြောင့်မြန်တာလဲ -** အသုံးပြုသူက ထို link ကို အမှန်တကယ် click လိုက်သည့်အခါ၊ စာမျက်နှာသည် browser ၏ cache ထဲတွင် တစ်စိတ်တစ်ပိုင်း သို့မဟုတ် အပြည့်အစုံ download လုပ်ပြီးသားဖြစ်နေသောကြောင့် ချက်ချင်းနီးပါး ပေါ်လာသည်။
- **စဉ်းစားရမည့်အချက် -** အကယ်၍ သင် prefetch လုပ်ထားသော စာမျက်နှာကို အသုံးပြုသူက ဝင်မကြည့်ဖြစ်ခဲ့လျှင် သူ၏ data ကို ဖြုန်းတီးရာကျနိုင်သည်။ ဒါပေမယ့် ဒီ tool များက smart ဖြစ်ပြီး၊ များသောအားဖြင့် connection မြန်ဆန်မှသာ prefetch လုပ်လေ့ရှိသည်။

### Speculation Rules API
- **ဘာလဲ -** browser များတွင် တစ်ခါတည်းပါဝင်လာသော API အသစ်တစ်ခုဖြစ်ပြီး၊ developer များအား custom JavaScript မလိုအပ်ဘဲ HTML ထဲမှာပင် Page များကို ကြိုတင် render လုပ်ရန် သတ်မှတ်နိုင်သည်။
- **ဘယ်လိုအလုပ်လုပ်လဲ -** `<script type="speculationrules">` tag ကိုထည့်သွင်းပြီး browser အား "ဤ Page ပေါ်ရှိ မည်သည့် link ကိုမဆို User က hover လုပ်သည့်အခါ၊ ထို Page ကို ကြိုတင် render လုပ်ထားလိုက်ပါ" ဟု ညွှန်ကြားနိုင်သည်။
- **Pre-rendering နှင့် Prefetching ကွာခြားချက် -** Pre-rendering သည် prefetching ထက် တစ်ဆင့်ပို၍ ရှေ့ရောက်သည်။ ၎င်းက HTML ကို download လုပ်ရုံသာမက၊ sub-resource အားလုံး (CSS, JS) ကိုပါ download လုပ်ပြီး၊ Page တစ်ခုလုံးကို နောက်ကွယ်ရှိ မမြင်ရသော tab တစ်ခုတွင် render ပါလုပ်ထားသည်။ အသုံးပြုသူက click လိုက်သည့်အခါ၊ browser က ထိုမမြင်ရသော Page ကို  ပြသလိုက်ရုံသာဖြစ်သည်။
- **ဘာကြောင့်မြန်တာလဲ -** ဤနည်းလမ်းက Page ကူးပြောင်းမှုကို "ချက်ချင်း" ဖြစ်စေသည်။
