## Primary Key

Primary Key ဆိုတာ database table တစ်ခုမှာရှိတဲ့ record(row) တစ်ခုစီကို unique ဖြစ်စွာ ထုတ်ယူပေးနိုင်သော field (သို့မဟုတ်) set of fields ဖြစ်ပြီး 
တခြား table တွေနဲ့ relationship တည်ဆောက်ဖို့ကိုလည်း ကူညီပေးပါတယ်။
Primary Key တစ်ခုဟာ Candidate Key အတွက် သတ်မှတ်ထားတဲ့ စည်းမျဉ်းတွေအတိုင်း အတိအကျ ကိုက်ညီရပါမယ်။ ဒီလိုအပ်ချက်ကို ပြည့်မီဖို့ကတော့ လွယ်ကူပါတယ်။ 
ဘာကြောင့်လဲဆိုတော့ Primary Key ကို table ထဲမှာရှိတဲ့ Candidate Key တွေထဲကနေ ရွေးချယ်ရတာမလို့ပဲ ဖြစ်ပါတယ်။
ကောင်းမွန်စွာ ဒီဇိုင်းဆွဲထားတဲ့ relational database ထဲက table တိုင်းမှာ Primary Key တစ်ခု ရှိကိုရှိပါတယ်။ 
Primary Key မရှိရင် အချက်အလက်တစ်ခုစီကို တိကျသေချာစွာ ရှာဖွေဖို့၊ ပြင်ဆင်ဖို့ ဒါမှမဟုတ် ဖျက်ပစ်ဖို့ဆိုတာ မဖြစ်နိုင်ပါဘူး။

### Best Practices
Primary Key ကို ရွေးချယ်တဲ့အခါ အောက်ပါအချက်တွေကို လိုက်နာသင့်ပါတယ်။

1. တိုတိုရှင်းရှင်းနဲ့ တိကျတဲ့ ဒေတာအမျိုးအစားရှိတဲ့ Candidate Key ကို ရွေးချယ်ပါ။ (ဥပမာ- ဂဏန်းအမျိုးအစားဖြစ်တဲ့ ID field)
    - သင့်မှာ field တစ်ခုတည်းပါဝင်တဲ့ Candidate Key နဲ့ field တွေပေါင်းစပ်ထားတဲ့ Candidate Key ရှိတယ်ဆိုရင် ရိုးရှင်းတဲ့ field တစ်ခုတည်းပါဝင်တဲ့ Candidate Key ကို ရွေးချယ်ပါ။
    - field အရေအတွက် အနည်းဆုံးပါဝင်တဲ့ Candidate Key ကို အသုံးပြုတာဟာ အကောင်းဆုံးပါပဲ။ (စီမံခန့်ခွဲရလွယ်ကူပြီး performance အတွက် ပိုကောင်းပါတယ်)
2. Natural Key တွေမရှိဘူး ဒါမှမဟုတ် field ထဲက data တွေမကြာခဏပြောင်းလဲနေမယ် (static မဖြစ်ဘူး) ဆိုရင် Artificial Key ကို အသုံးပြုပါ။
3. တူညီတဲ့ data မရှိရခြင်း (uniqueness) နဲ့ NULL မဖြစ်ရခြင်း (not-null) ဆိုတဲ့ စည်းမျဉ်းတွေကို အမြဲတမ်း ကိုက်ညီတဲ့ field ကိုရွေးပါ။

### Primary key က ဘာကြောင့်အရေးကြီးတာလဲ။
- Record တွေထပ်နေတာတွေ (duplicate records) မရှိအောင် ကာကွယ်ပေးပါတယ်။
- Primary key ကို တခြား table တွေက Foreign Key အဖြစ် အသုံးပြုပြီး relations တွေ တည်ဆောက်နိုင်ပါတယ်။
- Database တွေက Primary Key တွေကို index လုပ်ထားတဲ့အတွက် အချက်အလက်ရှာဖွေတာ (lookup) ကို ပိုမိုမြန်ဆန်စေပါတယ်။
