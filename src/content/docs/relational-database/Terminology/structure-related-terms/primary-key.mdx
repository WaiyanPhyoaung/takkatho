## Primary Key ဆိုတာ ဘာလဲ?

**Primary Key** ဆိုတာ Table တစ်ခုတည်းမှာ ရှိတဲ့ Record တစ်ခုချင်းစီကို တိတိကျကျ ခွဲခြားသတ်မှတ်ပေးနိုင်တဲ့ သီးသန့် အမှတ်အသား (Unique Identifier) တစ်ခု ဖြစ်ပါတယ်။ 

Table ထဲက Candidate Key တွေ အများကြီးထဲကမှ တစ်ခုတည်းကို ရွေးချယ်ပြီး **Primary Key** အဖြစ် သတ်မှတ်ပေးရတာပါ။ Primary Key ရဲ့ အဓိက တာဝန်က Table တစ်ခုလုံးမှာရှိတဲ့ Record တွေကို သီးခြားစီ (Unique) ဖြစ်အောင် ခွဲခြားပေးဖို့နဲ့ Data တွေကို တိတိကျကျ ပြန်လည်ရှာဖွေထုတ်ယူနိုင်ဖို့ပဲ ဖြစ်ပါတယ်။

> ဥပမာ - `Employees` Table မှာ ဝန်ထမ်းတွေရဲ့ `ID` (ဥပမာ - EMP001, EMP002) ဟာ Primary Key ဖြစ်ပါတယ်။ ဝန်ထမ်းနာမည် (Name) တွေက တူနိုင်ပေမယ့် (ဥပမာ - Mg Mg နှစ်ယောက် ရှိနိုင်တယ်)၊ သူတို့ရဲ့ `ID` ကတော့ ဘယ်တော့မှ မတူနိုင်ပါဘူး။

## Primary Key ရဲ့ အရေးပါပုံ

1. **တိကျစွာ ခွဲခြားပေးနိုင်ခြင်း (Uniqueness):** Table ထဲက Record တွေကို တစ်ခုနဲ့ တစ်ခု လုံးဝ မတူညီအောင် ခွဲခြားပေးပါတယ်။ ဒါမှသာ ကိုယ်လိုချင်တဲ့ Data အတိအကျကို ရွေးချယ်ထုတ်ယူနိုင်မှာပါ။
2. **Relationships တည်ဆောက်ခြင်း:** တခြား Table တွေနဲ့ ဆက်စပ်မှု (Relationships) တည်ဆောက်တဲ့အခါ (Foreign Key တွေကတစ်ဆင့်) ဒီ Primary Key ကိုပဲ အခြေခံပြီး ချိတ်ဆက်ရပါတယ်။ 

## စည်းကမ်းချက်များ

Primary Key တစ်ခုမှာ မဖြစ်မနေ ရှိရမယ့် စည်းကမ်းချက် ၂ ခု ရှိပါတယ်။ 
ဒါတွေကို **"Primary Key စည်းမျဉ်းများ (Primary Key Rules)"** လို့ ခေါ်ပါတယ်။

- **Rule 1: Unique ဖြစ်ရမည်။** Primary Key ရဲ့ တန်ဖိုးဟာ တခြား ဘယ် Record ရဲ့ တန်ဖိုးနဲ့မှ မတူရပါဘူး။ တစ်ခုတည်း သီးသန့် (Unique) ဖြစ်ရပါမယ်။
- **Rule 2: NULL မဖြစ်ရ။** Primary Key ဟာ တန်ဖိုး မရှိတာမျိုး (Null) လုံးဝ ခွင့်မပြုပါဘူး။ အမြဲတမ်း တန်ဖိုးတစ်ခုခု ရှိနေရပါမယ်။
