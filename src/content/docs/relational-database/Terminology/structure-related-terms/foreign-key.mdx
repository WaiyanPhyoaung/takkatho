## Foreign Key

Foreign Key က table နှစ်ခုကြားမှာ relationship တစ်ခုကို ဖန်တီးပေးပါတယ်။ table နှစ်ခုကို တစ်ခုနဲ့တစ်ခု relation ချိတ်ချင်တယ်ဆိုရင် 
ပထမ table က primary key ကို ကူးယူပြီး ဒုတိယ table ထဲကို ထည့်သွင်းခြင်းဖြင့် တည်ဆောက်နိုင်ပါတယ်။ 
အဲဒီအခါ ပထမ table ထဲက primary key က ဒုတိယ table မှာ Foreign Key ဖြစ်လာပါတယ်။
Foreign Key ဆိုတဲ့ နာမည် ဘာကြောင့်ပေါ်လာတာလဲဆိုတော့ ဒုတိယ table မှာ သူ့ကိုယ်ပိုင် Primary Key ရှိနေပြီးသား ဖြစ်ပြီး
ပထမ table ကနေ ယူဆောင်လာတဲ့ Primary Key က ဒုတိယ table အတွက် အပြင်ဘက်(foreign) က လာတဲ့ key ဖြစ်တဲ့အတွက် ဖြစ်ပါတယ်။

ဥပမာ - 

**Customers table**
| `CustomerID` (PK) | Name | Email |
| ---- | ---- | ---- |
| `1` | Alice | alice@gmail.com |
| `2` | Bob | bob@gmail.com |


> `CustomerID` က Customers table အတွက် primary key ဖြစ်ပါတယ်။

**Orders table**
| OrderID (PK) | OrderDate | `CustomerID` (FK) | 
| ---- | ----- | ---- |
| 101 | 2024-06-01 | `1` | 
| 102 | 2024-05-01 | `1` | 
| 103 | 2024-07-01 | `2` | 

> `OrderID` ဟာ Orders table အတွက် `Primary Key` ဖြစ်ပါတယ်။ 
>
> ဒီနေရာမှာ `CustomerID` ကတော့ Orders table မှာ `Foreign Key` ဖြစ်ပြီး၊ Customers table ထဲက `CustomerID (PK)` ကို ရည်ညွှန်းပါတယ်။ ဒီ `Foreign Key` က အော်ဒါတစ်ခုစီကို ဘယ် customer က မှာယူခဲ့တယ်ဆိုတာကို ချိတ်ဆက်ပေးတာပဲ ဖြစ်ပါတယ်။

Customer တစ်ယောက်က အော်ဒါတစ်ခု မှာတဲ့အခါ၊ အဲဒီအော်ဒါရဲ့အချက်အလက်တွေကို Orders table မှာ သိမ်းဆည်းပါတယ်။ 
ဒီအော်ဒါကို ဘယ် customer က မှာတယ်ဆိုတာသိဖို့ Customers table ထဲက customer ရဲ့ Primary Key ဖြစ်တဲ့ CustomerID ကို Orders table ထဲမှာ Foreign Key အနေနဲ့ ထည့်သွင်းပါတယ်။
