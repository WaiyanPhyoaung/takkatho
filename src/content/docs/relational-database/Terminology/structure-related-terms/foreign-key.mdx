## Foreign Key ဆိုတာ ဘာလဲ?

**Foreign Key** ဆိုတာ Table နှစ်ခုကြားမှာ ဆက်စပ်မှု (Relationship) တည်ဆောက်ဖို့ အသုံးပြုတဲ့ Field (သို့မဟုတ် Fields များ) ကို ခေါ်တာပါ။ 

Foreign Key ရဲ့ အဓိက တာဝန်က Table တစ်ခုက Data နဲ့ နောက် Table တစ်ခုက Data ကို အချင်းချင်း ချိတ်ဆက် (Link) ပေးဖို့ပဲ ဖြစ်ပါတယ်။

> လွယ်လွယ်ပြောရရင် Table တစ်ခုရဲ့ Primary Key ကို နောက် Table တစ်ခုထဲမှာ သွားထည့်လိုက်တဲ့အခါ၊ အဲဒီ ဒုတိယ Table ထဲက Field ကို **Foreign Key** လို့ ခေါ်ပါတယ်။

### ဥပမာ - ဝန်ထမ်းနဲ့ ဌာန

| Employees Table | | | | Departments Table | |
|---|---|---|---|---|---|
| **EmployeeID (PK)** | Name | **DepartmentID (FK)** | &lt;-&gt; | **DepartmentID (PK)** | DepartmentName |
| 1 | Alice | 101 | | 101 | HR |
| 2 | Bob | 102 | | 102 | IT |

ဒီ ဥပမာမှာ -
- `Departments` Table ရဲ့ **Primary Key** က `DepartmentID` ပါ။
- အဲဒီ `DepartmentID` ကို `Employees` Table ထဲမှာ ထည့်သွင်း အသုံးပြုထားတဲ့အခါ၊ `Employees` Table ထဲက `DepartmentID` ဟာ **Foreign Key** ဖြစ်သွားပါတယ်။

ဒီလို ချိတ်ဆက်လိုက်တဲ့အတွက် "Alice ဟာ HR ဌာနက ဖြစ်တယ်" ဆိုတာကို အလွယ်တကူ သိနိုင်သွားပါတယ်။

## Foreign Key ရဲ့ အရေးပါပုံ

1. **Data မှန်ကန်စေခြင်း (Referential Integrity):** Foreign Key ရှိနေတဲ့အတွက် မရှိတဲ့ ဌာန (ဥပမာ - DepartmentID 999) ကို ဝန်ထမ်း Table ထဲ ထည့်လို့ မရတော့ပါဘူး။ Data မှားယွင်းမှုတွေကို အလိုအလျောက် ကာကွယ်ပေးပါတယ်။
2. **Relationships ဖန်တီးပေးခြင်း:** Table တစ်ခုတည်းမှာ Data တွေ အကုန်ပြွတ်သိမ်းမယ့်အစား၊ သက်ဆိုင်ရာ Table အလိုက် ခွဲသိမ်းပြီး Foreign Key တွေကတစ်ဆင့် ပြန်လည် ချိတ်ဆက်ပေးနိုင်ပါတယ်။ ဒါဟာ Relational Database ရဲ့ အဓိက အသက်သွေးကြောပါပဲ။
