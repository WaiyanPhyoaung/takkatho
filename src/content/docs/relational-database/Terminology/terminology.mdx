## မရှိမဖြစ်လိုအပ်သော database ဆိုင်ရာဝေါဟာရများ

ဆေးပညာ၊ ဥပဒေ ဒါမှမဟုတ် အင်ဂျင်နီယာဘာသာရပ်တွေလိုပဲ Relational Database Design မှာလည်း သူ့ရဲ့ကိုယ်ပိုင် ဘာသာစကားရှိပါတယ်။ 
ဒီဝေါဟာရတွေက စကားလုံးအသုံးအနှုန်းများ(jargon) သက်သက်မဟုတ်ဘဲ Database တွေ ဘယ်လိုအလုပ်လုပ်လဲဆိုတဲ့ အခြေခံသဘောတရားတွေကို သတ်မှတ်ပေးထားတာပါ။ 
Database ဝေါဟာရ (Terminology) တွေကို ကျွန်တော်တို့ နားလည်ထားခြင်းအားဖြင့် အောက်ပါ အကျိုးကျေးဇူးတွေကို ရရှိမှာ ဖြစ်ပါတယ်။

### စနစ်များကို ပိုမိုကောင်းမွန်အောင် ဒီဇိုင်းထုတ်နိုင်ခြင်း။
Rows ဆိုတာ ဘာလဲ၊ columns  ဆိုတာ ဘာလဲ၊ primary key နှင့် foreign key တွေက ဘာလဲ၊ table အချင်းချင်း ဘယ်လိုချိတ်ဆက်လို့ရလဲ (ဥပမာ- One-to-One, One-to-Many) စတဲ့ ဝေါဟာရတွေကို
ကောင်းကောင်းနားလည်ထားရင်၊ ကိုယ့်ရဲ့ data တွေကို ဘယ်လို အကောင်းဆုံးပုံစံချပြီး စီစဉ်မလဲ၊ ဘယ်လိုချိတ်ဆက်မလဲဆိုတာကို မှန်ကန်တဲ့ ဆုံးဖြတ်ချက်တွေ ချနိုင်ပါလိမ့်မယ်။ 
ဒါမှပဲ ကိုယ်တည်ဆောက်တဲ့ systemက အလုပ်လည်း ပိုမြန်၊ လိုအပ်တဲ့ အချက်အလက်တွေကို ပြန်ရှာရလည်း ပိုလွယ်ကူမှာပါ။

### ရှင်းရှင်းလင်းလင်း ဆက်သွယ်ပြောဆိုနိုင်ခြင်း။
Database နယ်ပယ်မှာ လုပ်ကိုင်နေတဲ့ တခြားဆော့ဖ်ဝဲလ်ပညာရှင်တွေနှင့် စကားပြောဆို ဆွေးနွေးတဲ့အခါ ဒီဝေါဟာရတွေကို နားလည်ထားလျှင် အချင်းချင်း အလွယ်တကူ ဆက်သွယ်ပြောဆိုနိုင်ပြီး ကိုယ်ပြောချင်တဲ့ အကြောင်းအရာကိုလည်း ထိထိရောက်ရောက် ရှင်းပြနိုင်ပါလိမ့်မယ်။ 
ဘုံဘာသာစကား (common language) တစ်ခုရှိမှသာ အလုပ်တွေက ပိုပြီးမြန်ဆန် ချောမွေ့မှာဖြစ်ပါတယ်။

### Data များကို ဝေဖန်ပိုင်းခြား စဉ်းစားနိုင်ခြင်း။
Data တွေက database ထဲမှာ ဘယ်လိုသိမ်းဆည်းလဲ၊ ကိုယ်တစ်ခုခု ရှာလိုက်ရင်၊ ဒါမှမဟုတ် data အသစ် ထပ်ထည့်လိုက်ရင် ဘယ်လိုနည်းလမ်းတွေနဲ့ လုပ်ဆောင်ပြီး ပြန်ထုတ်ပေးလဲ (processing and retrieval) ဆိုတာကို အသေးစိတ် နားလည်ထားရင် 
ပိုပြီးကောင်းတဲ့ system တွေ ဘယ်လိုဖန်တီးမလဲ၊ system တစ်ခုမှာ ပြဿနာတွေပေါ်လာရင် ဘယ်နေရာကနေ စပြီး ဖြေရှင်းရမလဲဆိုတာကို ဝေဖန်သုံးသပ်ပြီး စဉ်းစားနိုင်လာပါလိမ့်မယ်။ 
ဒါက systemရဲ့ စွမ်းဆောင်ရည် (performance) ကို မြှင့်တင်ဖို့နှင့် လုံခြုံရေး (security) ကို ပိုကောင်းအောင် လုပ်ဆောင်ဖို့အတွက် အရေးကြီးပါတယ်။
