## အရေးကြီးတဲ့ Database အသုံးအနှုန်းများ

ဆေးပညာ၊ ဥပဒေ ဒါမှမဟုတ် အင်ဂျင်နီယာဘာသာရပ်တွေလိုပဲ Database Design မှာလည်း သူ့ဘာသာစကားနဲ့သူ ရှိပါတယ်။ 
ဒီအသုံးအနှုန်းတွေက ခက်ခဲအောင်သက်သက်သုံးထားတဲ့ Jargon တွေမဟုတ်ပါဘူး။ Database တွေ ဘယ်လိုအလုပ်လုပ်လဲဆိုတဲ့ အခြေခံသဘောတရားတွေကို သတ်မှတ်ပေးထားတဲ့ အဓိကကျတဲ့ စကားလုံးတွေ ဖြစ်ပါတယ်။
Database အသုံးအနှုန်း (Terminology) တွေကို သေသေချာချာ နားလည်ထားခြင်းအားဖြင့် အောက်ကလို အကျိုးကျေးဇူးတွေကို ရရှိမှာ ဖြစ်ပါတယ်။

### System တွေကို ပိုကောင်းအောင် ဒီဇိုင်းထုတ်နိုင်ခြင်း။
Row ဆိုတာဘာလဲ၊ Column ဆိုတာဘာလဲ၊ Primary Key နဲ့ Foreign Key က ဘာတွေလဲ၊ Table အချင်းချင်း One-to-One, One-to-Many စသဖြင့် ဘယ်လိုချိတ်ဆက်မလဲဆိုတဲ့ အသုံးအနှုန်းတွေကို 
ကောင်းကောင်းနားလည်ထားမှ ကိုယ့်ရဲ့ data တွေကို အကောင်းဆုံးပုံစံနဲ့ ဘယ်လိုသိမ်းမလဲ၊ ဘယ်လိုချိတ်ဆက်မလဲဆိုတာကို မှန်မှန်ကန်ကန် ဆုံးဖြတ်နိုင်မှာပါ။
ဒါမှပဲ ကိုယ်တည်ဆောက်လိုက်တဲ့ system က အလုပ်လုပ်တာမြန်ပြီး လိုအပ်တဲ့ အချက်အလက်တွေကို ပြန်ရှာရတာလည်း ပိုလွယ်ကူမှာ ဖြစ်ပါတယ်။

### ရှင်းရှင်းလင်းလင်း ဆက်သွယ်ပြောဆိုနိုင်ခြင်း။
တခြား developer တွေနဲ့ Database အကြောင်း ဆွေးနွေးတဲ့အခါ ဒီအသုံးအနှုန်းတွေကို နားလည်ထားရင် ကိုယ်ဆိုလိုချင်တဲ့အရာကို ထိထိရောက်ရောက် ပြောပြနိုင်ပြီး 
သူများပြောတာကိုလည်း အလွယ်တကူ နားလည်နိုင်ပါလိမ့်မယ်။ 

### Data များကို ဝေဖန်ပိုင်းခြား စဉ်းစားနိုင်ခြင်း။
Data တွေက database ထဲမှာ ဘယ်လိုသိမ်းဆည်းလဲ၊ ကိုယ်တစ်ခုခုရှာလိုက်ရင် နောက်ကွယ်မှာ ဘယ်လိုအလုပ်လုပ်ပြီး အချက်အလက်တွေ ပြန်ပေါ်လာတာလဲ (processing and retrieval) ဆိုတာကို နားလည်ထားရင် ပိုကောင်းတဲ့ system တွေကို ဘယ်လိုဖန်တီးမလဲ၊
ပြဿနာတစ်ခုဖြစ်လာရင် ဘယ်နေရာကစပြီး ဖြေရှင်းရမလဲဆိုတာကို ဝေဖန်ပိုင်းခြား စဉ်းစားနိုင်ပါလိမ့်မယ်။ ဒါက System ရဲ့ စွမ်းဆောင်ရည် (Performance) ကို မြှင့်တင်ဖို့နဲ့ လုံခြုံရေး (Security) ကို ပိုကောင်းအောင်လုပ်ဖို့အတွက် အရမ်းအရေးကြီးပါတယ်။
