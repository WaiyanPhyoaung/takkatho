## အရေးကြီးတဲ့ Database အသုံးအနှုန်းများ

ဆေးပညာ၊ ဥပဒေ၊ ဒါမှမဟုတ် အင်ဂျင်နီယာ ဘာသာရပ်တွေလိုပဲ Database နယ်ပယ်မှာလည်း ကိုယ်ပိုင် အသုံးအနှုန်း (Terminology) တွေ ရှိပါတယ်။ ဒီစကားလုံးတွေက ခက်ခဲအောင် သက်သက် လုပ်ထားတဲ့ Jargon တွေ မဟုတ်ပါဘူး။ Database တွေ ဘယ်လို အလုပ်လုပ်လဲဆိုတဲ့ အခြေခံ သဘောတရားတွေကို သတ်မှတ်ပေးထားတဲ့ အဓိကကျတဲ့ စကားလုံးတွေပဲ ဖြစ်ပါတယ်။

ဒီ အသုံးအနှုန်းတွေကို သေချာနားလည်ထားခြင်းအားဖြင့် အောက်ပါ အကျိုးကျေးဇူးတွေကို ရရှိနိုင်ပါတယ်:

### 1. ပိုမိုကောင်းမွန်သော System ဒီဇိုင်း ရေးဆွဲနိုင်ခြင်း
Row ဆိုတာ ဘာလဲ၊ Column က ဘာလဲ၊ Primary Key နဲ့ Foreign Key တွေက ဘာကွာလဲ၊ Table တွေကို One-to-One, One-to-Many စသဖြင့် ဘယ်လို ချိတ်ဆက်မလဲ ဆိုတာတွေကို သေချာနားလည်ထားမှသာ ကိုယ့်ရဲ့ Data တွေကို အကောင်းဆုံးပုံစံနဲ့ ဘယ်လို သိမ်းဆည်းမလဲ၊ ဘယ်လို ချိတ်ဆက်မလဲ ဆိုတာကို မှန်မှန်ကန်ကန် ဆုံးဖြတ်နိုင်မှာပါ။ ဒါမှသာ ကိုယ်တည်ဆောက်လိုက်တဲ့ System က အလုပ်လုပ်တာ မြန်ဆန်ပြီး လိုအပ်တဲ့ အချက်အလက်တွေကို အလွယ်တကူ ရှာဖွေနိုင်မှာ ဖြစ်ပါတယ်။

### 2. ရှင်းလင်းပြတ်သားစွာ ဆက်သွယ်ပြောဆိုနိုင်ခြင်း
တခြား Developer တွေနဲ့ Database အကြောင်း ဆွေးနွေးတဲ့အခါ ဒီ အသုံးအနှုန်းတွေကို သုံးနှုန်းမှသာ ကိုယ်ပြောချင်တဲ့ ဆိုလိုရင်းကို ထိထိရောက်ရောက် ရှင်းပြနိုင်မှာပါ။ သူတို့ပြောတာကိုလည်း ကိုယ်က အလွယ်တကူ နားလည်နိုင်မှာ ဖြစ်ပါတယ်။

### 3. Data များကို ဝေဖန်ပိုင်းခြား စဉ်းစားနိုင်ခြင်း
Database ထဲမှာ Data တွေ ဘယ်လို သိမ်းဆည်းထားသလဲ၊ Data တစ်ခုကို ရှာလိုက်တဲ့အခါ နောက်ကွယ်မှာ ဘယ်လို အလုပ်လုပ်ပြီး အဖြေထွက်လာတာလဲ (Processing and Retrieval) ဆိုတာကို နားလည်ထားမှသာ ပိုကောင်းတဲ့ System တွေကို ဖန်တီးနိုင်မှာပါ။ ပြဿနာတစ်ခုခု ဖြစ်လာရင်တောင် ဘယ်နေရာကနေ စပြီး ဖြေရှင်းရမလဲဆိုတာကို ဝေဖန်ပိုင်းခြား စဉ်းစားနိုင်မှာ ဖြစ်ပါတယ်။ ဒါဟာ System ရဲ့ စွမ်းဆောင်ရည် (Performance) ကို မြှင့်တင်ဖို့နဲ့ လုံခြုံရေး (Security) ကို ပိုကောင်းအောင် လုပ်ဖို့အတွက် အလွန် အရေးကြီးပါတယ်။
