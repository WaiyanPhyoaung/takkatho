## အရေးကြီးသော Database အသုံးအနှုန်းများ

Software Engineering နဲ့ Database နယ်ပယ်မှာ ကိုယ်ပိုင် Technical Terminology (ဝေါဟာရများ) ရှိကြပါတယ်။ ဒီ စကားလုံးတွေဟာ ခက်ခဲအောင် ဖန်တီးထားတဲ့ Jargon တွေ မဟုတ်ဘဲ၊ Database တွေ ဘယ်လို အလုပ်လုပ်သလဲ ဆိုတာကို အဓိပ္ပာယ် ဖွင့်ဆိုပေးထားတဲ့ အဓိက အခြေခံ စကားလုံးများ ဖြစ်ပါတယ်။

ဒီ ဝေါဟာရတွေကို ခင်ဗျား အနေနဲ့ ပိုင်ပိုင်နိုင်နိုင် နားလည်ထားတဲ့အခါ အောက်ပါ အကျိုးကျေးဇူး ၃ ခုကို ရရှိပါမယ်:

### 1. ပိုမိုကောင်းမွန်သော System Architecture ဒီဇိုင်းဆွဲနိုင်ခြင်း
Row (Record) ဆိုတာ ဘာလဲ၊ Column (Field) ဆိုတာ ဘာလဲ၊ Primary Key နဲ့ Foreign Key တွေ ဘယ်လို ကွာခြားသလဲ၊ Table တွေကို Relationship တွေနဲ့ ဘယ်လို ချိတ်ဆက်မလဲ ဆိုတာတွေကို နားလည်ထားမှသာ Data Structure ကို အမှားအယွင်းမရှိ ဒီဇိုင်းရေးဆွဲနိုင်မှာ ဖြစ်ပါတယ်။ ဒါမှသာ System ရဲ့ Performance မြန်ဆန်ပြီး Data ရှာဖွေရလွယ်ကူလာပါမယ်။

### 2. Team ထဲမှာ ရှင်းလင်းပြတ်သားစွာ Communication လုပ်နိုင်ခြင်း
တခြား Software Engineer တွေ၊ Database Administrator (DBA) တွေနဲ့ ဆွေးနွေးတိုင်ပင်တဲ့အခါ Standard Terminology တွေကို မှန်မှန်ကန်ကန် သုံးစွဲနိုင်မှသာ ကိုယ်ပြောချင်တဲ့ စနစ်ဒီဇိုင်းကို ထိရောက်စွာ ရှင်းပြနိုင်မှာ ဖြစ်ပါတယ်။

### 3. Data Processing နဲ့ Query Performance ကို ပိုင်းခြားစိတ်ဖြာနိုင်ခြင်း
Database ထဲမှာ Data တွေ ဘယ်လို Storage ဖြစ်သွားသလဲ၊ Query တစ်ခု ခေါ်လိုက်ရင် နောက်ကွယ်မှာ ဘယ်လို Process လုပ်သလဲဆိုတာကို နားလည်ထားမှသာ System မှာ Error ဒါမှမဟုတ် Bottleneck ဖြစ်လာတဲ့အခါ အလွယ်တကူ Debug လုပ်ပြီး Performance Optimization လုပ်နိုင်မှာ ဖြစ်ပါတယ်။
