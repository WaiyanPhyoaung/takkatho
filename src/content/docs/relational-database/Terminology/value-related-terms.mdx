## Data ဆိုတာ ဘာလဲ?

**Data** ဆိုတာ ခင်ဗျား သိုလှောင်လိုတဲ့ အချက်အလက် (Information) တန်ဖိုးတစ်ခုခုကို ရည်ညွှန်းတာ ဖြစ်ပါတယ်။ User နာမည်၊ ရက်စွဲ၊ ဈေးနှုန်း၊ ကိန်းဂဏန်းတွေ ကစလို့ အရာအားလုံးဟာ Data တွေပါပဲ။ ဒီ Data တွေကို Table ထဲက သက်ဆိုင်ရာ Columns / Fields တွေမှာ ဖြည့်သွင်းပြီး သိမ်းဆည်းကြပါတယ်။

## Null ဆိုတာ ဘာလဲ?

Database မှာ **Null** ဆိုတာ "တန်ဖိုး မရှိသေးပါ" (Missing / No Value) သို့မဟုတ် "တန်ဖိုးကို မသိရသေးပါ" (Unknown Value) ဆိုတဲ့ အဓိပ္ပာယ် ဖြစ်ပါတယ်။

<Aside type="caution" title="⚠️ အရေးကြီးသော သတိပြုရန်">
  **Null ဟာ သုည (0) မဟုတ်သလို၊ စာသားအလွတ် (Empty String `""`) လည်း မဟုတ်ပါဘူး။** Null ဆိုတာ Data မရှိသေးတဲ့ အခြေအနေ သီးသန့်ကို ညွှန်ပြတာ ဖြစ်ပါတယ်။
</Aside>

## Data Types

Table ထဲက Field (Column) တစ်ခုချင်းစီမှာ သိမ်းဆည်းမယ့် Data တွေအတွက် **Data Type** (အချက်အလက် အမျိုးအစား) ကို သေချာ သတ်မှတ်ပေးရပါတယ်။ ဒါမှသာ ဘယ်လို Data မျိုးကိုပဲ လက်ခံမယ်ဆိုတာ စနစ်တကျ ထိန်းချုပ် (Validate) လုပ်နိုင်မှာ ဖြစ်ပါတယ်။

အဓိက အများဆုံး အသုံးပြုကြတဲ့ Data Type အမျိုးအစား ၃ ခုမှာ:

- **String / Character (စာသား):** Text၊ အက္ခရာနဲ့ သင်္ကေတ တွေ ပါဝင်နိုင်ပါတယ်။ (ဥပမာ - နာမည်၊ Email၊ လိပ်စာ)
- **Numeric / Integer / Float (ကိန်းဂဏန်း):** သင်္ချာဆိုင်ရာ တွက်ချက်မှု (ပေါင်း၊ နုတ်၊ မြှောက်၊ စား) ပြုလုပ်နိုင်သော ဂဏန်းများ။ (ဥပမာ - စျေးနှုန်း၊ Stock အရေအတွက်)
- **DateTime / Timestamp (ရက်စွဲနှင့် အချိန်):** ရက်စွဲ၊ လ၊ နှစ် နဲ့ အချိန် စက်ဝန်းတွေကို သိမ်းဆည်းရန်။

<Aside title="💡 မှတ်ချက်">
  Database Engine အမျိုးအစား (ဥပမာ - PostgreSQL, MySQL, SQL Server) ပေါ်မူတည်ပြီး Data Type နာမည်အခေါ်အဝေါ် အနည်းငယ် ကွာခြားနိုင်သော်လည်း (ဥပမာ - `VARCHAR` vs `TEXT`၊ `INT` vs `INTEGER`) အခြေခံ သဘောတရား ကတော့ အတူတူပဲ ဖြစ်ပါတယ်။
</Aside>
