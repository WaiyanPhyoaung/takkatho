## Data ဆိုတာ ဘာလဲ?

**Data** ဆိုတာ အချက်အလက်တွေကို ရည်ညွှန်းတာပါ။ လူနာမည်တွေ၊ နေ့စွဲတွေ၊ ကိန်းဂဏန်းတွေ ကစလို့ အရာအားလုံးက Data တွေပါပဲ။ ဒီ Data တွေကို Table တွေထဲက နေရာလွတ်တွေ (Fields) မှာ ဖြည့်သွင်းပြီး သိမ်းဆည်းကြပါတယ်။

## Null ဆိုတာ ဘာလဲ?

Database မှာ **Null** ဆိုတာ "တန်ဖိုး မရှိဘူး" (No Value) သို့မဟုတ် "တန်ဖိုးကို မသိရသေးဘူး" (Unknown Value) ဆိုတဲ့ အဓိပ္ပာယ်ပါ။ 

> [!WARNING] 
> **Null ဟာ သုည (Zero) မဟုတ်သလို၊ စာသားအလွတ် (Blank Space) လည်း မဟုတ်ပါဘူး။** Null ဆိုတာက လုံးဝ ဘာမှမရှိတဲ့ အခြေအနေကို ဆိုလိုတာပါ။ 

## Data Types

Field (Column) တစ်ခုချင်းစီမှာ သိမ်းမယ့် Data တွေဟာ **Data Type** (အမျိုးအစား) တစ်ခုစီ သတ်မှတ်ထားဖို့ လိုပါတယ်။ ဒါမှသာ ဘယ်လိုအချက်အလက်မျိုးကို လက်ခံမယ်ဆိုတာ စနစ်တကျ ထိန်းချုပ်နိုင်မှာပါ။

Data Type တွေ အများကြီးရှိပေမယ့် အဓိကအားဖြင့် အောက်ပါ ၃ မျိုးကို အများဆုံး သုံးကြပါတယ်:

- **Character (စာသား):** အက္ခရာ၊ ဂဏန်းနဲ့ အထူးသင်္ကေတ (Special Characters) တွေ အားလုံး ပါဝင်နိုင်ပါတယ်။ (ဥပမာ - နာမည်၊ လိပ်စာ၊ ဖုန်းနံပါတ်)
- **Number (ကိန်းဂဏန်း):** အပေါင်း၊ အနုတ် ဂဏန်းတွေ ပါဝင်ပါတယ်။ သင်္ချာလို တွက်ချက်မှု (ပေါင်း၊ နုတ်၊ မြှောက်၊ စား) လုပ်လို့ရမယ့် Data တွေအတွက် သုံးပါတယ်။ (ဥပမာ - စျေးနှုန်း၊ အရေအတွက်)
- **DateTime (ရက်စွဲနှင့် အချိန်):** နေ့စွဲ၊ လ၊ နှစ် နဲ့ အချိန် တွေကို သိမ်းဆည်းဖို့ သုံးပါတယ်။

<Aside title="မှတ်ချက်">
Database အမျိုးအစား (ဥပမာ - MySQL, PostgreSQL, Oracle) ပေါ်မူတည်ပြီး Data Type တွေကို ခေါ်ဝေါ်တဲ့ နာမည်လေးတွေ နည်းနည်းစီ ကွာခြားနိုင်ပါတယ်။ ဒါပေမဲ့ အခြေခံ သဘောတရား ကတော့ အတူတူပါပဲ။ 
</Aside>
