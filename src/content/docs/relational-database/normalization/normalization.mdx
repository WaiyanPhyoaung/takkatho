## Normalization ဆိုတာ ဘာလဲ?

**Normalization** ဆိုတာ Database ထဲမှာရှိတဲ့ Data တွေကို စနစ်တကျ ဖြစ်အောင် စီစဉ်ဖွဲ့စည်းတဲ့ လုပ်ငန်းစဉ် (Process) တစ်ခုပါ။ အဓိက ရည်ရွယ်ချက်ကတော့ **Data တွေ ထပ်နေတာ (Data Redundancy)** တွေကို လျှော့ချဖို့နဲ့ **Data တွေ မှားယွင်းမှု မရှိစေဖို့ (Data Integrity)** ထိန်းသိမ်းဖို့ပဲ ဖြစ်ပါတယ်။

ဒီလိုလုပ်ဖို့အတွက် ကြီးမားတဲ့ Table ကြီးတွေကို သေးငယ်တဲ့ Table လေးတွေအဖြစ် ခွဲထုတ်ပြီး၊ သူတို့ကြားမှာ Relationships (ဆက်စပ်မှုတွေ) ကို စနစ်တကျ တည်ဆောက်ပေးရပါတယ်။

## ဘာကြောင့် Normalization လုပ်ဖို့ လိုအပ်တာလဲ?

Normalization မလုပ်ထားတဲ့ Database (ဥပမာ - အကုန်လုံးကို Excel လိုမျိုး ဇယားကြီး တစ်ခုတည်းမှာ ပြွတ်သိမ်းထားတာမျိုး) တွေမှာ အောက်ပါ ပြဿနာ (Anomalies) ၃ ခု ကြုံတွေ့ရတတ်ပါတယ်။ ဒါတွေကို **Data Anomalies** လို့ ခေါ်ပါတယ်။

### 1. Update Anomaly (ပြင်ဆင်ရာတွင် ကြုံရသော ပြဿနာ)

Data တစ်ခုကို ပြင်ချင်တဲ့အခါ နေရာပေါင်းများစွာမှာ လိုက်ပြင်နေရတဲ့ ပြဿနာပါ။

**ဥပမာ:** `Supplier` တစ်ယောက်ရဲ့ ဖုန်းနံပါတ် ပြောင်းသွားတယ် ဆိုပါစို့။ Normalization မလုပ်ထားရင် အဲဒီ Supplier နဲ့ ပတ်သက်တဲ့ Order တွေ ရှိသမျှ နေရာတိုင်းမှာ ဖုန်းနံပါတ်ကို လိုက်ပြင်နေရပါမယ်။ တစ်နေရာလေး ကျန်ခဲ့တာနဲ့ Data တွေက တလွဲစီ (Inconsistent) ဖြစ်ကုန်ပါလိမ့်မယ်။

### 2. Insertion Anomaly (အသစ်ထည့်ရာတွင် ကြုံရသော ပြဿနာ)

Table ထဲမှာ လိုအပ်တဲ့ Data အပြည့်အစုံ မရှိသေးလို့ Data အသစ်တစ်ခု ထည့်ခွင့်မရတဲ့ ပြဿနာပါ။

**ဥပမာ:** ဌာနအသစ် (New Department) တစ်ခု ဖွင့်လိုက်တယ်၊ ဒါပေမဲ့ အဲဒီဌာနမှာ အလုပ်လုပ်မယ့် ဝန်ထမ်း မရှိသေးဘူး ဆိုပါစို့။ Department နဲ့ Employee ကို Table တစ်ခုတည်းမှာ ရောသိမ်းထားရင်၊ ဝန်ထမ်းနာမည် မပါဘဲ ဌာနအသစ်နာမည်ကို ထည့်လို့ မရတဲ့ ပြဿနာမျိုး ဖြစ်ပါတယ်။

### 3. Deletion Anomaly (ဖျက်ရာတွင် ကြုံရသော ပြဿနာ)

Data တစ်ခုကို ဖျက်လိုက်တဲ့အခါ မဖျက်ချင်တဲ့ တခြား အရေးကြီး Data တွေပါ ရောပါသွားတဲ့ ပြဿနာပါ။

**ဥပမာ:** သင်တန်းတစ်ခုကို တက်နေတဲ့ ကျောင်းသားက တစ်ယောက်တည်းပဲ ရှိတယ် ဆိုပါစို့။ အဲဒီကျောင်းသားက သင်တန်းကနေ ထွက်သွားလို့ သူရဲ့ Record ကို ဖျက်လိုက်တဲ့အခါ၊ အဲဒီ သင်တန်း (Course) ရဲ့ အချက်အလက်တွေပါ အကုန် ပျောက်သွားတာမျိုး ဖြစ်ပါတယ်။ 

## Normal Forms (NF) ဆိုတာ ဘာလဲ?

ဒီလို ပြဿနာတွေကို ဖြေရှင်းဖို့ Normalization ကို အဆင့်ဆင့် (Step-by-step) လုပ်ဆောင်ရပါတယ်။ အဲဒီ အဆင့်တွေကို **Normal Forms (NF)** လို့ ခေါ်ပါတယ်။ အဓိက အဆင့်တွေကတော့ -

1. First Normal Form (1NF)
2. Second Normal Form (2NF)
3. Third Normal Form (3NF)

လက်တွေ့ လုပ်ငန်းခွင် (Production) တွေမှာ **Third Normal Form (3NF)** အထိ ရောက်အောင် လုပ်နိုင်ရင် Database ဒီဇိုင်းက တော်တော်လေး ကောင်းမွန်နေပြီလို့ သတ်မှတ်ကြပါတယ်။
