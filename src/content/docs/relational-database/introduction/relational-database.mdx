## Relational Database ဆိုတာ ဘာလဲ?

**Relational Database** ဆိုတာ Data တွေကို ဇယား (Table) တွေ အနေနဲ့ သိမ်းဆည်းတဲ့ စနစ်တစ်ခုပါ။ Table တစ်ခုချင်းစီမှာ Data တွေကို အတန်း (Row) တွေနဲ့ အကွက် (Column) တွေ ခွဲပြီး စနစ်တကျ သိမ်းဆည်းထားပါတယ်။

အရေးအကြီးဆုံး အချက်ကတော့ ဒီ Table တွေဟာ တစ်ခုနဲ့ တစ်ခု သီးခြားစီ ရှိနေတာမျိုး မဟုတ်ဘဲ၊ တစ်ခုနဲ့ တစ်ခု **ဆက်စပ်မှု (Relationships)** တွေ ရှိနေတာပဲ ဖြစ်ပါတယ်။

### ဥပမာ - ကျောင်းသားနဲ့ အတန်းစာရင်း

ဥပမာ - ကျောင်းသား (Student) Table နဲ့ အတန်း (Class) Table ဆိုပြီး ရှိတယ် ဆိုပါစို့။ 
- **Student Table** မှာ ကျောင်းသားနာမည်တွေ ရှိမယ်။
- **Class Table** မှာ အတန်းနာမည်တွေ ရှိမယ်။
ဒီ Table နှစ်ခုကို ချိတ်ဆက်လိုက်ခြင်းအားဖြင့် "ဘယ်ကျောင်းသားက ဘယ်အတန်းကို တက်နေလဲ" ဆိုတာကို အလွယ်တကူ ရှာဖွေနိုင်သွားပါတယ်။ 

တကယ်လို့ Table တွေ တိုက်ရိုက် မချိတ်ဆက်ထားရင်တောင် ကြားခံ Key တွေကနေတစ်ဆင့် ဆက်သွယ်မှု (Logical Connection) ရှိနေသရွေ့ Data တွေကို အလွယ်တကူ ဆက်စပ်ပြီး ဆွဲထုတ် (Query) လို့ ရပါတယ်။

## RDBMS (Relational Database Management System)

**RDBMS** ဆိုတာကတော့ Relational Database တွေကို တည်ဆောက်ဖို့၊ စီမံခန့်ခွဲဖို့နဲ့ အသုံးပြုဖို့ ဖန်တီးထားတဲ့ Software တစ်ခု ဖြစ်ပါတယ်။ 

Relational Model ဆိုတာက Data တွေကို ဘယ်လိုဖွဲ့စည်းတည်ဆောက်မယ် (Table, Row, Relationship) ဆိုတဲ့ သဘောတရားမျှသာ ဖြစ်ပြီး၊ အဲဒီ သဘောတရားကို လက်တွေ့မှာ အလုပ်ဖြစ်အောင် အကောင်အထည်ဖော်ပေးတာကတော့ **RDBMS** ပါပဲ။ 

RDBMS ကို Database ရဲ့ "ဦးနှောက်" လို့တောင် တင်စားလို့ ရပါတယ်။ သူက Data တွေ သိမ်းဆည်းတာ၊ ပြန်ထုတ်ယူတာ၊ လုံခြုံရေးထိန်းချုပ်တာ၊ အချက်အလက် မှန်ကန်တိကျအောင် လုပ်ပေးတာ တွေကို တာဝန်ယူ ကိုင်တွယ်ပေးပါတယ်။ ဒါကြောင့် Developer တွေအနေနဲ့ နောက်ကွယ်က Data သိမ်းဆည်းမှု လုပ်ငန်းစဉ်တွေကို ခေါင်းစားစရာမလိုဘဲ၊ Application ကို ဘယ်လိုရေးမလဲ ဆိုတာကိုပဲ အာရုံစိုက်လို့ ရသွားပါတယ်။

<Aside title="Theory vs. Practice">
- **Relational Database Model:** Data တွေကို Table တွေနဲ့ သိမ်းမယ်၊ Key တွေသုံးပြီး ချိတ်ဆက်မယ် ဆိုတဲ့ **သဘောတရား (Theory)** ပါ။
- **RDBMS:** အဲဒီ သဘောတရားကို အကောင်အထည်ဖော်ပေးတဲ့ **Software/Tool (Practice)** ပါ။
</Aside>
