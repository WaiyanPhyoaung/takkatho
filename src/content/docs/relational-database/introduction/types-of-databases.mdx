## Database အမျိုးအစားများ (Types of Databases)

Database တွေကို အဓိကအားဖြင့် အမျိုးအစား ၂ မျိုး ခွဲခြားနိုင်ပါတယ်:
1. **Relational Database**
2. **Non-Relational Database**

### 1. Relational Database (SQL Database)

**Relational Database** ဆိုတာ Data တွေကို ဇယား (Table) ပုံစံနဲ့ သိမ်းဆည်းပေးတဲ့ စနစ်ပါ။ Table တစ်ခုနဲ့ တစ်ခုကြားမှာ ဆက်စပ်မှု (Relationship) တွေ ရှိပါတယ်။ ဒီ Database တွေကို စီမံဖို့အတွက် **SQL (Structured Query Language)** ကို အသုံးပြုပါတယ်။

**အားသာချက်များ:**
- Data တွေကို စနစ်တကျ (Structured) သိမ်းဆည်းနိုင်တယ်။
- Table တွေကြားထဲက ဆက်စပ်မှု (Relationships) တွေကို ရှင်းရှင်းလင်းလင်း သတ်မှတ်နိုင်တယ်။
- အချက်အလက် မှန်ကန်တိကျမှု (Data Integrity) အတွက် စည်းမျဉ်း (Rules) တွေ ချမှတ်ထားလို့ ရတယ်။

**ဥပမာ:** MySQL, PostgreSQL, Oracle, Microsoft SQL Server

### 2. Non-Relational Database (NoSQL Database)

**Non-Relational Database** ကတော့ Data တွေကို Table တွေနဲ့ မသိမ်းဘဲ တခြားပုံစံတွေ (ဥပမာ - Document, Key-Value, Graph, Wide-Column) နဲ့ သိမ်းဆည်းတဲ့ စနစ်ပါ။

**အားသာချက်များ:**
- Data ပုံစံ (Schema) က ပြောင်းလွယ်ပြင်လွယ် (Flexible) ရှိတယ်။
- ပုံစံမကျတဲ့ Data တွေ (Unstructured Data) ဥပမာ - Text တွေ၊ JSON တွေကို သိမ်းဖို့ သင့်တော်တယ်။
- Data အရေအတွက် အရမ်းများလာတဲ့အခါ (High Volume) မြန်မြန်ဆန်ဆန် အလုပ်လုပ်နိုင်တယ်။

**ဥပမာ:** MongoDB, Redis, Cassandra, Neo4j

### ဘယ်လို အခြေအနေမှာ ဘာကို ရွေးချယ်သင့်လဲ?

- **Data တွေက တိကျတဲ့ ပုံစံတစ်ခု ရှိတယ်၊ Table တွေ အချင်းချင်းလည်း ဆက်စပ်မှုတွေ ရှိတယ်ဆိုရင်:** **Relational Database** ကို ရွေးချယ်သင့်ပါတယ်။ ဥပမာ - ဘဏ်လုပ်ငန်း (Banking System)၊ စာရင်းကိုင် (Accounting) ပိုင်းတွေမှာဆိုရင် Relational Database က အကောင်းဆုံးပါ။
- **Data ပုံစံတွေက အမြဲပြောင်းလဲနေတယ်၊ တိကျတဲ့ ပုံစံမရှိဘူးဆိုရင်:** **Non-Relational Database** ကို ရွေးချယ်သင့်ပါတယ်။ ဥပမာ - Social Media Feed တွေ၊ Product Catalog တွေအတွက် ပိုသင့်တော်ပါတယ်။
