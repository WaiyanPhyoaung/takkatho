## Database System အမျိုးအစားများ

Database ဆိုတိုင်း အလုပ်လုပ်ပုံချင်း မတူပါဘူး။ တချို့ Database တွေက data တွေကို မြန်မြန်ဆန်ဆန် ပြင်ဆင်ပြောင်းလဲနိုင်ဖို့ အဓိကထား တည်ဆောက်ထားပြီး  နေ့စဉ်လုပ်ငန်းဆောင်တာတွေ အဆင်ပြေချောမွေ့အောင် ကူညီပေးပါတယ်။
နောက်ထပ် Database အမျိုးအစားတွေကျတော့ data တွေကို နက်နက်နဲနဲ ခွဲခြမ်းစိတ်ဖြာသုံးသပ်ဖို့ (analyze လုပ်ဖို့) တည်ဆောက်ထားပါတယ်။ အရင်က သိမ်းထားတဲ့ historical data တွေထဲကနေ Trend တွေ၊ Pattern တွေကို ရှာဖွေဖို့ သုံးကြပါတယ်။
ဒီတော့ Database တွေကို လက်တွေ့မှာ ဘယ်လိုအသုံးပြုလဲဆိုတဲ့အပေါ် မူတည်ပြီး အဓိကကွဲပြားသွားတဲ့ Database အမျိုးအစားနှစ်ခုဖြစ်တဲ့ 
Operational Database (OLTP) နဲ့ Analytical Database (OLAP) အကြောင်းကို လေ့လာကြည့်ရအောင်။


## 1. Operational Databases (OLTP – Online Transaction Processing)
လူစည်ကားတဲ့ စူပါမားကတ်ကောင်တာမှာ ပစ္စည်းတွေ တန်းစီဝယ်နေတာ၊ ဒါမှမဟုတ် အွန်လိုင်းရှော့ပင်းမှာ အော်ဒါတွေ တဖွဲဖွဲဝင်နေတာကို မြင်ယောင်ကြည့်လိုက်ပါ။ 
Operational Database ဆိုတာ အဲဒီလို နေ့စဉ်လုပ်ငန်းဆောင်တာတွေရဲ့ နောက်ကွယ်က အဓိက အလုပ်လုပ်ပေးနေတဲ့ အခြေခံအုတ်မြစ်ဖြစ်ပါတယ်။ 
ပစ္စည်းဝယ်တာ၊ booking လုပ်တာ၊ ဆေးရုံမှာ လူနာမှတ်တမ်း သိမ်းတာ၊ ကုန်ပစ္စည်းစာရင်းတွေကို စီမံခန့်ခွဲတာမျိုး အချိန်နဲ့တပြေးညီ အဆက်မပြတ် ဖြစ်ပေါ်နေတဲ့ 
လုပ်ငန်းဆောင်တာတွေကို ကိုင်တွယ်ဖို့အတွက် Operational Database တွေကို အထူးဒီဇိုင်းထုတ်ထားပါတယ်။

### အဓိကအချက်များ (Key Features)
* **အမြဲတမ်းပြောင်းလဲနေတဲ့ Data (Dynamic Data):** Operational Database တွေက လုပ်ငန်းဆောင်တာတွေရဲ့ လက်ရှိအခြေအနေကို အချိန်နဲ့တပြေးညီ ဖော်ပြပေးနိုင်ဖို့ အမြဲတမ်း ပြောင်းလဲနေတဲ့ အချက်အလက်တွေကို သိမ်းဆည်းထားပါတယ်။
* **CRUD လုပ်ဆောင်ချက်များ (CRUD Operations):** Operational Database တွေဟာ Create (အသစ်ထည့်ခြင်း)၊ Read (ဖတ်ခြင်း)၊ Update (ပြင်ဆင်ခြင်း) နဲ့ Delete (ဖျက်ခြင်း) ဆိုတဲ့ CRUD လုပ်ငန်းစဉ်တွေကို အမြန်ဆုံးနဲ့ အထိရောက်ဆုံးဖြစ်အောင် အဓိကထား လုပ်ဆောင်ပေးပါတယ်။
* **မြန်ဆန်စိတ်ချရတဲ့ Transactions:**  Transaction တစ်ခုချင်းစီ (ဥပမာ- ဘဏ်ငွေလွှဲတာ) ကို မြန်မြန်ဆန်ဆန်၊ မှန်မှန်ကန်ကန်နဲ့ စိတ်ချရအောင် ကိုင်တွယ်ပေးနိုင်ပါတယ်။
* **အချိန်နဲ့တပြေးညီ တိကျမှု (Real-time Accuracy):**  Data အသစ်ဝင်လာတာ၊ အပြောင်းအလဲဖြစ်တာနဲ့ တစ်ပြိုင်နက် Database ထဲမှာ မှတ်တမ်းတင်ပါတယ်။ ဒါကြောင့် သင်ကြည့်လိုက်တဲ့အချိန်တိုင်းမှာ ရရှိတဲ့ data ဟာ နောက်ဆုံးအခြေအနေ (up-to-date) ဖြစ်နေဖို့လိုအပ်တဲ့ system တွေအတွက် အရမ်းအရေးကြီးပါတယ်။

### လက်တွေ့အသုံးချမှုများ
- **E-commerce နှင့် လက်လီအရောင်းဆိုင်များ (Retail & E-commerce):** ကုန်ပစ္စည်းစာရင်းများကို စီမံခန့်ခွဲခြင်း၊ အရောင်းအဝယ် Transaction များကို စီမံဆောင်ရွက်ခြင်း၊ အွန်လိုင်းအော်ဒါများနှင့် ပို့ဆောင်မှုများကို မှတ်တမ်းတင်ခြင်း။
- **ကျန်းမာရေးစောင့်ရှောက်မှု (Healthcare):** လူနာမှတ်တမ်းများ သိမ်းဆည်းခြင်းနှင့် ပြင်ဆင်ခြင်း၊ ဆေးခန်းပြချိန် ချိန်းဆိုမှုတွေကို စီမံခန့်ခွဲခြင်း၊  ဆေးဘက်ဆိုင်ရာပစ္စည်းများ အရေအတွက်၊ ကုန်ဆုံးရက်၊ ဝင်ရောက်မှုနဲ့ ထွက်ခွာမှု မှတ်တမ်းတွေကို စနစ်တကျ မှတ်တမ်းတင်ခြင်း။
- **ဘဏ်လုပ်ငန်း (Banking):** ဘဏ်စာရင်းလက်ကျန်များ ကိုင်တွယ်ခြင်း၊ ငွေသွင်းခြင်း၊ ငွေထုတ်ခြင်းနှင့် လွှဲပြောင်းခြင်းများ စီမံဆောင်ရွက်ခြင်း။
- **ပို့ဆောင်ရေးနှင့် ထောက်ပံ့ရေး (Logistics & Supply Chain):** ကုန်ပစ္စည်းတင်ပို့မှုများ မှတ်တမ်းတင်ခြင်း၊ ဂိုဒေါင်သိုလှောင်မှုများကို စီမံခန့်ခွဲခြင်း၊ ပစ္စည်းပို့ဆောင်ရေးလမ်းကြောင်းများကို အကောင်းဆုံးတွက်ချက်ခြင်း။

### Technical Features
- **High Transaction Volume:** Transaction လုပ်ငန်းဆောင်တာ ပမာဏအများကြီးကို တစ်ပြိုင်တည်း ကိုင်တွယ်နိုင်အောင် တည်ဆောက်ထားပါတယ်။
- **Data Integrity & Concurrency:** Data တွေ အမြဲတမ်းမှန်ကန်ပြီး တစ်ခုနဲ့တစ်ခု ကိုက်ညီမှုရှိနေဖို့ (Data Consistency) အာမခံပါတယ်။ ဥပမာ- ဘဏ်အကောင့်တစ်ခုကနေ ငွေထုတ်လိုက်ရင် လက်ကျန်ငွေကို ချက်ချင်းလျှော့ချရမယ်၊ ငွေထုတ်တဲ့မှတ်တမ်းလည်း ကျန်ခဲ့ရမယ်။ 
ဒီလို data တွေအားလုံး အမြဲချိတ်ဆက်မှန်ကန်နေဖို့ လိုအပ်ပါတယ်။ ဒါ့အပြင် သုံးစွဲသူအများကြီးက data ကို တစ်ချိန်တည်းမှာ ပြဿနာမရှိဘဲ ဝင်သုံးနိုင်၊ ပြင်နိုင်အောင် (Concurrency) လုပ်ဆောင်ပေးပါတယ်။
- **Relational (SQL):**  များသောအားဖြင့် OLTP ကို Relational Database Management Systems (RDBMS) တွေနဲ့ တည်ဆောက်ကြပြီး data တွေကို စီမံခန့်ခွဲဖို့ SQL (Structured Query Language) ကို အသုံးပြုပါတယ်။

### OLTP ကို ဘယ်အချိန်မှာ အသုံးပြုမလဲ။
သင့် application က အချိန်နှင့်တပြေးညီ data အပြောင်းအလဲတွေနဲ့ လုပ်ငန်းဆောင်တာတွေကို မြန်မြန်ဆန်ဆန်နဲ့ စိတ်ချလက်ချ ကိုင်တွယ်ဖို့ လိုအပ်တယ်ဆိုရင် OLTP ကိုရွေးချယ်နိုင်ပါတယ်။

## 2. Analytical Databases (OLAP – Online Analytical Processing)

Operational Database နဲ့ ပြောင်းပြန်အနေနဲ့ Analytical Database တွေကတော့ data တွေ အများကြီးကို ခွဲခြမ်းစိတ်ဖြာသုံးသပ်ပြီး အတွင်းကျကျ နားလည်နိုင်ဖို့ တည်ဆောက်ထားပါတယ်။
Historical Data ပမာဏအများကြီးကို ခွဲခြမ်းစိတ်ဖြာမှုလုပ်ပြီး စီးပွားရေးလုပ်ငန်းတွေနှင့် သုတေသီများအတွက် ခေတ်ရေစီးကြောင်းများ (Trend) ရှာဖွေနိုင်ဖို့၊ Data Behavior တွေအကြောင်း နားလည်နိုင်ဖို့ ကူညီပေးပါတယ်။
<br></br>
OLAP Database တွေက အောက်ကလိုမျိုး ရှုပ်ထွေးတဲ့ မေးခွန်းတွေကို ဖြေဖို့အတွက် အသင့်တော်ဆုံးပါပဲ။
- "ပြီးခဲ့တဲ့ ဘဏ္ဍာရေးနှစ်မှာ ကျွန်တော်တို့ရဲ့ ရောင်းအားစွမ်းဆောင်ရည် ဘယ်လိုတိုးတက်ပြောင်းလဲလာလဲ။"
- "လွန်ခဲ့တဲ့ သုံးလအတွင်း ဘယ်ကုန်ပစ္စည်းအမျိုးအစားတွေက စိတ်ဝင်စားမှု အများဆုံးရခဲ့သလဲ။"
- "ကျွန်တော်တို့ရဲ့ user အသစ်အများစုက ဘယ်ဒေသက လာကြတာလဲ။"

### အဓိကအချက်များ (Key Features)
- **မပြောင်းလဲသော/သမိုင်းကြောင်းဆိုင်ရာ ဒေတာ (Static/Historical Data):** Analytical Database တွေက မှတ်တမ်းတင်ပြီးသွားရင် အလွယ်တကူ ပြန်မပြောင်းတော့တဲ့ historical data အများကြီးကို အဓိကထား သိမ်းဆည်းပါတယ်။
- **ရှုပ်ထွေးတဲ့ Query တွေအတွက် အထူးပြုလုပ်ထားခြင်း (Optimized for Complex Queries):** Data တွေကို စုစည်းတာ (aggregating)၊ အကျဉ်းချုပ်တာ (summarizing) နဲ့ ချိတ်ဆက်တာ (joining) စတဲ့ ရှုပ်ထွေးတဲ့ query တွေကို ထိထိရောက်ရောက် လုပ်ဆောင်နိုင်အောင် ဒီဇိုင်းထုတ်ထားပါတယ်။
- **ထိုးထွင်းသိမြင်မှုများ (Powerful Insights):**  သုံးစွဲသူတွေကို data အပေါ် ရှုထောင့်ပေါင်းစုံကနေ ကြည့်ရှုနိုင်အောင် (slice and dice)၊ အသေးစိတ်တူးဖော်လေ့လာနိုင်အောင် (drill-down) လုပ်ပေးနိုင်ပြီး business အတွက် အရေးပါတဲ့ report တွေ ဖန်တီးပေးနိုင်ပါတယ်။

### လက်တွေ့အသုံးချမှုများ
- **စျေးကွက်ခွဲခြမ်းစိတ်ဖြာမှု (Marketing Analytics):** ယခင် Campaign တွေရဲ့ ထိရောက်မှု၊ Customer ခွဲခြားသတ်မှတ်ခြင်းနှင့် အနာဂတ် ခေတ်ရေစီးကြောင်း(Trend)များကို ခန့်မှန်းခြင်း။
- **သိပ္ပံဆိုင်ရာ သုတေသန (Scientific Research):** အချိန်နဲ့အမျှ ပြောင်းလဲတဲ့ data တွေကို လေ့လာခြင်း၊ large-scale simulationsနှင့် စမ်းသပ်မှုရလဒ်တွေထဲက Pattern တွေကို ဖော်ထုတ်ခြင်း။
- **အမှုဆောင်အဆင့် ဆုံးဖြတ်ချက်ချမှတ်ခြင်း (Executive Decision-Making):**  ကုမ္ပဏီစွမ်းဆောင်ရည်၊ စျေးကွက်အခြေအနေတွေကို ပြသတဲ့ Dashboard၊ Report တွေ ဖန်တီးပြီး မဟာဗျူဟာမြောက် စီမံကိန်းတွေ ရေးဆွဲရာမှာ ကူညီပေးပါတယ်။
- **စီးပွားရေး ထိုးထွင်းသိမြင်မှု (Business Intelligence - BI):** လုပ်ငန်းတစ်ခုလုံးမှာ data ကိုအခြေခံတဲ့ ဆုံးဖြတ်ချက်တွေ ချနိုင်ဖို့အတွက် လိုအပ်တဲ့ BI Tool တွေရဲ့ စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးပါတယ်။

## Relational Database

### Relational Database ဆိုတာ ဘာလဲ။

Relational Database Model ဆိုတာက ဒီနေ့ခေတ် Database Management မှာ အသုံးအများဆုံး Database Model ဖြစ်ပါတယ်။
ဘဏ္ဍာရေး(Finance)၊ ကျန်းမာရေးစောင့်ရှောက်မှု(Healthcare)၊ ပညာရေး(Education) ဒါမှမဟုတ် လက်လီရောင်းဝယ်ရေး(Retail) စတဲ့ နယ်ပယ်အသီးသီးမှာ
data တွေကို စီမံခန့်ခွဲဖို့ relational database ကိုပဲ သုံးနေကြပါတယ်။

> အဓိက idea ကတော့ data တွေကို Relations အဖြစ် စုစည်းထားတာပါပဲ။ အဲဒီ Relations တွေကို Table တွေလို့လည်း မြင်နိုင်ပါတယ်။ ဒီ Table တွေကို အောက်ပါအရာတွေနဲ့ ဖွဲ့စည်းထားပါတယ်။
> - Rows (Records သို့မဟုတ် tuples လို့လည်း ခေါ်ပါတယ်): အရာဝတ္ထုတစ်ခု ဒါမှမဟုတ် အချက်အလက်တစ်ခု (Entity) ကို ကိုယ်စားပြုပါတယ်။ ဥပမာ - လူတစ်ယောက်၊ ပစ္စည်းတစ်ခု။
> - Columns (attributes သို့မဟုတ် fields လို့လည်း ခေါ်ပါတယ်): Data ရဲ့ အမျိုးအစား characteristic ကို ကိုယ်စားပြုပါတယ်။ ဥပမာ - နာမည်၊ အသက်၊ အရောင်။
>
> Table တိုင်းမှာ Primary Key တစ်ခုစီရှိပါတယ်။
> Primary key ဆိုတာက table မှာရှိတဲ့ row တစ်ခုချင်းစီကို တိတိကျကျ ညွှန်းဆိုနိုင်တဲ့ column တစ်ခု (ဒါမှမဟုတ် column အစုအဝေးတစ်ခု) ဖြစ်ပါတယ်။

### Table တစ်ခုနဲ့တစ်ခု ဘယ်လိုချိတ်ဆက်ထားသလဲ။

Relational Database တွေရဲ့ အဓိကစွမ်းဆောင်ရည်ကတော့ သီးခြားစီရှိနေတဲ့ table တွေကို relationships တွေသုံးပြီး ချိတ်ဆက်နိုင်တာပဲဖြစ်ပါတယ်။
ဒီလိုချိတ်ဆက်ဖို့အတွက် table နှစ်ခုမှာရှိတဲ့ တူညီတဲ့ fields (matching values) တွေကို အသုံးပြုပါတယ်။(ဥပမာ- Primary Key နှင့် Foreign Key)

**ဥပမာလေးနဲ့ ကြည့်ရအောင်။**



စာသင်ကျောင်းတစ်ခုရဲ့ data တွေကို မှတ်ထားတယ်ဆိုပါစို့။

ကျောင်းသားစာရင်း (Student Table) ဆိုတဲ့ table တစ်ခုမှာ ကျောင်းသားတစ်ယောက်ချင်းစီတိုင်းအတွက် ကျောင်းသားကုတ်နံပါတ် (Primary Key) တွေရှိပါမယ်။ (ဥပမာ - S001, S002 စသည်ဖြင့်)။
သင်တန်းအပ်နှံမှုစာရင်း (Course Enrollment Table) နောက်ထပ် table တစ်ခုမှာ ဘယ်ကျောင်းသားက ဘယ်သင်တန်းတက်တယ်ဆိုတာ မှတ်ထားချင်တယ်ဆိုပါတော့။ ဒီအခါမှာ
သင်တန်းအပ်နှံမှုစာရင်း table ထဲမှာ ကျောင်းသားတွေကို မှတ်သားဖို့ ကျောင်းသားစာရင်း Table ရဲ့ primary key ဖြစ်တဲ့ ကျောင်းသားကုတ်နံပါတ် ကို Foreign Key အနေနဲ့ လာထည့်ပေးရပါမယ်။ 
ဒီလိုချိတ်ဆက်ထားခြင်းအားဖြင့် "ဘယ်ကျောင်းသားက ဘယ်သင်တန်းတက်တယ်" ဆိုတာကို table နှစ်ခုကို ဆက်စပ်ကြည့်ပြီး အလွယ်တကူ သိနိုင်သွားမှာပါ။ ဒါဟာ Relational Database တွေရဲ့ အဓိကအားသာချက်ပါပဲ။

အသုံးများသော Relationship အမျိုးအစား သုံးခုရှိပါတယ်။

- **One-to-One (1:1)** <br />
Table A ထဲက Record တစ်ခုက Table B ထဲက Record တစ်ခုတည်းနဲ့ပဲ ဆက်စပ်နေတာ။<br />
ဥပမာ- User တစ်ယောက်မှာ Profile တစ်ခုပဲရှိတာမျိုး။ <br />
- **One-to-Many (1:N)** <br />
Table A ထဲက Record တစ်ခုက Table B ထဲက Record အများကြီးနဲ့ ဆက်စပ်နေတာ။ <br />
ဥပမာ- Customer တစ်ယောက်က Order တွေ အများကြီးမှာယူတာမျိုး။ <br />
- **Many-to-Many (M:N)** <br />
Table A ထဲက Record တွေက Table B ထဲက Record အများကြီးနဲ့ ဆက်စပ်နိုင်သလို၊ Table B ဘက်ကလည်း Table A ထဲက Record အများကြီးနဲ့ ပြန်ဆက်စပ်နေတာ။ <br />
ဥပမာ- ကျောင်းသားတွေအများကြီးက သင်တန်းတွေအများကြီးကို တက်ရောက်နေတာမျိုး။ <br />

ဒီ relationships တွေကနေတစ်ဆင့် ချိတ်ဆက်ထားတဲ့ table ပေါင်းများစွာထဲက အချက်အလက်တွေကို လိုအပ်သလို ရှာဖွေကြည့်ရှုနိုင်ပါတယ်။ 
တကယ်လို့ အဲဒီ table တွေက တိုက်ရိုက်ချိတ်ဆက်မထားရင်တောင်မှ ကြားခံ key တွေကနေတစ်ဆင့် ဆက်သွယ်မှု (logical connection) ရှိနေသရွေ့ data တွေကို ဆက်စပ်ရှာဖွေနိုင်ပါတယ်။

## Relational Database Management System

Relational Database Management System (RDBMS) ဆိုတာ relational database တစ်ခုကို ဖန်တီးဖို့၊ စီမံခန့်ခွဲဖို့၊ ပြင်ဆင်ဖို့နဲ့ အပြန်အလှန် အလုပ်လုပ်ဖို့ ကူညီပေးတဲ့ ဆော့ဖ်ဝဲ (Software) တစ်ခုဖြစ်ပါတယ်။  
Relational Model က data တွေကို သဘောတရားအရ ဘယ်လိုဖွဲ့စည်းတည်ဆောက်ထားတယ် (Table, Row, Relationship) ဆိုတာကို ဖော်ပြပေးပြီး၊ RDBMS က အဲဒါတွေကို လက်တွေ့မှာ အလုပ်ဖြစ်အောင် လုပ်ဆောင်ပေးတဲ့ tool ဖြစ်ပါတယ်။
RDBMS ကို database ရဲ့ "ဦးနှောက်" လို့ မြင်လို့ရပါတယ်။ Data သိမ်းဆည်းတာ၊ ပြန်ထုတ်တာ၊ လုံခြုံရေး၊ တိကျမှန်ကန်မှု စတာတွေကို တာဝန်ယူကိုင်တွယ်ပေးပါတယ်။ 
ဒါကြောင့် developer တွေနဲ့ application တွေအနေနဲ့ data ကို ဘာလုပ်ချင်လဲဆိုတာကိုပဲ အာရုံစိုက်ဖို့လိုပြီး၊ နောက်ကွယ်မှာ data တွေ ဘယ်လိုသိမ်းဆည်းထားလဲဆိုတာကို ခေါင်းစားနေစရာ မလိုတော့ပါဘူး။

>Relational Database Model က သဘောတရား (Theory) ကို ပေးပါတယ်။ ဥပမာ - data တွေကို table တွေနဲ့ သိမ်းမယ်၊ key တွေသုံးပြီး ချိတ်ဆက်မယ်၊ data မှန်ကန်ဖို့ စည်းမျဉ်းတွေ သတ်မှတ်မယ် စသဖြင့်ပေါ့။
>အဲဒီသဘောတရားတွေကို လက်တွေ့အကောင်အထည်ဖော်ပေးတာက RDBMS ဆိုတဲ့ tool ပါပဲ။ RDBMS က relational database တွေကို ထိထိရောက်ရောက်၊ လုံလုံခြုံခြုံ တည်ဆောက်ဖို့၊ စီမံခန့်ခွဲဖို့နဲ့ အသုံးပြုဖို့ ကူညီပေးပါတယ်။
