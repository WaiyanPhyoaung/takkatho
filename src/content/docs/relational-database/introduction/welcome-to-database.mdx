## Database တွေ ဘာကြောင့် အရေးပါနေရတာလဲ။

သင်တို့အကြိုက်ဆုံး Application တွေက သင့်အကြောင်း (သင်ကြိုက်တဲ့အရာတွေ၊ သင့်လှုပ်ရှားမှု (activity) တွေ၊ သင့် friends တွေ)  ဘယ်လိုများ အကုန်မှတ်မိနေလဲဆိုတာ စဉ်းစားဖူးပါသလား။ အဖြေကတော့ Database ကြောင့်ပဲဖြစ်ပါတယ်။

## Database ဆိုတာ ဘာလဲ။
database ဆိုတာ အချက်အလက်တွေကို စနစ်တကျသိမ်းဆည်းထားတဲ့ ဒစ်ဂျစ်တယ် ဖိုင်တွဲကြီးတစ်ခုလို့ မြင်ယောင်ကြည့်ပါ။ ဒီဖိုင်တွဲကြီးက အရမ်းကို စနစ်တကျရှိတယ်၊ သိမ်းဆည်းထားတဲ့အချက်အလက်တွေကိုလည်း မြန်မြန်ဆန်ဆန်ပြန်လည်ရှာဖွေနိုင်တယ်၊ တခြားသူတွေကိုလည်း အလွယ်တကူ မျှဝေနိုင်တယ်။ ဒါဟာ database ရဲ့ အဓိပ္ပာယ်ပါပဲ။
အနှစ်ချုပ်ပြောရရင် 
> **Database ဆိုသည်မှာ အဖွဲ့အစည်းတစ်ခု (သို့မဟုတ်) အဖွဲ့အစည်းရဲ့ လုပ်ငန်းစဉ်တစ်ခုအကြောင်းကို စီမံခန့်ခွဲဖို့အတွက် စနစ်တကျစုစည်းထားတဲ့ Data တွေပဲဖြစ်ပါတယ်။** 
>
စက္ကူဖိုင်တွေနဲ့ပဲဖြစ်ဖြစ်၊ ခေတ်မီ Software တွေနဲ့ပဲဖြစ်ဖြစ်၊ သတ်မှတ်ထားတဲ့ ရည်ရွယ်ချက်တစ်ခုခုအတွက် Data တွေကို စနစ်တကျ စုဆောင်းသိမ်းဆည်းနေတယ်ဆိုရင် Database နဲ့ အလုပ်လုပ်နေတာပါပဲ။
ဒီနေ့ခေတ်မှာ သင်ကိုယ်တိုင်တောင် မသိလိုက်ဘဲနဲ့ database တွေနဲ့ အကြိမ်ပေါင်းများစွာ ထိတွေ့ခဲ့ဖူးပါလိမ့်မယ်။ (ဥပမာ - Online ကနေ ပစ္စည်းတွေ ဝယ်ခဲ့တာမျိုး)

### Databases Around You: Practical Use Cases from Daily Life
#### ဒီ Module ထဲက အကြောင်းအရာတွေကို ပိုပြီးနားလည်လာအောင် လက်တွေ့ကျတဲ့ ဥပမာလေးတွေနဲ့ ထပ်ပြီးလေ့လာကြည့်ရအောင်။
#### 1. The Local Cafe: From Notepad to Customer Management System
သင့်မှာ ကော်ဖီဆိုင်လေးတစ်ဆိုင် ပိုင်ဆိုင်တယ်လို့ စိတ်ကူးကြည့်လိုက်ပါ။ စတင်ဖွင့်လှစ်ချိန်မှာတော့ Customer တွေရဲ့ Order တွေကို မှတ်စုစာအုပ်လေးထဲမှာ ရိုးရိုးလေးရေးမှတ်ထားပြီး ပုံမှန်လာတဲ့သူတွေကိုတောင် မျက်မှန်းတန်းမိနေပါလိမ့်မယ်။
သို့သော်လည်း သင့်ကော်ဖီဆိုင် လူကြိုက်များလာတာနှင့်အမျှ ဒီစနစ်က ရှုပ်ထွေးလာပါလိမ့်မယ်။ သင့်အနေနဲ့ အောက်ပါအချက်အလက်တွေကို စီမံခန့်ခွဲဖို့ ပိုမိုစနစ်တကျတဲ့ နည်းလမ်းတစ်ခု လိုအပ်လာပါလိမ့်မယ်။
* Customer ကြိုက်နှစ်သက်မှုများ- "ဒီဖောက်သည်က almond milk ကြိုက်တာလား၊ oat milk ကြိုက်တာလား။"
* Loyalty Points များ- "ဒီဖောက်သည်မှာ ကော်ဖီအခမဲ့ သောက်နိုင်ဖို့အတွက် အမှတ်ဘယ်လောက်ရှိနေပြီလဲ။"
* မှာယူမှုမှတ်တမ်းများ- "ဒီဖောက်သည်ရဲ့ အမြဲတမ်းမှာနေကျ မုန့်ကဘာလဲ။"
* တုံ့ပြန်ချက်များ- "ဒီဖောက်သည်က အသစ်လုပ်ထားတဲ့ ပေါင်မုန့်နဲ့ပတ်သက်ပြီး ဘာများတုံ့ပြန်ချက်ပေးခဲ့သလဲ။"

#### 2. Your Digital Life: Social Media and Beyond
သင့် Social Media ပရိုဖိုင်ကိုစဉ်းစားကြည့်ပါ။ သင့်အသုံးပြုသူအမည် (username)၊ ကိုယ်ရေးအကျဉ်း (biography)၊ ပို့စ်များ၊ သင် follow လုပ်ထားသူများ၊ သင်ရရှိသော like များနှင့် comment များအားလုံးက
 စီမံခန့်ခွဲရမယ့် အချက်အလက်(information)တွေပဲ ဖြစ်ပါတယ်။ 
Database ကတော့ အဲဒီအချက်အလက်တွေကို သိမ်းဆည်းပေးတဲ့ မမြင်ရတဲ့ စက်(Engine) တစ်ခုပါပဲ။
* အသုံးပြုသူ ပရိုဖိုင်များ (User profiles):  Username၊ Bio၊ Follower စာရင်းများ။
* ပို့စ်များနှင့် ဗီဒီယိုများ (Posts and videos): ဘယ်သူတင်ခဲ့တာလဲ၊ ဘယ်အချိန်ကလဲ၊ သူတို့၏ အကြောင်းအရာများ။
* မှတ်ချက်များနှင့် Like များ (Comments and likes): User၊ Post တွေနှင့် ချိတ်ဆက်ထားမှုများ။ <br />

Database မရှိရင် Platform တစ်ခုအနေနဲ့ သင့်စိတ်ကြိုက် NewsFeed ကို ပြသဖို့၊ Interaction အသစ်တွေအကြောင်း အသိပေးဖို့၊ Connection အသစ်တွေကို အကြံပြုဖို့ဆိုတာ မဖြစ်နိုင်သလောက်ပါပဲ။

#### 3. Gaming Worlds: Saving Your Progress and More
ဒီဘက်ခေတ် Video Game တွေမှာ ကစားသမားတွေရဲဲ့ Account တွေ၊ Character တွေ၊ ရရှိထားတဲ့ Achievement တွေ၊ ပစ္စည်းများသိမ်းထားတဲ့ Inventory တွေ၊ ပြီးတော့ Score တွေရှိကြတယ်။ 
ဒါတွေ၏နောက်ကွယ်မှာ Database တွေက အောက်ပါအချက်အလက်တွေကို သိမ်းဆည်းထားပါတယ်။
* ကစားသမား Profile များ - Account အသေးစိတ်အချက်အလက်များ၊ Character အချက်အလက်များ။
* ဂိမ်း Progress များ - သိမ်းဆည်းထားတဲ့ အခြေအနေ၊ ပြီးစီးခဲ့တဲ့ Level။
* Leaderboard များ - ကစားသမား အဆင့်သတ်မှတ်ချက်တွေနဲ့ Score များ။
* ဂိမ်းအတွင်း ဝယ်ယူမှုများ - Virtual Item တွေ ဝယ်ယူခဲ့တဲ့ မှတ်တမ်းများ။ <br />

သင့် Progress တွေကို အလိုအလျောက် သိမ်းဆည်းပေးတယ်။ သင်၏ Leaderboard အဆင့်ကိုလည်း တိတိကျကျ တွက်ချက်ပေးတယ်။ ပြီးတော့ သင်ဂိမ်းကို ဘယ်နေရာမှာပဲ ရပ်ခဲ့သည်ဖြစ်စေ၊ တခြား Device တစ်ခုမှာပဲဖြစ်စေ ဂိမ်းကိုဝင်ရောက်၍ လက်ရှိနေရာကနေပဲ ပြန်လည်စတင်နိုင်တာတွေဟာ database ကြောင့်ပဲဖြစ်ပါတယ်။

#### Key Takeaways
data အမျိုးမျိုးကို သိမ်းဆည်းပြီး ပြန်လည်ကြည့်ရှုနိုင်သော၊ data ကိုလိုအပ်သလို စီမံခန့်ခွဲနိုင်သော app တစ်ခုကို တွေ့ရမယ်ဆိုရင် – database ရဲ့ လုပ်ဆောင်ချက်ကို မြင်တွေ့နေတာပါပဲ။

### Activity: Spot the database
နားလည်မှုကို စမ်းသပ်ကြည့်ကြပါစို့။ သင်နေ့စဉ်အသုံးပြုနေသော App တွေကို စဉ်းစားကြည့်ပြီး Database တစ်ခုက မည်သည့်နေရာတွေမှာ အချက်အလက်(information) များကို သိမ်းဆည်းခြင်းနှင့် စီမံခန့်ခွဲခြင်းတို့လုပ်နေမည်ကို ခန့်မှန်းကြည့်ပါ။

အောက်ပါဥပမာများကို စဉ်းစားကြည့်ပါ။  <br />

**Instagram: ၎င်း၏ Database က ဘယ်လို Data အမျိုးအစားတွေကို စီမံခန့်ခွဲတယ်လို့ သင်ထင်ပါသလဲ။** <br />
*Possible Answers: Users များ၊ Posts များ၊ Likes များ၊ Comments များ၊ Stories များ၊ Follower ချိတ်ဆက်မှုများ။* 

**Amazon: Amazon၏ Database က မည်သည့်အချက်အလက်များကို လိုအပ်ပါသလဲ။** <br />
*Possible Answers: ထုတ်ကုန်များ၊ customer များ၊ အော်ဒါများ၊ ငွေပေးချေမှု အချက်အလက်များ၊ ပို့ဆောင်မည့်လိပ်စာများ၊ ရှာဖွေမှုမှတ်တမ်း (search history)။*

**အွန်လိုင်းဂိမ်းများ (ဥပမာ- Fortnite, Mobile Legends): ဤဂိမ်းများ လုပ်ဆောင်ရန် မည်သည့်ဒေတာများက အရေးကြီးပါသလဲ။** <br />
*Possible Answers: ကစားသမားများ၊ ရမှတ်များ၊ အောင်မြင်မှုများ (achievements)၊ အဆင့်များ (ranks)၊ ဂိမ်းတွင်းသုံးငွေ (in-game currency)၊ ဇာတ်ကောင် စိတ်ကြိုက်ပြင်ဆင်မှု (character customization)၊ သူငယ်ချင်းစာရင်းများ။*

**အစားအသောက်ပို့ဆောင်ရေးအက်ပ် (ဥပမာ- FoodPanda, GrabFood): ဤနေရာတွင် မည်သည့်အချက်အလက် အမျိုးအစားများကို သိမ်းဆည်းပြီး ချိတ်ဆက်ထားပါသလဲ။** <br />
*Possible Answers:  မီနူးများ၊ စားသောက်ဆိုင်များ၊ ပို့ဆောင်သူများ (riders)၊ ဖောက်သည် မှာယူမှုများ၊ ပို့ဆောင်မည့်လိပ်စာများ၊ ငွေပေးချေမှု အသေးစိတ်အချက်အလက်များ၊ အဆင့်သတ်မှတ်ချက်များ (ratings)။*
