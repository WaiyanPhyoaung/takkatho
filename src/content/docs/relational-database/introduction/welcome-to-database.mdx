## Database ဆိုတာ ဘာလဲ?

**Database** ဆိုတာ ခင်ဗျား သိမ်းဆည်းချင်တဲ့ Data တွေကို စနစ်တကျ ပြုပြင်ဖန်တီး၊ စုစည်းပြီး သိုလှောင်ပေးထားတဲ့ နေရာတစ်ခု ဖြစ်ပါတယ်။ အပြင်ကမ္ဘာနဲ့ ဥပမာပေးရရင် စာရွက်စာတမ်းတွေကို သပ်သပ်ရပ်ရပ် ဖိုင်တွဲပြီး သိမ်းထားတဲ့ စာရွက်စာတမ်းစင် (Filing Cabinet) ကြီးနဲ့ တူပါတယ်။

ယနေ့ခေတ်မှာတော့ Database အများစုကို Digital Format နဲ့ ကွန်ပျူတာတွေ၊ Server တွေပေါ်မှာ စနစ်တကျ သိုလှောင် သိမ်းဆည်းလာကြပါတယ်။

### Database ရဲ့ အဓိက အခန်းကဏ္ဍ

Modern Software တွေ၊ Website တွေနဲ့ Mobile Application တော်တော်များများဟာ Data တွေကို အခြေခံပြီး အလုပ်လုပ်ကြရတာပါ။ User တွေရဲ့ အချက်အလက်တွေ (ဥပမာ - နာမည်၊ Email၊ Password၊ Order History) ကို စိတ်ချလက်ချ သိမ်းဆည်းထားဖို့ လိုအပ်သလို၊ လိုအပ်တဲ့အခါ ပြန်လည် ရှာဖွေတာ၊ အသစ် ထပ်ထည့်တာ၊ ပြင်ဆင်တာ ဒါမှမဟုတ် ဖျက်ပစ်တာ (CRUD Operations) တွေကို မြန်မြန်ဆန်ဆန် လုပ်ဆောင်နိုင်ဖို့ **Database** က မရှိမဖြစ် လိုအပ်ပါတယ်။

ပုံမှန်အားဖြင့် Software Application တစ်ခုမှာ အဓိက အစိတ်အပိုင်း ၃ ခု ပါဝင်လေ့ရှိပါတယ်:

1. **User Interface (UI):** User တွေ မျက်စိနဲ့ မြင်တွေ့ အသုံးပြုရတဲ့ အပိုင်း (Frontend)
2. **Business Logic:** Application ဘယ်လို အလုပ်လုပ်ရမယ်ဆိုတာကို ထိန်းချုပ်ထားတဲ့ Logic အပိုင်း (Backend)
3. **Database:** Data တွေကို ကာလရှည် စနစ်တကျ သိမ်းဆည်းပေးထားတဲ့ Storage အပိုင်း

ဒီအစိတ်အပိုင်း ၃ ခုမှာ **Database** ဟာ Application တစ်ခုလုံးရဲ့ အသက်သွေးကြော (Core Component) ပဲ ဖြစ်ပါတယ်။ Database မပါဘဲနဲ့ ခင်ဗျားရဲ့ Application က Data တွေကို အချိန်ကြာမြင့်စွာ မှတ်မိနေနိုင်မှာ မဟုတ်ပါဘူး။
